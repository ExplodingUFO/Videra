# Phase 189 Plan: Shared Host-Seam Tightening for WPF

## Goal

Introduce one small shared host-input synchronization seam so Avalonia and the repository-only WPF smoke proof stop assembling the same native-host sequence manually.

## Tasks

1. Tighten the core seam.
   - Add one internal `RenderSessionOrchestrator` path that synchronizes backend preference, bound handle, size, and render scale together.
   - Keep existing public/internal point methods (`Attach`, `BindHandle`, `Resize`) intact.
   - Avoid backend-contract widening and avoid new public host abstractions.

2. Reuse the seam on the shipped Avalonia viewer path.
   - Route the native-handle-created and sized-host synchronization path through the new shared seam.
   - Keep the no-size and handle-destroy paths explicit.

3. Reuse the seam on the repository-only WPF proof path.
   - Replace the hand-built attach/bind/resize sequence in `WpfSmoke` with the same shared seam.
   - Keep `WpfSmoke` a thin proof host rather than a reusable adapter layer.

4. Verify with focused tests.
   - Integration coverage for the core orchestrator shared host path.
   - Integration/configuration evidence that Avalonia bridge and WPF smoke now consume that shared seam.

## Acceptance

- One narrow shared host-sync seam exists in `Videra.Core`.
- Avalonia and `WpfSmoke` both use that seam for the sized, bound native-host path.
- No second public UI package line or generalized multi-shell abstraction is introduced.
