# Requirements

Active milestone: `v2.8 Release Candidate Contract Closure`

Release-candidate requirement numbering starts at `RC-01`.

- [x] `RC-01` Before implementation starts, lock `v2.8` to release-candidate contract closure for the existing public package/API/release/docs surface and freeze explicit non-goals around runtime/rendering/chart/backend breadth, extra UI adapters, compatibility shims, migration adapters, and actual public publishing.
- [x] `RC-02` The shipped public package/API surface has a deterministic drift guard that can run in repo/CI and makes intentional API changes reviewable without adding compatibility or transition layers.
- [x] `RC-03` Public and preview release workflows have a non-publishing dry-run/evidence path that validates tag-derived versions, package set, symbols, README/license metadata, package-size budgets, and release asset expectations.
- [x] `RC-04` README, capability matrix, package matrix, support matrix, release policy, releasing runbook, changelog/release-note guidance, localized docs, and repository tests describe the same release-candidate boundary.

## Out of Scope

| Item | Excluded from `v2.8` |
|------|------------------------|
| New runtime, renderer, material, glTF, backend, or platform features | Not part of this milestone. |
| New chart families or `SurfaceCharts` feature breadth | Not part of this milestone. |
| Extra public UI adapters | Not part of this milestone. |
| Compatibility shims, migration adapters, or fallback layers | Not part of this milestone. |
| Actual `nuget.org` or GitHub Packages publication | Not part of this milestone. |
| Release tag creation | Not part of this milestone. |
