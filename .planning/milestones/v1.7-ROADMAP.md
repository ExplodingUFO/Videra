## Overview

`v1.7` 收口了 viewer 的 scene realisation / residency / upload pipeline，而不是继续扩 demo 或再拆更大的 public architecture。`v1.6` 建好的 authoritative `SceneDocument`、backend-neutral import、rehydration baseline、以及 direct device/surface seam 在这轮被进一步收成一条更稳定的内部运行时主线：document 只管 scene truth，delta 只管变化，residency 只管 GPU 状态，upload queue 只在 frame prelude 里按预算 drain，backend recovery 只围绕 retained truth 做 dirty reupload。

这轮 milestone 先补了 dedicated Avalonia/runtime test harness，再补资产指标与 scene contract，随后完成 `Items` 增量 mutation、delta/apply、budgeted upload queue、backend recovery diagnostics 和 unified import service，最后只用一个窄 `Scene Pipeline Lab`、文档和 repository guards 证明这条内部 contract。结果是 `VideraViewRuntime.Scene` 重新回到 orchestration-only 角色，而 scene upload / residency / recovery 不再依赖同步 GPU upload 或 ad hoc engine scans。

## Milestone Summary

**Key decisions:**
- Keep `VideraEngine` as the only public extensibility root and keep `VideraView` as a forwarding shell; all new scene-pipeline work stays internal.
- Treat scene realisation as four separate concerns: document truth, delta planning, residency state, and budgeted upload/apply.
- Move GPU upload into render/device cadence through frame-prelude draining instead of doing it synchronously during scene mutation APIs.
- Delay broader demo work and native mesh preprocess work until the scene pipeline contract is stable and benchmarked.

**Issues deferred:**
- Shared CPU mesh payload / copy-reduction work after residency and upload seams stabilize
- Native mesh preprocess or Rust offload before benchmarks prove importer/mesh prep is the next hotspot
- Broader multi-surface or offscreen productization beyond the runtime/scene pipeline scope
