# Requirements

Active milestone: `v2.6 Render Pipeline Diagnostics Closure`

Render-pipeline-diagnostics requirement numbering starts at `RPD-01`.

- [x] `RPD-01` Before implementation starts, lock `v2.6` to render-pipeline diagnostics closure on the existing viewer/runtime path and freeze explicit non-goals around lighting/shadows/post-processing, animation, extra UI adapters, `Wayland` / `OpenGL` / `WebGL`, backend API expansion, and new material/glTF feature breadth.
- [x] `RPD-02` The existing pipeline snapshot and diagnostics surfaces expose a coherent pass/feature story for `Opaque`, `Transparent`, `Overlay`, `Picking`, and `Screenshot`, including object counts and fallback/status data that can be asserted without backend-specific public APIs.
- [x] `RPD-03` The viewer/runtime evidence path validates transparent, picking, screenshot, and overlay diagnostics through focused tests or smoke evidence so regressions can be caught without introducing a broader render framework.
- [x] `RPD-04` Docs, support wording, sample/demo text, and repository guardrails all describe the render-pipeline diagnostics contract honestly and keep advanced renderer breadth explicitly deferred.

## Out of Scope

| Item | Excluded from `v2.6` |
|------|------------------------|
| Lighting / shadows / environment maps / post-processing | Not part of this milestone. |
| Animation / skeleton / morph / mixer | Not part of this milestone. |
| New material or glTF feature breadth | Not part of this milestone. |
| New UI adapters | Not part of this milestone. |
| `Wayland` / `OpenGL` / `WebGL` / backend API expansion | Not part of this milestone. |
| Generic engine-style render graph or plugin framework | Not part of this milestone. |
| New chart families or `SurfaceCharts` widening | Not part of this milestone. |
