# Videra 开源准备 - v1 需求

**生成日期：** 2026-03-28  
**最近修订：** 2026-04-08  
**来源：** 代码库分析、领域研究、用户确认、已接受的 milestone scope 调整

> 本文件反映 **2026-04-08 已接受的 alpha-ready scope**。  
> 其中：
> - `PLAT-01` 表示 Linux `X11 native + Wayland-session XWayland compatibility` 的真实支持口径，而不是 compositor-native Wayland embedding
> - `MAC-01` 表示 Objective-C runtime / Metal 互操作的集中化与硬化，而不是在本 milestone 内完成更高层 safer binding replacement
> - compositor-native Wayland embedding 与更高层 macOS safer binding replacement 已明确 deferred 到后续里程碑

## v1 需求

### 测试基础设施 (TEST)

| REQ-ID | 需求 | 描述 |
|--------|------|------|
| TEST-01 | 测试框架 | 集成 xUnit 2.9.x 作为测试框架，Moq 4.20.x 用于模拟，FluentAssertions 7.0.x 用于断言 |
| TEST-02 | 核心单元测试 | 为核心抽象和关键行为路径编写单元测试 |
| TEST-03 | 平台集成测试 | 为平台特定后端编写 matching-host 集成测试，验证渲染管线端到端功能 |
| TEST-04 | 覆盖率报告 | 配置 Coverlet 6.0.x 生成代码覆盖率报告 |

### 日志系统 (LOG)

| REQ-ID | 需求 | 描述 |
|--------|------|------|
| LOG-01 | Serilog 集成 | 集成 Serilog 4.2.x 结构化日志，通过 Microsoft.Extensions.Logging 集成到 DI 容器 |
| LOG-02 | 调试代码清理 | 移除所有生产代码中的 Console.WriteLine 调试语句 |
| LOG-03 | 调试计数器移除 | 移除 macOS `MetalCommandExecutor` 中的静态调试计数器 |

### 错误处理 (ERROR)

| REQ-ID | 需求 | 描述 |
|--------|------|------|
| ERROR-01 | 域特定异常 | 定义域特定异常类型：`GraphicsInitializationException`、`ShaderCompilationException`、`ResourceCreationException`、`PipelineStateException` 等 |
| ERROR-02 | 结构化错误信息 | 异常包含相关诊断信息（Windows HRESULT、Vulkan 错误码、Metal 错误描述） |
| ERROR-03 | 用户错误消息 | 提供用户友好的错误消息，用于 UI 显示、Demo 状态和日志记录 |

### 代码质量 (QUALITY)

| REQ-ID | 需求 | 描述 |
|--------|------|------|
| QUAL-01 | 静态分析 | 集成 Microsoft.CodeAnalysis.NetAnalyzers 9.x 和 SonarAnalyzer.CSharp 10.x |
| QUAL-02 | 未实现方法修复 | 移除或实现所有 `NotImplementedException` 方法，或从接口中移除不支持的方法 |
| QUAL-03 | 安全加固 | 为不安全指针操作和原生边界添加边界检查与失败路径保护 |

### 跨平台支持 (PLAT)

| REQ-ID | 需求 | 描述 |
|--------|------|------|
| PLAT-01 | Linux 显示服务器支持 | Linux 原生路径支持 `X11`，Wayland 会话自动解析并 truthfully 回落到 `XWayland` compatibility path；compositor-native Wayland embedding 延后 |
| PLAT-02 | 动态库加载 | 使用 `NativeLibrary` / `dlopen` 风格的动态加载与 fallback resolver，替代硬编码库路径 |
| PLAT-03 | 跨平台构建与验证 | Windows、Linux、macOS 在本地与 GitHub-hosted matching-host native validation 中都能成功编译/验证当前支持范围 |

### macOS 平台范围 (MAC)

| REQ-ID | 需求 | 描述 |
|--------|------|------|
| MAC-01 | ObjC/Metal 互操作收口 | 集中并硬化 Objective-C runtime / Metal 互操作入口；更高层 safer binding replacement 延后 |
| MAC-02 | 深度缓冲一致性 | 统一各平台的深度缓冲管理，确保正确的创建、清理和调整大小行为 |

### 资源管理 (RES)

| REQ-ID | 需求 | 描述 |
|--------|------|------|
| RES-01 | 资源清理验证 | 部分初始化失败时正确回滚，资源释放与重绑定路径可验证 |
| RES-02 | 热路径清理 | 移除渲染循环热路径中的 Console I/O 和不必要的字符串格式化/隐式状态分叉 |
| RES-03 | 线框颜色更新优化 | 优化 `Object3D.UpdateWireframeColor`，避免每次颜色更改时完整拷贝顶点数组 |

### 文档 (DOC)

| REQ-ID | 需求 | 描述 |
|--------|------|------|
| DOC-01 | API 文档 | 为公开 API 添加 XML 注释并支持文档生成 |
| DOC-02 | 用户文档 | 编写 README、示例、安装说明、基本使用教程，并保持 Demo/install truth 与实现一致 |
| DOC-03 | 完整文档 | 编写架构说明、贡献指南、故障排查、发布/验证边界说明，并保持中英文镜像一致 |

### 安全 (SEC)

| REQ-ID | 需求 | 描述 |

| SEC-02 | 库句柄验证 | 验证所有平台库句柄和函数指针，实现缺失库的回退机制 |

| REQ-ID | 需求 | 描述 |

## 反模式（明确不做的事）

- **Compositor-native Wayland embedding in this milestone** — 当前 milestone truthful closure 是 `X11 native + Wayland-session XWayland compatibility`
- **macOS higher-level safer binding replacement in this milestone** — 当前 milestone 只要求 centralized/hardened ObjC runtime / Metal boundary
- **NuGet.org 双发** — 当前只准备并维护 GitHub Packages 发布链路
- **向后兼容性冻结** — 允许为清理和契约修复调整现有 API
- **多线程软件渲染器** — 当前单线程 CPU 渲染器足够
- **LOD 支持** — 不实现细节级别系统
- **GUI 属性编辑器** — Videra 是引擎，不是编辑器
- **资产业务线工具** — 使用现有 Blender / glTF 工具
- **物理模拟** — 超出范围
- **多人游戏** — 超出范围
- **VR/AR 支持** — 需要平台特定 SDK
- **移动平台** — iOS / Android 超出范围
