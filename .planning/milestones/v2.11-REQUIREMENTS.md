# Requirements

Active milestone: `v2.11 Static glTF/PBR Renderer Consumption`

Renderer-consumption requirement numbering starts at `MDC-01`.

## Milestone Goal

On the existing viewer/runtime line, consume already-imported static glTF/PBR metadata in the actual renderer so the shipped scene truth is visible on screen instead of stopping at imported-asset/runtime retention.

## Requirements

- [x] `MDC-01` Renderer-consumption scope lock
  - Freeze `v2.11` to renderer consumption of already-imported static glTF/PBR metadata plus golden-scene evidence.
  - Keep animation, skeletons, morph targets, broader lighting/shadows/post-processing breadth, extra UI adapters, new chart families, broader importer/extension coverage, backend/API expansion, compatibility shims, and migration adapters out of scope.
  - Keep package publishing, repository tag creation, and unrelated performance/package-slimming work out of scope.

- [x] `MDC-02` Texture-transform and UV-set renderer consumption
  - Make the renderer consume existing `KHR_texture_transform` offset/scale/rotation and texture-coordinate override truth on the shipped static-scene path.
  - Keep the implementation on the current material/runtime line instead of introducing a generic material system rewrite.
  - Prove the behavior with focused golden-scene or repository evidence.

- [x] `MDC-03` Occlusion-aware static-scene consumption and golden-scene coverage
  - Make the current renderer consume existing occlusion texture binding/strength truth on the shipped static-scene path.
  - Add bounded golden-scene coverage for the metadata that is now renderer-consumed.
  - Keep broader shading/runtime-feature expansion out of scope.

- [ ] `MDC-04` Renderer-consumption docs and repository guardrails
  - Align docs, demos/support wording, and repository guards with the metadata that is now consumed end-to-end.
  - Make the remaining imported-only metadata and deferred renderer breadth explicit.
  - Leave later normal-map/tangent/advanced PBR breadth to future milestones.

## Out of Scope

| Item | Excluded from `v2.11` |
|---|---|
| Runtime breadth | No new animation, skeleton, morph, general material-system rewrite, or broader runtime-asset model widening. |
| Renderer breadth | No new lights, shadows, environment maps, post-processing, or broad advanced transparency-system work. |
| Import breadth | No new importer format/extension support beyond existing static glTF/PBR metadata already present in runtime truth. |
| Charts/UI/platform | No new chart family, chart kernel widening, extra UI adapter, `WebGL` / `OpenGL` pursuit, or backend API expansion. |
| Compatibility | No compatibility shim, migration adapter, downgrade path, or transitional contract. |
