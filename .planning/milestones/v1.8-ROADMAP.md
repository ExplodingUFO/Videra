# Milestone v1.8: Scene Residency Efficiency and Mesh Payload Optimization

**Status:** COMPLETE 2026-04-17
**Target phases:** 56-59
**Committed workstreams:** 4
**Research:** skipped — this is direct follow-on optimization work and the milestone strategy was supplied in the prompt

## Overview

`v1.8` 不再继续扩 viewer public surface，也不做大而全 demo。它只盯 `v1.7` 已经稳定下来的 scene pipeline，把下一层最值钱的性能和 correctness 问题收掉：dirty transition 不再跟着每帧 cadence 乱跑，upload queue 真正按帧预算工作，CPU mesh payload 不再被 `Object3D` 反复复制持有，同时 recovery / regression tests 把这些优化锁住。

这轮里程碑接受一个明确的 scope 边界：先做 event-driven dirty、heuristic/runtime-mode aware budget、shared payload 和 correctness；不在这一轮碰 public API 扩张、不做 broader demo、也不把 hardware telemetry 或 Rust/native preprocess 硬塞进同一个闭环。

## Phase 56: Event-Driven Scene Residency and Dirty Signaling

**Status:** COMPLETE  
**Goal:** Make residency dirtiness event-driven so render cadence stops globally dirtying upload candidates and upload work only appears when scene truth or resource epochs actually change.  
**Depends on:** v1.7 complete  
**Requirements:** `RES-03`, `RES-04`, `RES-05`  
**Plans:** 3 complete

**Success criteria:**
1. `SceneResidencyRegistry` only dirties entries on scene mutation, resource-epoch change, or explicit rehydrate requests.
2. `RuntimeFramePrelude` no longer performs a global dirty sweep as part of every frame.
3. Entries already resident or already uploading are not repeatedly requeued or reuploaded.

## Phase 57: Shared Mesh Payload and Object Memory Model

**Status:** COMPLETE  
**Goal:** Reduce duplicate CPU-side mesh retention by introducing shared payload semantics and tightening what `Object3D` keeps after deferred materialization.  
**Depends on:** Phase 56  
**Requirements:** `PERF-01`, `PERF-02`, `PERF-03`  
**Plans:** 3 complete

**Success criteria:**
1. Imported/deferred geometry can share a single mesh payload across realized objects without per-object vertex/index duplication.
2. `Object3D` keeps only the CPU mesh state required for reupload, picking, and wireframe according to an explicit retention rule.
3. `SceneObjectFactory` moves to the shared payload model without changing the public import or viewer entrypoints.

## Phase 58: Budgeted Upload Scheduling and Frame Coordination

**Status:** COMPLETE  
**Goal:** Make upload drain honor per-frame object/byte budgets and keep all GPU realization in frame-prelude/device cadence rather than in public scene mutation APIs.  
**Depends on:** Phases 56-57  
**Requirements:** `UPLD-01`, `UPLD-02`, `UPLD-03`  
**Plans:** 3 complete

**Success criteria:**
1. `SceneUploadQueue` drains only within object and byte budgets and stops cleanly when the frame budget is exhausted.
2. Upload budgets vary between interactive and steady-state runtime modes using heuristic/runtime-state inputs rather than a single static budget.
3. Public scene mutation paths stay free of synchronous GPU resource creation.

## Phase 59: Scene Residency, Payload, and Recovery Validation

**Status:** COMPLETE  
**Goal:** Prove the new residency, payload, and upload-budget contract with focused tests so the performance improvements do not regress correctness or recovery behavior.  
**Depends on:** Phases 56-58  
**Requirements:** `TEST-03`, `TEST-04`, `TEST-05`  
**Plans:** 3 complete

**Success criteria:**
1. Tests prove dirty marking and upload transitions avoid duplicate upload of already-resident or in-flight entries.
2. Tests prove shared payload semantics reduce copy count without breaking wireframe, picking, or rehydration.
3. Tests prove backend/device rebuild rehydrates retained scene truth without reimport and still respects frame-upload budgets.

## Milestone Summary

**Key decisions:**
- Keep `VideraEngine` as the only public extensibility root and keep `VideraView` as a forwarding shell; all new work stays inside internal scene/runtime seams.
- Treat this milestone as performance-and-correctness follow-through on `v1.7`, not as a new architecture or demo-expansion push.
- Replace cadence-driven dirtying with event-driven residency transitions before touching more aggressive upload heuristics.
- Tackle shared payload/copy reduction now, but keep hardware telemetry and native preprocess out of the initial scope.

**Issues deferred:**
- Hardware-telemetry-driven budget autotuning beyond runtime/queue-aware heuristics
- Native mesh preprocess or Rust acceleration before benchmarks justify it
- Broader multi-surface or offscreen productization beyond the scene residency/payload closure
