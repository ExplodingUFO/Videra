# Roadmap

## Target Milestone

`v2.28 Streaming Performance Hardening`

Goal: harden the large-dataset performance slice by turning SurfaceCharts columnar scatter into a diagnosable streaming/FIFO path, adding chart-local interaction/refine policy, and strengthening benchmark/docs guardrails without expanding into new chart families or broad renderer architecture.

## Scope Guard

- No broad architecture rewrite, ECS conversion, or general-purpose engine pivot.
- No new chart families before the columnar scatter streaming path is stable.
- No merging SurfaceCharts into `VideraView`.
- No animation, skeleton, morph, shadows, environment maps, post-processing, WebGL/OpenGL backend, or shader/material node system.
- No GPU-driven chart rewrite or indirect rendering path in this milestone.
- No release tags, public package publication, feed mutation, or release workflow execution.
- Keep changes small, chart-local, measurable, and tied to diagnostics or benchmark evidence.
- Keep unstable performance measurements evidence-only until CI history supports hard thresholds.

### Phase 247: Columnar Streaming/FIFO Contract Hardening

**Status:** completed  
**Depends on:** `v2.28 milestone start`  
**Requirement:** `SPH-01`, `SPH-02`

Tighten the SurfaceCharts columnar scatter API around append/FIFO semantics and validation.

Success criteria:
1. Columnar append supports bounded retained data with predictable FIFO trimming.
2. Replace and append paths share clear validation for column lengths, finite coordinates, positive finite sizes, and FIFO capacity.
3. The implementation stays inside SurfaceCharts core data structures and does not introduce a generic streaming framework.
4. Existing point-object scatter series remain behavior-compatible.

### Phase 248: Streaming Diagnostics and Demo Evidence

**Status:** completed  
**Depends on:** Phase 247  
**Requirement:** `SPH-03`, `SPH-04`

Expose streaming/FIFO truth through chart diagnostics and one focused demo/consumer evidence path.

Success criteria:
1. Rendering status or chart-local diagnostics report retained point count, appended batch count, FIFO capacity, and pickable point count.
2. Support summaries include those values without replacing existing SurfaceCharts rendering status.
3. SurfaceCharts demo can exercise streaming append/FIFO behavior with clear diagnostics.
4. Consumer smoke or repository tests prove the packaged path can use the streaming API.

### Phase 249: Interaction Quality Refine Policy

**Status:** completed  
**Depends on:** Phase 248  
**Requirement:** `IQR-01`, `IQR-02`, `IQR-03`

Use the existing SurfaceCharts interaction-quality concept to distinguish interactive work from idle/refine work.

Success criteria:
1. Chart-local policy can mark interaction-time updates separately from settled/refine updates.
2. Interactive mode reduces selected expensive chart work without changing viewer contracts.
3. Idle/refine state restores deterministic full-quality rendering.
4. SurfaceCharts remains an independent control/package family.

### Phase 250: Streaming Benchmark Evidence

**Status:** completed  
**Depends on:** Phase 247  
**Requirement:** `PEG-01`, `PEG-02`, `PEG-03`

Add evidence for streaming append/FIFO cost and allocation behavior while keeping noisy metrics out of hard gates.

Success criteria:
1. Benchmarks capture columnar append, FIFO trim, retained point count, and allocation behavior.
2. Benchmark contract includes the new evidence surfaces.
3. Hard thresholds are unchanged unless a metric is demonstrably stable on CI.
4. Existing threshold documentation reflects current CI noise and threshold intent.

### Phase 251: Documentation and Repository Truth Closure

**Status:** completed  
**Depends on:** Phase 248, Phase 249, Phase 250  
**Requirement:** `PEG-04`

Close docs and repository tests around the new streaming and interaction-quality contracts.

Success criteria:

2. SurfaceCharts docs keep chart/viewer ownership boundaries explicit.
3. Repository tests guard the public docs/API truth for streaming diagnostics and interaction-quality behavior.

## Current Baseline

- SurfaceCharts has `ScatterColumnarData` and `ScatterColumnarSeries` with `ReplaceRange`, `AppendRange`, optional FIFO capacity, and `Pickable=false` by default.
- Scatter diagnostics can report point and pickable counts, but streaming append/FIFO history and interaction/refine state are not first-class diagnostics.
- Benchmarks include existing SurfaceCharts residency/probe/render-host evidence, but not columnar streaming/FIFO evidence.
- Viewer instance-batch work remains evidence-backed and intentionally narrow; GPU instanced rendering is deferred.
