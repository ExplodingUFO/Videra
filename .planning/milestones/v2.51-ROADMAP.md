# Roadmap: v2.51 Professional Chart Output and Dataset Evidence

## Milestone Goal

Make Plot-authored SurfaceCharts easier to export, attach, reproduce, and diagnose as professional 3D chart outputs without widening backend/runtime scope or restoring removed alpha APIs.

## Phase Details

### Phase 356: Chart Output and Dataset Evidence Inventory

**Status:** complete
**Bead:** Videra-779.1
**Goal:** Map the current chart output, report, dataset metadata, demo/support, consumer smoke, Doctor, docs, tests, and guardrail surfaces before changing code.

**Success criteria:**
1. Inventory classifies output/export/report/dataset evidence gaps as implement, document, defer, or reject.
2. Target examples show concise Plot-owned output report and dataset reproducibility usage.
3. Non-goals explicitly reject old chart views, direct `Source`, generic plotting engine scope, backend expansion, hidden fallback/downshift, and god-code.
4. Handoff identifies disjoint implementation phases and write boundaries.

**Output:** `.planning/phases/356-chart-output-and-dataset-evidence-inventory/SUMMARY.md`

### Phase 357: Chart Output Contract

**Status:** complete
**Bead:** Videra-779.2
**Depends on:** Phase 356
**Goal:** Add a bounded chart-local output/report contract for professional chart reports without introducing a renderer/backend rewrite.

**Success criteria:**
1. `VideraChartView.Plot` owns the output/report request surface.
2. Output evidence includes chart kind, series identity, active series, style, color map, precision profile, dimensions, and rendering status.
3. Unsupported output capabilities return explicit diagnostics rather than fallback behavior.
4. Tests cover output contract semantics without exposing backend internals.

**Output:** `.planning/phases/357-chart-output-contract/SUMMARY.md`

### Phase 358: Dataset Metadata and Reproducibility Evidence

**Status:** complete
**Bead:** Videra-779.3
**Depends on:** Phase 356
**Goal:** Expose deterministic dataset identity, shape, range, unit, sampling, and precision evidence for Plot-authored surface, waterfall, and scatter series.

**Success criteria:**
1. Surface, waterfall, and scatter Plot series report deterministic dataset identity and shape metadata.
2. Dataset evidence includes ranges, units, sampling/spacing where available, point or cell counts, and precision policy.
3. Evidence is suitable for support bundles and minimal reproduction workflows without leaking internal runtime state.
4. Evidence updates deterministically when Plot series are added, removed, cleared, or replaced.

**Output:** `.planning/phases/358-dataset-metadata-and-reproducibility-evidence/SUMMARY.md`

### Phase 359: Demo and Consumer Smoke Output Evidence

**Status:** complete
**Bead:** Videra-779.4
**Depends on:** Phases 357 and 358
**Goal:** Refresh SurfaceCharts demo/support evidence so professional chart output and dataset metadata can be attached to support reports.

**Success criteria:**
1. Demo chart data loading uses only `VideraChartView.Plot.Add.*`.
2. Demo and consumer-smoke artifacts include output report status and dataset reproducibility evidence.
3. Doctor or repository support checks parse the evidence without requiring a running UI.
4. Demo remains bounded and does not become a broad chart editor or workbench.

**Output:** `.planning/phases/359-demo-and-consumer-smoke-output-evidence/SUMMARY.md`

### Phase 360: Professional Chart Output Guardrails and Docs

**Status:** complete
**Bead:** Videra-779.5
**Depends on:** Phase 359
**Goal:** Align docs, public roadmap, repository guardrails, and Beads state around the chart-local output/evidence contract.

**Success criteria:**
1. Repository guardrails block old chart controls and direct public `Source` APIs from returning.
2. Guardrails keep chart output work out of backend expansion, generic plotting engine scope, compatibility wrappers, hidden fallback/downshift behavior, and god-code.
3. Package/sample docs show concise Plot-owned output and dataset evidence usage.
4. Beads/public roadmap and local planning state are clean.

**Output:** `.planning/phases/360-professional-chart-output-guardrails-and-docs/SUMMARY.md`
