# Milestone v1.11: Alpha Happy-Path Stabilization and Diagnostics Productization

**Status:** COMPLETE 2026-04-18
**Target phases:** 68-71
**Committed workstreams:** 4
**Research:** skipped — the milestone strategy comes directly from the current repo state, alpha.3 public baseline, and the supplied priority to favor usability/productization over more internal optimization

## Overview

`v1.11` 不再继续沿着大架构改造、功能扩张或新一轮内部性能线往下钻。它把下一轮重点收成更窄但更值钱的 alpha 产品化工作：冻结唯一 public happy path、提供可复制的 diagnostics snapshot / reproduction pack、把反馈与支持面围绕同一 canonical consumer story 收口，并把 benchmark gate 保持成 trend-friendly 的回归守门器。目标不是新增更多底层能力，而是让外部用户更愿意试、试了更容易反馈、反馈后更容易定位。

这轮里程碑有一个明确边界：不做 native Wayland compositor-hosted 嵌入、不把 `Videra.SurfaceCharts.*` 变成 public package、不扩大 viewer public surface，也不重开另一轮 deeper runtime optimization。先把 `0.1.0-alpha.3` 这条公开消费路径压成更稳定、更单一的 alpha 契约。

## Phase 68: Happy-Path API Freeze and Onboarding Unification

**Status:** COMPLETE  
**Goal:** Collapse the default public `VideraView` story into one canonical alpha onboarding path so first-scene setup does not branch across docs, samples, and smoke assets.  
**Depends on:** v1.10 complete  
**Requirements:** `API-07`, `API-08`, `API-09`  
**Plans:** 3 planned

**Success criteria:**
1. Root README, Avalonia README, `Videra.MinimalSample`, and `Videra.ConsumerSmoke` all teach the same `Options -> LoadModel(s) -> FrameAll/ResetCamera -> BackendDiagnostics` flow.
2. Compatibility and advanced entrypoints are preserved only in explicit advanced/compatibility sections instead of the default onboarding narrative.
3. A new alpha consumer can reach first scene without learning `Engine`, frame hooks, pass contributors, or multiple configuration styles.

## Phase 69: Diagnostics Snapshot Export and Reproduction Pack

**Status:** COMPLETE  
**Goal:** Turn diagnostics collection into a copy-pasteable alpha support artifact instead of a manual assembly exercise.  
**Depends on:** Phase 68  
**Requirements:** `DIAG-02`, `DIAG-03`, `DIAG-04`  
**Plans:** 3 planned

**Success criteria:**
1. The default viewer path can export or copy a diagnostics snapshot that includes package/version truth, backend resolution, fallback data, and platform/display-server truth.
2. Troubleshooting docs and issue templates request that same diagnostics snapshot rather than hand-built environment summaries.
3. The diagnostics snapshot works without requiring direct `Engine` extensibility or host-specific custom instrumentation.

## Phase 70: Alpha Feedback Loop and Support Closure

**Status:** COMPLETE  
**Goal:** Make alpha support routing and reproduction expectations explicit so external reports arrive with enough signal to debug quickly.  
**Depends on:** Phases 68-69  
**Requirements:** `FB-02`, `FB-03`, `DOC-06`  
**Plans:** 3 planned

**Success criteria:**
1. Issue/reporting surfaces ask whether the problem reproduces in `Videra.MinimalSample` or `Videra.ConsumerSmoke`.
2. Public docs present supported paths, compatible paths, and deferred paths through one stable alpha support story.
3. Feedback templates, troubleshooting docs, and sample guidance all point to the same canonical reproduction and diagnostics contract.

## Phase 71: Benchmark Stewardship and Alpha Release Stability

**Status:** COMPLETE  
**Goal:** Keep benchmark evidence and release validation aligned with the canonical alpha consumer path without prematurely turning every metric into a hard blocker.  
**Depends on:** Phases 68-70  
**Requirements:** `PERF-09`, `REL-03`, `DOC-07`  
**Plans:** 3 planned

**Success criteria:**
1. Benchmark workflows preserve trend-friendly artifacts and documentation that explain how to compare runs over time.
2. Release/smoke workflows validate the same canonical alpha happy path and package baseline described in public docs.
3. Benchmark, release, support, and onboarding docs all reinforce one alpha consumer path instead of drifting apart.

## Milestone Summary

**Key decisions:**
- Usability/productization stayed ahead of deeper internal optimization because the highest remaining leverage was external adoption friction, not missing runtime primitives.
- The default public consumer path is now singular and typed, with compatibility and advanced extensibility clearly moved behind explicit opt-in sections.
- Diagnostics collection is now a reusable snapshot artifact shared by the minimal sample, consumer smoke, docs, and issue templates.
- Benchmark workflows remain trend-oriented alpha evidence and are now aligned with release and smoke validation instead of living as a separate performance story.
- Native Wayland embedding and broader package expansion remain deferred until real alpha feedback proves they outrank onboarding/support friction.

**Issues deferred:**
- Native compositor-hosted Wayland embedding
- Publishing `Videra.SurfaceCharts.*` as public consumer packages
- Another deep scene/runtime optimization loop before external alpha evidence demands it
- Broader public API or plugin/discovery surface expansion before the canonical happy path stabilizes
