# Requirements

Active milestone: `v2.10 Imported Asset Runtime Tightening`

Imported-asset/runtime-tightening requirement numbering starts at `IAT-01`.

## Milestone Goal

On the existing viewer/runtime line, tighten the transition from imported asset truth to uploaded/rendered runtime objects so the hot path becomes more primitive-centric, more granular, and less dependent on object-level guards.

## Requirements

- [x] `IAT-01` Imported-asset runtime tightening scope lock
  - Freeze `v2.10` to imported-asset/runtime tightening only.
  - Keep animation, lighting/shadows, post-processing, extra UI adapters, new chart families, broader backend/API work, compatibility shims, and migration adapters out of scope.
  - Keep actual public publishing and release-tag creation out of scope.

- [x] `IAT-02` Primitive-level upload/render bridge
  - Allow opaque and transparent primitives from the same imported asset to coexist without rejecting the whole asset as a mixed object-level case.
  - Make upload/render truth finer than a single `Object3D` carrier.
  - Preserve transparent feature truth through the primitive-level path rather than widening into a broader transparency system rewrite.

- [x] `IAT-03` Finer-grained scene delta and upload queue truth
  - Introduce explicit change kinds rather than a single coarse reupload signal.
  - Add queue coalescing and visibility/interaction-aware prioritization to the upload path.
  - Keep backend rebind and normal uploads aligned on the same queue truth.

- [x] `IAT-04` Primitive-centric runtime truth and guardrails
  - Align docs, diagnostics, repository tests, and support wording with the primitive-centric runtime story.
  - Keep mixed opaque/transparent participation, delta/upload vocabulary, and non-goals explicit.
  - Leave later renderer consumption of broader static glTF/PBR metadata to a future milestone.

## Out of Scope

| Item | Excluded from `v2.10` |
|---|---|
| Renderer breadth | No new lighting, shadows, environment maps, post-processing, or animation/mixer work. |
| Import breadth | No broader importer/extension coverage beyond what is needed for primitive-centric runtime tightening. |
| Charts breadth | No new chart family or chart kernel widening. |
| UI/platform breadth | No extra UI adapter, no `WebGL`/`OpenGL` backend pursuit, no backend API expansion. |
| Compatibility | No compatibility shim, migration adapter, fallback runtime path, or transitional contract. |
