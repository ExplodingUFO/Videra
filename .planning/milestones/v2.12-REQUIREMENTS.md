# Requirements

Active milestone: `v2.12 SurfaceCharts Residency and Probe Efficiency`

SurfaceCharts efficiency requirement numbering starts at `SCE-01`.

## Milestone Goal

On the existing `SurfaceCharts` family line, reduce tile residency churn, probe latency, and unnecessary render/overlay invalidation or allocation on the current surface path so large-matrix interaction gets steadier without widening chart/runtime/platform scope.

## Requirements

- [x] `SCE-01` SurfaceCharts efficiency scope lock
  - Freeze `v2.12` to residency churn, probe latency, and invalidation/allocation efficiency work on the shipped `SurfaceChartView` line.
  - Keep new chart families, chart-kernel widening, generic `Chart3D`, viewer/runtime breadth, backend/UI/platform expansion, compatibility shims, and migration adapters out of scope.
  - Keep package publishing, repository tag creation, and unrelated viewer/runtime renderer breadth out of scope.

- [x] `SCE-02` Tile residency prioritization and prefetch tightening
  - Reduce camera-movement residency churn on the existing cache-backed / overview-first surface path.
  - Improve request prioritization and prefetch behavior without replacing the current chart-local architecture.
  - Prove the tighter behavior with focused benchmark or repository evidence.

- [x] `SCE-03` Probe latency and interaction-stability tightening
  - Reduce probe latency and avoid unnecessary interaction-path churn on the shipped `SurfaceChartView` line.
  - Keep the work inside the current chart-local probe/render path instead of widening into a generic analytics kernel.
  - Prove the improvement with focused benchmark or repository evidence.

- [ ] `SCE-04` Invalidation, allocation, and benchmark guardrails
  - Tighten render/overlay invalidation and allocation pressure around the churn/probe hotspots touched by this milestone.
  - Align benchmark/docs/repository guards with the newly tightened SurfaceCharts efficiency story.
  - Leave broader chart-family breadth, package reshaping, and runtime-platform work to future milestones.

## Out of Scope

| Item | Excluded from `v2.12` |
|---|---|
| Chart breadth | No new chart family, no generic `Chart3D`, no chart-kernel widening beyond the existing surface path. |
| Runtime/platform breadth | No new viewer/runtime renderer work, no backend/UI/platform expansion, no `WebGL` / `OpenGL` pursuit. |
| Product shape | No package slimming, installer reshaping, or public package publishing work in this milestone. |
| Compatibility | No compatibility shim, migration adapter, downgrade path, or transitional contract. |
