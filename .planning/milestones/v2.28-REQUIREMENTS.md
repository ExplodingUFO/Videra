# Requirements: v2.28 Streaming Performance Hardening

**Defined:** 2026-04-27  
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v2.28 Requirements

### SurfaceCharts Streaming and FIFO

- [x] **SPH-01**: SurfaceCharts columnar scatter exposes a bounded streaming/FIFO update path that can append batches without unbounded retained memory growth.
- [x] **SPH-02**: Streaming updates preserve clear validation behavior for mismatched columns, non-finite coordinates, invalid sizes, and FIFO capacity.
- [x] **SPH-03**: Rendering status and support summaries report retained point count, appended batch count, FIFO capacity, and pickable point count for columnar scatter paths.
- [x] **SPH-04**: Demo and consumer-smoke evidence can exercise streaming append/FIFO behavior without adding a new chart family.

### Interaction Quality and Refinement

- [x] **IQR-01**: SurfaceCharts interaction-quality policy distinguishes interactive updates from idle/refine updates using existing chart-local ownership boundaries.
- [x] **IQR-02**: Interactive mode can reduce expensive chart work without changing public viewer contracts or merging SurfaceCharts into `VideraView`.
- [x] **IQR-03**: Refine/idle state restores full-quality chart rendering deterministically after interaction settles.

### Performance Evidence and Guardrails

- [x] **PEG-01**: Benchmarks capture streaming append/FIFO cost, retained point count behavior, and allocation evidence for columnar scatter.
- [x] **PEG-02**: No new hard benchmark threshold is promoted unless the metric is stable enough on CI; unstable metrics remain evidence-only.
- [x] **PEG-03**: Existing benchmark thresholds stay aligned with CI reality and documented noise characteristics.
- [x] **PEG-04**: Repository tests guard docs/API truth for streaming, interaction-quality, and diagnostics fields.

## Future Requirements

### Viewer Runtime Breadth

- **VRT-01**: GPU-backed instanced draw path.
- **VRT-02**: Multi-geometry batching.
- **VRT-03**: Per-instance material overrides.
- **VRT-04**: Transparent instance sorting.
- **VRT-05**: GPU-driven culling or indirect draw.

### SurfaceCharts Breadth

- **SCH-01**: Additional chart families.
- **SCH-02**: Generic chart kernel redesign.
- **SCH-03**: Viewer/chart mode unification.

### Renderer Breadth

- **RND-01**: Animation, skeleton, morph, shadows, environment maps, post-processing, WebGL/OpenGL backend, or material node system.

## Out of Scope

| Feature | Reason |
|---|---|
| Broad architecture rewrite | The milestone hardens the performance data path from v2.27, not the whole engine architecture. |
| New chart families | Streaming and interaction-quality policy should prove the current scatter path first. |
| SurfaceCharts merged into `VideraView` | Chart ownership remains package-local and independent. |
| GPU-driven chart rewrite | First close API, diagnostics, and evidence around the existing chart-local path. |
| Release publication | This milestone is product/performance hardening, not a release cutover. |
