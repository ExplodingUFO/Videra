# Requirements: v2.29 CI/PR Reality for Streaming Performance

## Milestone Goal

Take the completed v2.28 streaming-performance branch through remote PR, GitHub CI, review, merge, and cleanup reality while preserving the narrow performance-hardening scope.

## Active Requirements

### PR and Branch Truth

- [ ] **CPR-01**: Maintainers can inspect a remote PR branch whose diff contains the intended v2.28 streaming-performance commits and no unrelated work.
- [ ] **CPR-02**: Maintainers can review a PR description that states scope, non-goals, verification already performed, and expected CI risk areas.

### CI Reality and Failure Closure

- [ ] **CPR-03**: Maintainers can see the status of all required GitHub CI checks for the PR, including failure URLs and classifications when failures occur.
- [ ] **CPR-04**: Maintainers can accept focused fixes for CI failures that are either code defects or incorrect CI/test contracts, without widening the milestone into new product features.

### Merge and Cleanup

- [ ] **CPR-05**: Maintainers can merge the PR after required checks pass and leave local `master`, the remote branch, and the local worktree in a clean state.
- [ ] **CPR-06**: Maintainers can read final planning evidence that records merge result, residual risk, and the next recommended milestone direction.

## Future Requirements

- Promote streaming/FIFO benchmark metrics from evidence-only to hard thresholds after enough CI history exists.
- Add further SurfaceCharts streaming quality policies only after v2.28 behavior is merged and stable on `master`.

## Out of Scope

- New chart families or chart API breadth.
- Renderer rewrite, GPU-driven chart rewrite, or new backend architecture.
- Public package publication, release tags, or feed mutation.
- Compatibility layers, migration adapters, or fallback paths not required by CI failure closure.
- New demo features unrelated to proving or diagnosing the v2.28 branch in PR/CI.
