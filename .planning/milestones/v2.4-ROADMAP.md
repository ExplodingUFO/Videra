# Roadmap

## Active Milestone

### `v2.4 Runtime Truth Unification`

- Status: phase-complete locally; PR/merge and milestone audit/closeout next
- Scope: unify `import -> scene truth -> upload -> render -> backend rebind` around one runtime asset model without widening into animation, lighting/shadow breadth, extra UI adapters, or backend/API expansion

## Operational Baseline

- `v2.3` is now archived locally as the shipped surface-analytics-depth baseline.
- `SceneDocument`, `ImportedSceneAsset`, `SceneNode`, `MeshPrimitive`, `MaterialInstance`, `Texture2D`, and `Sampler` already exist as the intended runtime asset model, but the public/runtime path still carries `Object3D`-centric truth and upload bridging seams.
- Static glTF/PBR baseline exists, but broader glTF breadth, animation, lighting/shadows, extra UI adapters, and `Wayland` / `OpenGL` / `WebGL` work remain out of scope.
- `v2.4` focuses on unifying runtime truth first; broader runtime breadth should only follow after this boundary is stable.

## Phases

### Phase 148: Runtime Truth Scope Lock

- Goal: lock runtime-truth unification before new scene/runtime edits start.
- Depends on: `v2.3` closeout
- Status: `complete`

Plans:

- [x] lock `SceneDocument` / runtime-asset truth unification into scope before new runtime edits
- [x] freeze non-goals around animation, lighting/shadows/post-processing, extra UI adapters, and `WebGL` / `OpenGL` / backend API breadth
- [x] keep the milestone inside the existing viewer/runtime `1.0` boundary

### Phase 149: SceneDocument and Imported-Asset Truth

- Goal: make `SceneDocument` and related runtime state preserve authoritative imported-asset identity and runtime asset truth instead of treating `Object3D` as the long-term public scene boundary.
- Depends on: Phase 148
- Status: `complete`

Plans:

- [x] converge scene truth on `SceneDocument` plus runtime asset identity rather than ad-hoc `Object3D` ownership
- [x] keep retained imported assets and scene-entry identity stable across scene updates
- [x] avoid widening into new runtime feature families while reshaping truth

### Phase 150: Unified Upload, Render, and Backend Rebind Path

- Goal: route upload, render, and backend-rebind paths through the unified runtime truth while demoting `Object3D` payload bridging to internal render/cache concerns only.
- Depends on: Phase 149
- Status: `complete`

Plans:

- [x] remove duplicated or lossy scene semantics between import, upload, and render paths
- [x] keep backend rebind and retained-asset reuse coherent on the unified runtime model
- [x] avoid a broad rendering-feature expansion while changing the path

### Phase 151: Runtime Truth Diagnostics and Guardrails

- Goal: align docs, diagnostics, samples, and repository guardrails with the unified runtime truth and the current static-scene boundary.
- Depends on: Phase 149 and Phase 150
- Status: `complete`

Plans:

- [x] align product wording so viewer/runtime docs teach the same runtime-truth story
- [x] update repository/CI guardrails around the unified runtime model and current static-scene boundary
- [x] keep sample/support truth honest without widening package or platform claims
