### Summary

1. `VideraView` now forwards through `VideraViewRuntime`, but `BackendDiagnostics`, `RenderCapabilities`, selection/annotation state, and scene APIs still project the same public truth.
2. Render scheduling is now invalidation-driven and interactive-lease-aware, while diagnostics, fallback, and software presentation behavior remain aligned with the runtime shell.
3. Scene ownership now flows through backend-neutral assets and runtime-owned `SceneDocument` truth instead of split `Items` / `Engine` ownership.
4. Interaction and overlay semantics now sit in Core services, leaving Avalonia runtime/bridge code narrower and easier to reason about.
5. Internal graphics abstraction and `VideraEngine` decomposition now align code, docs, and repository guards around the same shell/runtime/bridge/engine layering story.
