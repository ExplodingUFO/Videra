# Requirements: Videra v1.25 SurfaceCharts Productization and Minimal Chart Kernel

**Defined:** 2026-04-21
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v1 Requirements

Requirements committed for the `v1.25` milestone. Each requirement maps to one roadmap phase.

### SurfaceCharts Product Boundary

- [x] **CHRT-01**: `Videra.SurfaceCharts` has one explicit shipped package matrix, support level, and install story on the Avalonia path.

### Packaged First-Chart Flow

- [x] **CHRT-02**: A packaged first-chart consumer flow validates install -> render -> probe/support artifact on the supported host path.

### Release and Repository Truth

- [ ] **CHRT-03**: Docs, release flow, package validation, and repository guards enforce the same SurfaceCharts product truth.

### Minimal Chart Kernel

- [x] **CHRT-04**: A minimal chart-kernel seam isolates reusable chart contracts/runtime pieces from `SurfaceChartView` and surface-only rendering without opening generic `Chart3D` or second-series scope.

## Future Requirements

Deferred after `v1.25`. These remain visible but are not part of the current roadmap.

### Additional Chart Breadth

- **CHRT-05**: Additional chart families or broader `Chart3D` extraction only follow after the minimal kernel proves reusable on a second concrete chart product.

### UI Validation

- **UIAD-01**: A second UI adapter proves the host seams outside Avalonia after the chart and viewer product lines are both stable.

### Advanced Runtime Features

- **ADV-01**: Lighting expansion, shadows, environment maps, transparent-sorting overhauls, post-processing, and animation remain an opt-in `2.0` line rather than a `1.x` runtime-core requirement.

## Out of Scope

Explicitly excluded from `v1.25` to keep the milestone focused on one real public chart product line.

| Feature | Reason |
|---------|--------|
| Renaming the shipped chart package IDs to `Videra.Charts.Surface` | The current repo truth and project files already center on `Videra.SurfaceCharts.*`; rename churn does not outrank productization |
| Generic `Chart3D` extraction or second chart families | `v1.25` only extracts the minimum kernel seam needed to avoid future surface-only coupling |
| Viewer/runtime backend expansion such as `OpenGL` or `WebGL` | Backend expansion still ranks below productizing the existing chart line |
| A second UI adapter | This milestone keeps the chart product line on the existing Avalonia path |
| Lighting-system expansion, post-processing, or animation | Advanced runtime breadth remains a later product milestone |
