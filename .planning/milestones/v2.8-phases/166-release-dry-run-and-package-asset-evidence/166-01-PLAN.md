# Phase 166 Plan: Release Dry Run and Package Asset Evidence

## Goal

Validate tag/version/package asset expectations without publishing to public feeds.

## Tasks

1. Add a release dry-run script that packs every public package from the public API contract manifest.
2. Reuse `Validate-Packages.ps1` so dry-run evidence checks package set, versions, symbols, readmes, icon/license/repository metadata, dependency boundaries, and size budgets.
3. Add a read-only GitHub workflow that runs the dry-run script and uploads evidence artifacts.
4. Add repository tests that prevent the dry-run path from gaining feed-push behavior or drifting from the canonical package contract.
5. Verify with focused tests, local dry-run script execution, and whitespace checks.

## Acceptance Criteria

- Release dry-run has a first-class script and workflow.
- The workflow has no package-feed write permissions, feed token usage, `dotnet nuget push`, or GitHub release creation.
- The dry-run script writes machine-readable and human-readable evidence artifacts.
- The dry-run package set is driven by `eng/public-api-contract.json`.
- Public and preview publish workflows continue to call `Validate-Packages.ps1` before their existing push steps.
