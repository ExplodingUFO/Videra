# Phase 165 Plan: Public API Contract Guardrails

## Goal

Make public API drift explicit and reviewable for the shipped release-candidate package surface.

## Tasks

1. Add a source-controlled public API contract manifest.
2. Add a repository test that validates manifest integrity against package projects and source roots.
3. Add a repository test that scans top-level public type declarations and compares them exactly with the manifest.
4. Run focused repository tests and whitespace validation.

## Acceptance Criteria

- `eng/public-api-contract.json` exists and lists every public package in the release-candidate public package line.
- Every manifest package points to an existing `.csproj` and source root.
- The test fails on accidental addition or removal of a top-level public type in any public package.
- Platform packages are explicitly guarded as exposing zero top-level public types.
- No runtime, rendering, chart, backend, compatibility, or publish behavior changes are introduced.
