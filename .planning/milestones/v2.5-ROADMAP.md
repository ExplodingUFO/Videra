# Roadmap

## Active Milestone

### `v2.5 Static glTF/PBR Breadth`

- Status: phases `152-155` complete locally; `v2.5` audit `passed`
- Scope: extend the unified runtime-truth path with one bounded static-scene glTF/PBR breadth slice without widening into animation, lighting/shadow breadth, extra UI adapters, or backend/API expansion

## Operational Baseline

- `v2.4` is now archived locally as the shipped runtime-truth-unification baseline.
- The viewer/runtime path now teaches one direct `SceneDocument` / imported-asset / upload / render / backend-rebind story.
- Static glTF/PBR remains the shipped baseline; `v2.5` is the first follow-on milestone that broadens that baseline after runtime truth was unified.
- `v2.5` stays inside static-scene material/runtime breadth; animation, lighting/shadows, extra UI adapters, and `Wayland` / `OpenGL` / `WebGL` work remain out of scope.

## Phases

### Phase 152: Static glTF/PBR Breadth Scope Lock

- Goal: lock `v2.5` to one bounded static-scene glTF/PBR breadth slice before code changes start.
- Depends on: `v2.4` closeout
- Status: `complete`

Plans:

- [x] lock `v2.5` to static-scene glTF/PBR breadth on the unified runtime-truth path
- [x] freeze non-goals around animation / skeleton / morph / mixer, lighting/shadows/post-processing, extra UI adapters, and `Wayland` / `OpenGL` / `WebGL` / backend API expansion
- [x] choose one concrete next implementation slice instead of reopening broad runtime breadth

### Phase 153: Per-Primitive Material Participation and Mixed-Alpha Runtime Preservation

- Goal: preserve per-primitive material participation and mixed-alpha semantics on the unified runtime path instead of collapsing imported assets to one object-wide alpha truth.
- Depends on: Phase 152
- Status: `complete`

Plans:

- [x] preserve primitive/material segmentation for non-`Blend` imported mixes instead of forcing one object-wide alpha truth
- [x] keep mixed `Blend` / non-`Blend` assets behind an explicit runtime guardrail until transparent primitives can be sorted independently
- [x] validate runtime-owned scene/service seams and clipped-payload tangent preservation against the new participation model

### Phase 154: Occlusion and Texture-Transform Material Contracts

- Depends on: Phase 153
- Status: `complete`

Plans:

- [x] preserve occlusion and transformed texture-binding truth through `ImportedSceneAsset` material catalogs

### Phase 155: Static Runtime Breadth Truth and Guardrails
