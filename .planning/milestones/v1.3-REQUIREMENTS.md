# Requirements: Videra v1.3 Camera-True Surface Charts

**Defined:** 2026-04-16
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v1.3 Requirements

### Camera & Projection

- [x] **CAM-01**: Host 和 chart runtime 通过单一的 `SurfaceViewState` / `SurfaceCameraFrame` 主线驱动渲染、overlay 和交互，而不是继续维护 public view-state 与 internal viewport/projection 双系统。
- [x] **CAM-02**: Renderer、overlay 和后续 picking/LOD 共用同一套 project/unproject 数学，并在默认视角下继续保留当前可见效果与 sibling boundary。

### Picking & Probe Truth

- [x] **PICK-01**: End user 的 hover、pinned probe 和 focus anchor 基于 screen-ray -> heightfield intersection 命中真实曲面，而不是基于屏幕到 viewport 的线性 sample 映射。
- [x] **PICK-02**: Pinned probe 和 zoom/focus anchor 在 orbit / refine / tile-upgrade 之后保持稳定，并能从 approximate truth 升级到 exact truth。

### Camera-Aware LOD

- [x] **LOD-01**: Tile request planning 根据 projected footprint / screen error 选择层级与优先级，而不是只根据 viewport density 做静态推断。
- [x] **LOD-02**: `Interactive` / `Refine` 质量路径在相机移动时保持响应，并在停稳后补回正确 detail，而不会因为小幅 orbit 造成无意义 tile churn。

### GPU Resident Path

- [x] **GPU-01**: GPU 主渲染路径不再构建不必要的 software-scene 中间产物，并使用更瘦的 resident 表示而不是长期持有完整 CPU 展开几何/颜色数据。
- [x] **GPU-02**: Patch topology、tile resource upload 和 GPU residency 更新具备更低分配与更低重复构建成本，并保持 software fallback truth 不回归。

### Shader / Backend Color Mapping

- [x] **SHDR-01**: 切换 color map 时，GPU 主路径不再按 resident tile 数量线性重建颜色数组和资源。
- [x] **SHDR-02**: Surface、legend 和 value-range contract 在 backend-side color mapping 下继续保持一致 truth，并且不会把新需求反推回 `VideraView` 公共抽象。

### Professional Overlays

- [x] **OVR-01**: Axis / grid / legend overlay 在常见相机视角下具备稳定的可见边选择、tick 密度变化和 label collision handling。
- [x] **OVR-02**: Host 可以配置 formatter、单位、标题、minor ticks、grid plane 与 axis-side 行为，而不需要把 chart-specific 逻辑泄漏进 `VideraView`。

## v1.4+ / Future Requirements

### Processing / Native Follow-through

- **PROC-01**: Pyramid reduction、payload decode、batch read 等 processing hotspots 只有在 benchmark 证明值得时才引入进一步的 pooling / native acceleration。
- **PROC-02**: 任何后续 Rust/native adoption 都限制在 `Videra.SurfaceCharts.Processing` / coarse data seams，不进入 Avalonia interaction、chart runtime orchestration 或 `VideraView` 公共边界。

### Advanced Analysis

- **ANLY-01**: End user can overlay contours, slices, or derived analysis layers on top of the surface after the camera-true rendering spine stabilizes.
- **ANLY-02**: End user can compare multiple linked views or synchronized markers across related datasets after the base chart interaction and overlay contracts mature.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Folding surface-chart semantics back into `VideraView` | Violates the sibling-boundary contract already locked by docs, demo, and repository tests |
| Viewer-side PBR / shadow / shader-graph expansion | `v1.3` 聚焦 chart-local camera/projection/picking/LOD/GPU spine，而不是扩张通用 viewer feature 面 |
| Broad Processing/Rust rewrite before post-PR5 benchmarks | 当前 optional PR7 明确是 evidence-gated deferred scope，不属于 committed milestone spine |
| Streaming sources, multi-view analysis layers, or contour/slice overlays | 这些能力要等 camera-true contract 与 professional overlay baseline 稳定后再扩展 |
| compositor-native Wayland embedding | Separate platform milestone, not part of the chart professionalization roadmap |
