# Requirements

Active milestone: `v2.9 Release Candidate Final Validation`

Release-candidate final-validation requirement numbering starts at `RCV-01`.

## Milestone Goal

In the current public package/release-candidate boundary, make the candidate validation process auditable, credential-free, abortable, and ready for a later human-controlled release cutover without creating tags or publishing packages in this milestone.

## Requirements

- [x] `RCV-01` Final-validation scope lock
  - Freeze `v2.9` to release-candidate validation only.
  - Keep runtime/rendering/material/chart/backend/UI-adapter breadth out of scope.
  - Keep compatibility shims, fallback behavior, migration adapters, actual public publishing, and tag creation out of scope.

- [x] `RCV-02` Candidate version/tag simulation guardrails
  - Validate tag-derived version semantics against package metadata and release dry-run output without creating a repository tag.
  - Keep the validation credential-free and deterministic in local and CI contexts.
  - Fail clearly when candidate version metadata, package identity, or dry-run release assumptions drift.

- [x] `RCV-03` Release evidence packet and review index
  - Provide a single candidate evidence index for CI checks, release-dry-run artifacts, package validation, benchmark gates, native validations, consumer smoke, and support artifacts.
  - Make the packet reviewable without requiring maintainers to inspect disconnected workflow artifacts manually.
  - Keep the packet read-only and non-publishing.

- [x] `RCV-04` Release abort and human cutover truth
  - Document and guard the abort criteria for failed candidate validation.
  - Make manual cutover boundaries explicit, including "do not publish until human approval".
  - Align repository docs/tests with the final-validation and abort/cutover story.

## Out of Scope

| Item | Excluded from `v2.9` |
|---|---|
| Runtime breadth | No new scene/runtime model, renderer pass, material, lighting, animation, or backend capability. |
| Chart breadth | No new chart family, chart kernel widening, or chart rendering feature. |
| UI adapters | No Avalonia API widening and no second UI adapter work. |
| Compatibility | No compatibility shim, migration adapter, fallback release path, or transitional contract. |
| Publishing | No public package publishing, GitHub release creation, or repository release tag creation. |
