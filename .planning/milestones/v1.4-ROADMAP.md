# Milestone v1.4: Open Source Consumption and Release Surfaces

**Status:** COMPLETE 2026-04-16
**Target phases:** 29-34
**Committed workstreams:** 6
**Research:** skipped — milestone strategy was supplied directly in the prompt

## Overview

`v1.4` 没有重做目录树，而是把公开消费面收紧到几个真正会被陌生人碰到的入口：README、package matrix、support matrix、公开 feed、NuGet metadata、issue/discussion/security routing、release page，以及 dependency maintenance automation。

这个里程碑的核心不是新增渲染能力，而是让仓库对外讲同一套话：哪些包是真正的公开包，哪些模块仍是 source-only，默认走哪个 feed，维护者怎么发布、怎么收 issue、怎么保持依赖更新和 release surface 不漂移。

## Milestone Summary

**Key decisions:**
- 不大搬目录，继续沿用当前仓库骨架，把主要力气放在对外入口、发布边界和协作入口。
- `nuget.org` 是公开 stable install path，GitHub Packages 只保留 preview/internal truth。
- `Videra.SurfaceCharts.*` 明确保留为 source-only modules，而不是在没有发布承诺的情况下被 README 暗示成公开包。
- package metadata、release pages、issue/discussion/security routing 和 dependency automation 都属于开源可消费性的同一条真相链路。

**Issues resolved:**
- README 现在明确区分公开可安装包、source-only modules 和 demos/samples。
- 公开 feed truth 已经收敛到 `nuget.org` public + GitHub Packages preview/internal。
- 公开包已补齐 icon、SourceLink、`.snupkg` 和 package validation。
- GitHub collaboration surfaces、release surfaces 和 dependency maintenance 入口现在对齐成一套公开流程。

**Issues deferred:**
- 把 `Videra.SurfaceCharts.*` 直接提升为公开发布包
- 更大范围的网站/社区/门户建设
- 任何与当前开源消费面无关的 renderer/runtime/chart feature expansion
