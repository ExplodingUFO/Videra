# Requirements: Videra v1.14 Compatibility and Quality Hardening

**Defined:** 2026-04-20
**Archived:** 2026-04-20
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v1.14 Requirements

### Linux Compatibility Truth

- [x] **HOST-01**: Linux Avalonia hosts surface resolved display-server and fallback truth consistently across runtime diagnostics, diagnostics snapshots, demos, and support artifacts.
- [x] **HOST-02**: Linux troubleshooting, architecture, and support wording explain `X11` / `XWayland` host limits truthfully and do not imply that adding `OpenGL` solves compositor-native Wayland constraints.
- [x] **HOST-03**: Linux validation evidence clearly distinguishes supported `X11` and Wayland-session `XWayland` paths from unsupported compositor-native Wayland hosting.

### Consumer and Sample Validation

- [x] **CONS-01**: packaged consumer smoke runs as routine pull-request evidence instead of only manual or release-time validation.
- [x] **CONS-02**: the documented advanced public references (`Videra.ExtensibilitySample` and `Videra.InteractionSample`) gain executable validation for their runtime contracts.
- [x] **CONS-03**: diagnostics and support artifacts emitted by smoke and sample validation stay aligned with alpha issue-reporting expectations.

### Backend Contract Consistency

- [x] **BACK-01**: built-in backends expose one documented common minimum contract for pipeline creation, buffer binding, depth semantics, and failure behavior.
- [x] **BACK-02**: unsupported operations in built-in backends are either normalized or removed from the common abstraction so tests and consumers do not depend on backend-specific quirks.
- [x] **BACK-03**: backend selection labels, diagnostics, and public docs keep telling the truth about supported native paths without introducing an `OpenGL` product promise.

### Quality Gates and Evidence

- [x] **QUAL-01**: analyzer warning debt is triaged and high-value touched surfaces run under stricter warning expectations.
- [x] **QUAL-02**: benchmark and consumer-path evidence become actionable merge-time signals instead of purely observational artifacts.
- [x] **QUAL-03**: CI, docs, and planning artifacts align on one alpha-ready definition of “green” for compatibility and regression protection.

## Requirement Outcomes

### Validated

- `HOST-01` / `HOST-02` / `HOST-03` validated through diagnostics formatter updates, consumer-smoke artifacts, repository guards, and Linux support docs.
- `CONS-01` / `CONS-02` / `CONS-03` validated through routine packaged consumer smoke, explicit sample-contract evidence, and aligned release/readme wording.
- `BACK-01` / `BACK-02` / `BACK-03` validated through abstraction docs, backend smoke coverage, and support-matrix/docs truth.
- `QUAL-01` / `QUAL-02` / `QUAL-03` validated through curated warnings-as-errors builds, packaged-consumer quality-gate evidence, and aligned CI/doc language.

### Adjusted

- `BACK-03` is satisfied, but the audit recorded one accepted tech-debt item: the repository contract test still checks `OpenGL` token presence rather than full “no product promise” semantics.

### Deferred Future Requirements

- **GL-01**: evaluate an explicit `OpenGL` backend only after host-compatibility hardening and backend contract normalization prove it would unlock real target environments.
- **WAYLAND-05**: native compositor-hosted Wayland embedding remains deferred until the UI stack offers a maintainable native-host path or consumer evidence raises its priority.
- **EPS-01**: explicit multi-backend epsilon harnessing for inspection-fidelity comparisons remains deferred until compatibility/quality hardening stops dominating alpha risk.
- **PERF-16**: deeper clip-preview or snapshot-export performance work beyond the current same-API fast paths remains deferred until benchmark evidence identifies a dominant bottleneck.
- **PKG-05**: publishing `Videra.SurfaceCharts.*` as public consumer packages remains deferred unless strategy deliberately shifts from viewer depth to product-width expansion.
- **EXT-05**: plugin/package discovery remains deferred while `VideraEngine` stays the sole public extensibility root.
