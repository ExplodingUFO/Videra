Create `SurfaceChartInteractionPropagator.cs` — host-owned propagator (per D-02, D-03, D-04):

The class takes a `SurfaceChartLinkGroup` and three boolean flags in its constructor:
- `propagateSelection` (default false)
- `propagateProbe` (default false)
- `propagateMeasurement` (default false)

Implementation details:

1. **Constructor**: Store link group and flags. Iterate `_linkGroup.Members` and subscribe to:
   - `SelectionReported` event (if `propagateSelection` or `propagateMeasurement` is true)
   - For probe propagation, there is no dedicated "ProbeReported" event. Probe is hover-based.
     The propagator exposes a `PropagateProbe(VideraChartView sourceChart, Point screenPosition)`
     method that the host calls when it wants to propagate a probe from one chart to linked charts.

2. **Selection propagation** (`OnSelectionReported` handler):
   - Guard: `if (_isPropagating || !_propagateSelection) return;`
   - Set `_isPropagating = true`
   - For each member that is NOT the sender:
     - Call `member.TryCreateSelectionReport(e.Report.ScreenStart, e.Report.ScreenEnd, out _)`
     - The TryCreateSelectionReport internally maps screen positions to sample-space, so we pass the source chart's screen coordinates. The linked chart will resolve them against its own data. Per the research, if charts share the same data the selection will appear at the correct position.
   - Finally: `_isPropagating = false`

3. **Probe propagation** (`PropagateProbe` public method):
   - Guard: `if (_isPropagating || !_propagateProbe) return false;`
   - The host calls this with the source chart and screen position
   - Set `_isPropagating = true`
   - For each member that is NOT the source chart:
     - Call `member.TryResolveProbe(screenPosition, out var probe)` — this uses the source chart's screen position, which may not map correctly to a different chart with different view size.
     - **Better approach**: Since probe coordinates are sample-space, the host should resolve the probe on the source chart first, then the propagator calls a method to highlight the same sample point on linked charts. However, `TryResolveProbe` takes screen coordinates.
     - **Practical approach**: The propagator calls `TryResolveProbe` with the same screen position on each linked chart. Each chart resolves against its own projection. If charts share data and have similar view sizes, the probe will be at a similar position. Document this as a known limitation (different view sizes produce different screen-to-sample mapping).
   - Finally: `_isPropagating = false`

4. **Measurement propagation** (within `OnSelectionReported` when `_propagateMeasurement` is true):
   - Same guard pattern
   - For each member that is NOT the sender:
     - Call `member.TryCreateSelectionMeasurementReport(e.Report.ScreenStart, e.Report.ScreenEnd, out _)`

5. **`CaptureInteractionState()` method** — returns a `SurfaceChartLinkedInteractionState` record for this propagator's link group configuration.

6. **Dispose**: Unsubscribe from all events. Set `_disposed = true`.

7. **Constructor validation**: Throw `ArgumentException` if link group has no members.
