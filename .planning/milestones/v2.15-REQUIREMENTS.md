# Requirements

Active milestone: `v2.15 Opt-In Direct Lighting Baseline`

Direct-lighting requirement numbering starts at `LGT-01`.

## Milestone Goal

Start the deferred advanced-runtime line with one bounded opt-in direct-lighting baseline on the existing static-scene viewer/runtime path, without widening into shadows, environment maps, post-processing, animation, new UI/package lines, or broader runtime/package expansion.

## Requirements

- [x] `LGT-01` Opt-in direct-lighting scope lock
  - Freeze `v2.15` to one bounded direct-lighting slice on the existing viewer/runtime line.
  - Keep shadows, environment maps, post-processing, animation, new UI/package lines, backend/platform expansion, broader importer/chart widening, compatibility shims, and migration adapters out of scope.
  - Keep public package publishing and repository tag creation out of scope.

- [x] `LGT-02` Direct-lighting material and render contract
  - Add only the minimum material/render seam needed for one honest direct-lighting baseline on the shipped static-scene path.
  - Avoid generic lighting frameworks, broad shader-system rewrites, or speculative advanced-runtime abstractions.
  - Prove the new contract with focused code evidence.

- [x] `LGT-03` Repo-owned lighting proof and diagnostics evidence
  - Strengthen one repository-owned viewer proof path so it demonstrates the direct-lighting baseline on a supported host path.
  - Keep diagnostics/support artifacts coherent with the new lighting baseline.
  - Keep the proof repository-owned and non-public.

- [x] `LGT-04` Direct-lighting truth and guardrails
  - Align docs, support wording, and repository guardrails with the shipped direct-lighting baseline.
  - Keep shadows, post-processing, animation, environment maps, and broader advanced-runtime breadth explicitly deferred.
  - Leave package-split or optional-pack publishing decisions to later milestones.

## Out of Scope

| Item | Excluded from `v2.15` |
|---|---|
| Advanced renderer breadth | No shadows, environment maps, post-processing, animation, skeletons, morph targets, mixers, or broader lighting system work. |
| Public package expansion | No new optional runtime package line or release-path changes. |
| UI/platform scope | No second public UI adapter/package line and no backend/platform expansion. |
| Product breadth | No broader importer-format, chart-family, or generic engine/runtime-platform widening. |
| Compatibility | No compatibility shim, migration adapter, downgrade path, or transitional dual-renderer contract. |
