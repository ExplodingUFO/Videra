# Requirements: v2.38 SurfaceCharts Support Experience Closure

**Defined:** 2026-04-28  
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v2.38 Requirements

Requirements for this milestone. Each maps to one roadmap phase.

### Scope and Inventory

- [x] **INV-01**: Maintainers can see the current SurfaceCharts demo, Performance Lab, support bundle, Doctor, and integration-test lifecycle gaps before implementation begins.
- [x] **INV-02**: The milestone records which items are supportability defects versus broad feature requests, with explicit deferrals for new chart families, renderer rewrites, and `VideraView` integration.

### SurfaceCharts Support UX

- [x] **SUX-01**: SurfaceCharts demo users can copy or export a concise support report that includes active chart type, rendering status, fallback state, interaction quality, visible/resident tile or point data, streaming/FIFO counters, and runtime/backend context.
- [x] **SUX-02**: The support report uses existing chart/runtime diagnostics instead of duplicating a parallel diagnostics model.
- [x] **SUX-03**: English and Chinese docs explain how to attach the SurfaceCharts support report and what it does not prove.

### Lifecycle Reliability

- [x] **LIFE-01**: SurfaceCharts demo and integration-test host paths have a deterministic lifecycle guard that prevents long-running or orphaned test-host behavior in automated verification.
- [x] **LIFE-02**: Lifecycle verification records timeout, shutdown, and failure evidence without adding broad CI suppressions or blanket test skips.
- [x] **LIFE-03**: The guardrail remains scoped to demo/test-host reliability and does not change chart rendering semantics.

### Evidence Alignment

- [x] **EVID-01**: Performance Lab and SurfaceCharts support evidence share consistent scenario, settings, rendering status, interaction-quality, streaming/FIFO, and visual-evidence reference vocabulary.
- [x] **EVID-02**: Doctor or support-evidence discovery can surface the new SurfaceCharts support report when present, while remaining passive and repo-local.
- [x] **EVID-03**: Repository tests prevent support evidence from being documented as a benchmark guarantee, visual-regression gate, or GPU-performance claim.

### Closeout

- [x] **CLOSE-01**: Beads issue state, `.beads/issues.jsonl`, Docker Dolt status, branch/worktree state, and planning traceability are clean after the milestone phases complete.

## Future Requirements

Deferred to future milestones.

### Product Expansion

- **FUT-01**: New SurfaceCharts chart families can be considered after support evidence and lifecycle reliability are stable.
- **FUT-02**: Stable benchmark threshold promotion for SurfaceCharts streaming or Performance Lab metrics should wait for repeatable CI data.
- **FUT-03**: Broader renderer/backend capability expansion remains deferred until a dedicated architecture milestone has concrete evidence.

### Architecture

- **ARCHF-01**: SurfaceCharts remains an independent package/control family and should not become a `VideraView` mode.
- **ARCHF-02**: Visual evidence remains evidence-only unless a future milestone deliberately designs a visual-regression contract.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Broad renderer architecture rewrite | Current goal is supportability and reliability closure, not renderer expansion. |
| New chart families | Existing SurfaceCharts support paths need stronger diagnostics before adding type breadth. |
| SurfaceCharts inside `VideraView` | Project architecture keeps chart semantics independent from viewer contracts. |
| Animation, shadows, post-processing, WebGL/OpenGL, or environment maps | These remain outside the desktop viewer/supportability milestone. |
| Blanket CI skips or timeout suppressions | Lifecycle issues should be bounded and evidenced, not hidden. |
| Public release, package publication, release tag, or feed mutation | This milestone is product/support quality work, not release shipping. |
