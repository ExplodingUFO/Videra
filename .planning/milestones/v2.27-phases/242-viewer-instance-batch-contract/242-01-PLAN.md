# Phase 242 Plan: Viewer Instance Batch Contract

## Goal

Expose a small, validated public instance-batch contract without broad scene architecture changes.

## Success Criteria

1. Public API exposes an instance-batch descriptor/entry shape without ECS, plugin discovery, or multi-geometry batching.
2. First version supports one `MeshPrimitive`, one `MaterialInstance`, per-instance `Matrix4x4`, optional color, optional object id, and `Pickable`.
3. Transparent sorting, per-instance material, indirect draw, GPU culling, and multi-geometry batching remain explicitly out of scope.
4. Invalid descriptors fail early with clear argument errors.

## Tasks

1. Add contract types.
   - `InstanceBatchDescriptor`
   - `InstanceBatchEntry`

2. Integrate with `SceneDocument`.
   - Add public `InstanceBatches`.
   - Add `AddInstanceBatch(...)`.
   - Preserve batches through entry-only mutations except explicit clear.

3. Validate inputs.
   - Empty transforms.
   - Color/id length mismatch.
   - `Guid.Empty` object ids.
   - Mesh/material mismatch.
   - Non-finite transforms/colors.
   - Transparent `Blend` material.

4. Update docs and tests.
   - Root/Core docs mention contract and exclusions.
   - Focused tests cover valid and invalid descriptors.

## Risks

- A contract without runtime behavior can be mistaken as a complete feature. Mitigation: docs and phase artifacts explicitly say runtime/picking are Phase 243.
