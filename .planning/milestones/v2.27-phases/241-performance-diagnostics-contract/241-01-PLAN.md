# Phase 241 Plan: Performance Diagnostics Contract

## Goal

Expose narrow performance diagnostics for viewer and SurfaceCharts without broad renderer refactoring or invented measurements.

## Success Criteria

1. Viewer diagnostics can report draw calls, instance count, vertex count, upload bytes, resident resource count/bytes, and pickable object count where measurable.
2. SurfaceCharts rendering status can report visible tile count and resident tile bytes without changing chart/viewer ownership boundaries.
3. Diagnostics snapshot formatting and support docs label metrics as measured, estimated, or unavailable.
4. Existing object-count diagnostics remain source-compatible and documented as scene counts, not draw-call metrics.

## Tasks

1. Extend viewer diagnostics contract.
   - Add nullable renderer-cost metrics to `VideraBackendDiagnostics`.
   - Add measured/estimated resource fields backed by scene residency.
   - Keep `LastFrameUploadedBytes` source-compatible.

2. Extend SurfaceCharts render status.
   - Add `VisibleTileCount` and `ResidentTileBytes` to snapshot/status.
   - Populate GPU bytes from resident GPU resources.
   - Populate software bytes from resident tile geometry estimates.

3. Update support formatting and docs.
   - Emit new diagnostics in `VideraDiagnosticsSnapshotFormatter`.
   - Label object counts as scene counts and unsupported metrics as `Unavailable`.
   - Update English and Chinese docs plus SurfaceCharts demo/smoke summaries.

4. Verify narrowly.
   - Run formatter, SurfaceCharts render host, bridge integration, repository docs, and demo configuration tests.

## Risks

- Draw-call/instance/vertex counts are not currently measured by the backend path. Mitigation: expose nullable fields and formatter `Unavailable` semantics now; populate later when Phase 243 adds a measurable path.
- SurfaceCharts software bytes are estimates, not native GPU residency truth. Mitigation: docs and demo summary label the source.
