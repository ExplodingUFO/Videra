# Milestone v1.1: Render Pipeline Architecture

**Status:** SHIPPED 2026-04-09  
**Phases:** 9-12  
**Total Plans:** 13

## Overview

`v1.1` 将 Videra 从“alpha 阶段能跑、平台口径真实”的状态，推进到“渲染管线契约清晰、内部 orchestration 可维护、对外扩展入口已经发货并有 sample/docs/guard 固定”的状态。这个里程碑没有扩张大型渲染功能，而是先把后续功能演进最容易失控的边界收口。

## Phases

### Phase 9: Render Pipeline Inventory and Contract Extraction

**Goal:** 把现有 end-to-end frame path 审计成显式 pipeline contract，并抽出稳定 stage/pass 语义。  
**Depends on:** `v1.0` archived baseline  
**Plans:** 3 plans

Plans:
- [x] `09-01` Engine-level frame plan and pipeline snapshot
- [x] `09-02` Diagnostics consume pipeline truth
- [x] `09-03` Architecture/docs truth and repository guards

**Details:**
- 抽出了 `RenderPipelineStage`、`RenderFramePlan`、`RenderPipelineSnapshot`
- 让 `VideraView.BackendDiagnostics` 直接消费 pipeline truth
- 用 docs 和 repository guards 固定 stage vocabulary

### Phase 10: Host-Agnostic Render Orchestration

**Goal:** 把纯渲染 orchestration 从 Avalonia view/session/native host 细节中继续解耦，并建立 headless 可回归验证。  
**Depends on:** Phase 9  
**Plans:** 3 plans

Plans:
- [x] `10-01` Host-agnostic render-session orchestration
- [x] `10-02` `VideraView` session bridge extraction
- [x] `10-03` Docs and boundary guards

**Details:**
- 引入 `RenderSessionInputs`、`RenderSessionSnapshot`、`RenderSessionOrchestrator`
- 引入 `VideraViewSessionBridge`
- 把 `VideraView` 收回到 UI shell，而不是继续承担低层 session sync

### Phase 11: Public Extensibility APIs

**Goal:** 向库使用者开放稳定、可理解的 pipeline 扩展接口，而不是要求改引擎内部实现。  
**Depends on:** Phase 10  
**Plans:** 3 plans

Plans:
- [x] `11-01` Core public extensibility contract
- [x] `11-02` Avalonia host-app path and diagnostics projection
- [x] `11-03` Docs, guards, and final regression matrix

**Details:**
- 发货了 `IRenderPassContributor`
- 发货了 `RegisterPassContributor(...)` / `ReplacePassContributor(...)` / `RegisterFrameHook(...)` / `GetRenderCapabilities()`
- 让 `VideraView.RenderCapabilities` 和 `BackendDiagnostics` 成为宿主应用侧稳定查询入口

### Phase 12: Developer-Facing Samples, Docs, and Compatibility Guards

**Goal:** 用 sample、文档和 contract tests 固定新的扩展模型，防止它再次漂移成内部实现细节。  
**Depends on:** Phase 11  
**Plans:** 4 plans

Plans:
- [x] `12-01` Narrow extensibility sample
- [x] `12-02` Lifecycle and availability contract
- [x] `12-03` English onboarding and repository guards

**Details:**

- 发布 `docs/extensibility.md` 与 `docs/zh-CN/extensibility.md`
- 用 sample/config/docs/localization guards 固定 public extensibility contract

## Milestone Summary

**Key Decisions:**
- 先显式化 pipeline truth，再开放 public extensibility API
- 先抽 internal orchestration seam，再让 `VideraView` 暴露稳定 host-app surface
- public extensibility 必须和 sample、docs、localization guards 一起发货
- 继续把大型渲染功能延后，先验证架构是否能承受真实用户入口

**Issues Resolved:**
- 解决了 pipeline contract 只存在于隐式实现、难以被 docs/tests/diagnostics 共用的问题
- 解决了 render session orchestration 与 Avalonia view/native host 边界过于耦合的问题
- 解决了 public extensibility “有 API、没 sample、没 contract page、没 guard”的半成品状态

**Issues Deferred:**
- compositor-native Wayland embedding
- higher-level macOS safer binding replacement
- plugin-style package discovery model
- 在稳定 contract 之上新增大型渲染功能

**Technical Debt Incurred:**
- `Videra.ExtensibilitySample` 中 contributor status 可能比当前帧晚一帧反映；这是低优先级、非阻塞 debt
- Nyquist validation 在 workflow config 中仍然关闭，milestone audit 依赖 phase VERIFICATION artifacts 和直接仓库证据完成
