# Phase 173 Plan: Primitive-Level Upload and Mixed Transparency Bridge

## Goal

Make imported runtime entries primitive-centric enough that mixed opaque/transparent participation no longer fails at the object guard.

## Tasks

1. Introduce imported runtime object expansion.
   - Build one internal runtime object per imported primitive instance.
   - Preserve transformed mesh payload and material alpha on each runtime object.

2. Update runtime entry truth.
   - Let imported `SceneDocumentEntry` carry multiple internal runtime objects.
   - Keep external object entries single-object.
   - Flatten the runtime scene from entry-owned runtime objects instead of one object per entry.

3. Update residency/upload/engine attachment.
   - Carry runtime-object lists through residency records.
   - Upload and attach all runtime objects for one imported entry.
   - Remove the mixed `Blend`/non-`Blend` guard from the runtime hot path.

4. Add focused tests.
   - Mixed-alpha imported runtime entries no longer fail.
   - Scene document flattening includes all runtime objects from one entry.
   - Upload queue and engine applicator handle multi-object imported entries.

## Acceptance

- Mixed opaque and transparent primitive participation no longer guards the whole imported asset as one object-level failure.
- Runtime upload/render truth is finer than a single `Object3D` carrier.
- Repository tests cover the new primitive-centric bridge.
