# Plan 115-01: Inspection Session Contract

**Phase:** 115
**Goal:** make the typed inspection session contract internally consistent across state capture/restore, sample truth, and repository tests

## Tasks

1. Extend `VideraInspectionState` to carry annotations and thread clone/apply logic through the runtime inspection path.
   Verify: integration coverage proves `CaptureInspectionState()` / `ApplyInspectionState(...)` round-trip annotations alongside camera, selection, snap mode, clipping, and measurements.

2. Simplify inspection bundle persistence so `inspection-state.json` is the single typed session artifact instead of splitting annotations into a parallel file.
   Verify: bundle export/import integration coverage proves annotations still replay correctly and no longer depend on a separate annotation artifact.

3. Align public docs and the interaction sample with the new session truth.
   Verify: sample/repository tests and targeted Release builds pass with wording that teaches one canonical inspection-session flow.

## Success Criteria

1. `VideraInspectionState` is sufficient to restore one inspection session through public APIs, including annotations.
2. `VideraInspectionBundleService` persists/restores the same typed session truth without a parallel annotation file.
3. `Videra.InteractionSample`, `README.md`, and `src/Videra.Avalonia/README.md` describe the same inspection-session contract.
4. Targeted inspection integration tests and sample/repository truth tests pass.
