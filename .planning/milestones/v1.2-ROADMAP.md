# Milestone v1.2: Professional Surface Charts

**Status:** SHIPPED 2026-04-16  
**Phases:** 13-22 (`effective delivered spine: 15-22`)  
**Total Delivered Plans:** 24

## Overview

`v1.2` 把 surface-chart 模块家族从 host-driven thin shell 推进成真正可用的专业 3D 图表子系统，同时继续保持它与 `VideraView` 的 sibling boundary。这个里程碑先交付了轴/图例/probe、GPU 主渲染路径和大数据 data-path，再通过 re-audit 发现并补回原始 runtime/view-state 与 built-in camera workflow 缺口，最后用 cleanup phases 收敛 audit truth 与 batch-read adoption debt。

归档里保留了原始 Phase `13-14` 作为 superseded audit history，因为这两个 phase 的原始承诺并没有在当时 checkout 落地。实际发货 spine 是 Phase `15-22`，其中 Phase `19-20` 恢复了原始 `VIEW-*` / `INT-*` scope，Phase `21-22` 收敛了 post-audit tech debt。

## Milestone Summary

**Key Decisions:**
- 保留 Phase `15-18` 作为已发货 baseline，并追加 Phase `19-20` 恢复原始 `VIEW-*` / `INT-*` scope，而不是伪造 Phase `13-14` 完成。
- 保持 surface-chart 与 `VideraView` 的 sibling boundary，不把 chart semantics 回灌到 viewer 主线。
- 让 GPU main path 与 software fallback 共享同一套 chart contract，并通过 demo/docs/tests 固定 truth。
- 把 cleanup debt 显式建模为 Phase `21-22`，而不是把 milestone audit 的残余问题继续留在口头说明里。

**Issues Resolved:**
- runtime/view-state public contract 缺口已由 Phase `19` 收敛。
- built-in interaction / camera workflow 缺口已由 Phase `20` 收敛。
- summary metadata / superseded verification wording / milestone-artifact drift 已由 Phase `21` 收敛。
- batch tile read seam “存在但未被 live scheduler 消费”的 debt 已由 Phase `22` 收敛。

**Issues Deferred:**
- compositor-native Wayland embedding
- higher-level macOS safer binding replacement
- viewer-side PBR / shadow / shader-graph expansion
- contours / slices / streaming sources / richer analysis layers beyond the base professional chart slice

**Technical Debt Incurred:**
- v1.2 closeout 在本地 planning 层面完成，但仓库仍是 dirty worktree，未形成干净 git tag / immutable release point。
- Raw phase execution history 仍保留在 `.planning/phases/`，未移动到 `v1.2-phases/` archive 目录。
- Nyquist validation 仍然在 workflow config 中关闭，因此 audit 依赖 phase verification artifacts 与直接仓库证据。
