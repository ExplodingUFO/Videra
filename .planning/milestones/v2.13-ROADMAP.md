# Roadmap

## Active Milestone

`v2.13 Package Install Boundary Tightening`

Goal: tighten the default Avalonia install surface and make importer-backed loading an explicit package/install choice so package truth, sample truth, and repository guardrails line up without widening runtime/chart/platform scope.

## Scope Guard

- No renderer/material/runtime feature breadth, no new chart family, no chart-kernel widening, and no backend/UI/platform expansion.
- No broader importer-format coverage, no animation/lighting/shadows/post-processing work, and no `WebGL` / `OpenGL` pursuit.
- No compatibility shims, migration adapters, downgrade paths, or transitional dual-default install contracts.
- No public package publishing or repository release tag creation.
- `.planning` remains local-only and does not define a repository release boundary by itself.
