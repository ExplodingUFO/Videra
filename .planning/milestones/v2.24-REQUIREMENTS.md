# Requirements: v2.24 Analyzer 10 and Dependency Hygiene

**Defined:** 2026-04-27  
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v2.24 Requirements

Requirements for turning analyzer and dependency major updates into deliberate, reviewable maintenance work. This milestone addresses the closed Dependabot analyzer 10 PR and the follow-up package hygiene gap without broad product architecture changes, compatibility layers, release publishing, or new feature breadth.

### Analyzer Rule Policy

- [x] **ARP-01**: Maintainer can identify which analyzer 10 rules are accepted, fixed, suppressed, or explicitly disabled for Videra, including at least `CA1051`, `CA1720`, `CA1822`, and `CA1859`.
- [x] **ARP-02**: Repository configuration documents the difference between actionable analyzer failures and rules intentionally excluded because they would cause public API churn or low-value style churn.
- [x] **ARP-03**: Analyzer upgrade policy keeps `TreatWarningsAsErrors` meaningful without requiring broad architecture rewrites, compatibility shims, or unrelated cleanup.

### Analyzer 10 Adoption

- [x] **ANA-01**: Repository can build and run quality-gate evidence with `Microsoft.CodeAnalysis.NetAnalyzers` `10.0.203` or a deliberately chosen analyzer 10 version.
- [x] **ANA-02**: Analyzer 10 adoption passes the existing Release quality path without hiding compiler errors, package validation errors, native validation failures, or benchmark failures.
- [x] **ANA-03**: Analyzer failures in CI produce focused guidance that points maintainers to rule policy and the smallest expected fix path.

### Dependency Update Hygiene

- [x] **DEP-01**: Dependabot configuration prevents or reduces partial multi-project test-tooling updates such as a single-project `coverlet.collector` bump.
- [x] **DEP-02**: Repository checks detect version drift for repo-wide test tooling packages such as `coverlet.collector` and any other package the milestone explicitly classifies as shared test infrastructure.
- [x] **DEP-03**: Maintainer docs define how to handle robot dependency PRs: merge directly, rebase and retest, close and replace with a repo-owned PR, or defer as a dedicated phase.

### Quality Gate Evidence

- [x] **QGE-01**: CI or repository verification proves analyzer policy and dependency hygiene checks are exercised during the normal quality path.
- [x] **QGE-02**: Local validation commands for analyzer/package hygiene are documented and produce evidence suitable for release-readiness review.
- [x] **QGE-03**: v2.24 closeout records any remaining analyzer rules, dependency-update limitations, or Dependabot follow-ups as explicit deferred items instead of hidden CI debt.

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---|---|
| Broad architecture rewrite | The analyzer upgrade is maintenance work; it must not become Core/Import/Backend/Avalonia/SurfaceCharts redesign. |
| Public API churn for low-value analyzer suggestions | Rules such as `CA1051` and `CA1859` can conflict with performance structs or stable contracts; policy must be deliberate. |
| Compatibility shims or downgrade paths | Project default remains direct fixes and explicit policy, not fallback layers. |
| Central package management migration | Useful later, but too large for this bounded dependency hygiene milestone. |
| Real release publishing or tag creation | v2.24 is maintenance quality work, not a public release operation. |
| Product feature breadth | No animation, skeletal/morph, shadows/post-processing, WebGL/OpenGL, extra UI adapter, new backend, or new chart family work. |
