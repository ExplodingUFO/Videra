# Milestone v1.13: Inspection Fidelity

**Status:** COMPLETE 2026-04-19
**Target phases:** 76-79
**Committed workstreams:** 4
**Research:** skipped — the milestone strategy comes directly from the current repo state, the shipped `v1.12` inspection design notes, and the approved direction to deepen inspection truth before widening product surface

## Overview

`v1.13` 的主线不是再加一批新的 viewer feature，而是把 `v1.12` 刚刚交付的 inspection workflow 从“已经能用”推进到“更可信、更可复现、更容易支持”。当前最值钱的缺口已经很明确：measurement 仍继承 bounds-based picking 的精度画像，clipping 仍以 CPU-side truth path 为主，snapshot export 也优先 deterministic correctness 而不是速度。

因此这轮里程碑聚焦四件事：mesh-accurate picking truth、viewer-first snap modes、same-API fast paths、以及可重放的 inspection bundle。目标不是把 `Videra` 变成 editor，也不是扩一条新的 package/product 线，而是在不改 public story 的前提下，让 inspection 结果更值得信任。

这轮里程碑同样有清晰边界：selection 对 host 仍然保持 object-level public 语义；不做 editor-style gizmos / transform handles；不做 compositor-native Wayland；不做 plugin/package discovery；也不在同一里程碑里把 `Videra.SurfaceCharts.*` 推成 public alpha 包线。

## Phase 76: Mesh-Accurate Hit Truth

**Status:** COMPLETE 2026-04-19  
**Goal:** Replace bounds-based inspection hits with richer mesh-accurate truth while preserving the current object-level public selection story.  
**Depends on:** v1.12 complete  
**Requirements:** `HIT-01`, `HIT-02`, `HIT-03`  
**Plans:** 3

**Success criteria:**
1. Core picking resolves a richer hit record containing at least `ObjectId`, world hit point, world normal, primitive/triangle identity, and hit distance.
2. Measurement, annotation-anchor, and future probe workflows consume the richer hit truth by default without forcing a new primitive-level public selection API.
3. The same restored inspection state yields pick/measure outcomes within a documented epsilon across hardware backends and software fallback.

## Phase 77: Viewer-First Snap Modes and Measurement Reproducibility

**Status:** COMPLETE 2026-04-19  
**Goal:** Add explicit measurement snap modes on top of richer hit truth without drifting into editor semantics.  
**Depends on:** Phase 76  
**Requirements:** `SNAP-01`, `SNAP-02`, `SNAP-03`  
**Plans:** 3

**Success criteria:**
1. Users can switch measurement snapping between `Free`, `Vertex`, `EdgeMidpoint`, `Face`, and `AxisLocked` inside the existing viewer-first `Measure` story.
2. Snap behavior composes with `Navigate` / `Select` / `Annotate` / `Measure` and does not require transform handles, gizmos, or authoring affordances.
3. Restored inspection state preserves snap mode and measurement outcomes reproducibly enough to satisfy the same epsilon contract taught by the milestone.

## Phase 78: Same-API Fast Paths and Inspection Benchmarks

**Status:** COMPLETE 2026-04-19  
**Goal:** Keep `ClippingPlanes` and `ExportSnapshotAsync(...)` stable while adding narrow same-API fast paths and benchmark evidence.  
**Depends on:** Phase 76  
**Requirements:** `PERF-12`, `PERF-13`, `PERF-14`  
**Plans:** 3

**Success criteria:**
1. Repeated clipping work can reuse cached deterministic truth for the same normalized plane signatures without changing the outward `ClippingPlanes` contract.
2. Snapshot export prefers live readback where compatible and falls back to the current software export path without changing the outward API contract.
3. Dedicated picking, clipping, and snapshot benchmarks plug into the existing benchmark workflow and produce trend-friendly evidence instead of one-off local numbers.

## Phase 79: Reproducible Inspection Bundle and Documentation Truth

**Status:** COMPLETE 2026-04-19  
**Goal:** Turn inspection support into a replayable artifact workflow and align docs around the fidelity story.  
**Depends on:** Phases 76-78  
**Requirements:** `DIAG-06`, `DIAG-07`, `DOC-10`  
**Plans:** 3

**Success criteria:**
1. Hosts can export an inspection bundle containing inspection state, diagnostics snapshot, rendered snapshot, and asset manifest through a narrow support workflow.
2. Interaction/demo surfaces can generate and restore bundles so camera, clipping, measurements, annotations, and snap mode round-trip as the same support truth, while consumer smoke emits the same bundle contract from packed public packages.
3. Root README, Avalonia README, `ARCHITECTURE.md`, troubleshooting, and alpha-feedback docs all describe the same inspection-fidelity story and explicit non-goals.

## Milestone Summary

**Key decisions:**
- Prioritize inspection fidelity over widening package/product surface area.
- Improve hit truth underneath the existing public object-level selection contract instead of reopening the viewer API boundary.
- Add snap modes that guide inspection work, but stop short of editor tooling.
- Use same-API fast paths with truthful fallback instead of introducing separate performance-only APIs.
- Treat the inspection bundle as a support/reproduction artifact, not as the start of a larger project/package format.

**Planned outcomes:**
- Mesh-accurate picking becomes the shared truth source for measurements, anchors, and future inspection probes.
- Clipping truth reuse and snapshot export get capability-aware acceleration without sacrificing deterministic cross-backend behavior.
- Inspection benchmarks make pick/clip/export regressions visible in the same workflow that already carries viewer and surface-chart evidence.
- Alpha support can ask for one reproducible inspection bundle instead of a manually assembled bug report.

**Issues deferred:**
- Richer measurement types such as angle/area/volume workflows beyond the current snap-first scope
- Editor-style clipping gizmos, transform handles, or general scene authoring
- Native compositor-hosted Wayland work
- Publishing `Videra.SurfaceCharts.*` as public consumer packages
- Plugin/package discovery or a broader extensibility system
- Another deep performance milestone beyond the new inspection fast paths and benchmarks
