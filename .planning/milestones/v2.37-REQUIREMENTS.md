# Requirements: v2.37 Dependabot Dependency Triage Closure

**Defined:** 2026-04-28  
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v2.37 Requirements

Requirements for this milestone. Each maps to one roadmap phase.

### Dependabot Inventory

- [x] **DEP-01**: Maintainers can see every open Dependabot PR with branch, package family, old/new versions, touched files, CI status, and mergeability.
- [x] **DEP-02**: Maintainers can see which dependency PRs overlap, supersede each other, or must be serialized before any update branch is modified.
- [x] **DEP-03**: Dependency triage records explicit merge, replace, or close decisions without changing unrelated product code.

### Compatibility Verification

- [x] **VER-01**: Accepted analyzer and test-tooling updates are verified with the narrowest relevant restore/build/test commands before merge.
- [x] **VER-02**: Accepted runtime/package dependency updates are verified against public package boundaries and representative consumer smoke paths.
- [x] **VER-03**: Verification output records any warnings, API breaks, transitive dependency changes, or reason a Dependabot PR is rejected.

### Targeted Remediation

- [ ] **FIX-01**: Any dependency-induced failure is fixed in a scoped branch or worktree that owns only the affected package/test area.
- [ ] **FIX-02**: Remediation does not loosen analyzer quality gates, suppress tests broadly, or add fallback compatibility layers.
- [ ] **FIX-03**: Each remediation has a Beads handoff note with owner, changed files, validation command, and PR update path.

### CI and Merge Closure

- [ ] **CI-01**: Each accepted dependency update is observed through GitHub checks after push or PR refresh.
- [ ] **CI-02**: Superseded or rejected Dependabot PRs are closed with rationale after the accepted path is clear.
- [ ] **CI-03**: Final branch state, Beads ready queue, Docker Dolt status, and Git worktree status are clean after merge/closeout.

## Future Requirements

Deferred to future milestones.

### Dependency Policy

- **DEPF-01**: A broader dependency upgrade policy, automated grouping strategy, or release-train process can be designed after this backlog is closed.
- **DEPF-02**: Major framework/runtime target changes remain deferred until a dedicated compatibility milestone.

### Product Work

- **PRODF-01**: New renderer, chart, Demo, importer, backend, or API feature work is deferred until dependency triage is closed.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Broad package-management redesign | Current goal is to close existing robot PRs, not redesign dependency policy. |
| Runtime target upgrade beyond current project intent | Dependency PRs should not silently change supported runtime/platform contracts. |
| Product feature work | This milestone is maintenance and compatibility focused. |
| Analyzer/test suppression to make upgrades pass | Failures should be fixed narrowly or the PR should be rejected. |
| Merging all Dependabot PRs blindly | Overlapping PRs may need supersession, replacement, or closure. |
| Public release, package publication, release tag, or feed mutation | Dependency triage is not a release milestone. |
