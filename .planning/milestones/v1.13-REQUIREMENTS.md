# Requirements: Videra v1.13 Inspection Fidelity

**Defined:** 2026-04-19
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v1.13 Requirements

### Mesh-Accurate Inspection Hits

- [x] **HIT-01**: picking resolves a mesh-accurate hit record that includes at least `ObjectId`, world hit point, world normal, primitive/triangle identity, and hit distance.
- [x] **HIT-02**: measurement, annotation-anchor, and future inspection-probe workflows consume the richer hit truth by default while host-facing selection semantics remain object-level.
- [x] **HIT-03**: the same scene and restored inspection state yield pick/measure outcomes within an explicit epsilon across hardware backends and software fallback.

### Viewer-First Measurement Fidelity

- [x] **SNAP-01**: users can switch measurement snapping between `Free`, `Vertex`, `EdgeMidpoint`, `Face`, and `AxisLocked` without entering an editor-style manipulation mode.
- [x] **SNAP-02**: snap behavior stays aligned with the existing `Navigate` / `Select` / `Annotate` / `Measure` interaction story and does not require transform handles or gizmos.
- [x] **SNAP-03**: restored inspection state preserves active snap mode and measurement outcomes reproducibly enough to satisfy the documented epsilon contract.

### Same-API Fast Paths and Benchmarks

- [x] **PERF-12**: `ClippingPlanes` keeps its current public contract while the runtime can reuse cached deterministic clip truth for repeated normalized plane signatures instead of recomputing every request.
- [x] **PERF-13**: `ExportSnapshotAsync(...)` keeps its current public contract while compatible live sessions can prefer direct readback and transparently fall back to the current software export path when needed.
- [x] **PERF-14**: dedicated inspection benchmarks cover picking, clipping, and snapshot export and integrate with the existing benchmark workflow so regressions become trend evidence instead of ad hoc observations.

### Inspection Support and Documentation Productization

- [x] **DIAG-06**: hosts can export a reproducible inspection bundle that contains inspection state, diagnostics snapshot, rendered snapshot, and an asset manifest.
- [x] **DIAG-07**: interaction/demo surfaces can generate and restore inspection bundles so camera, clipping, measurements, annotations, and snap mode round-trip as the same support truth, while consumer smoke emits the same bundle contract for automated evidence.
- [x] **DOC-10**: root docs, Avalonia docs, architecture docs, troubleshooting, and alpha-feedback guidance all describe the same inspection-fidelity story and current non-goals consistently.

## v1.14+ / Future Requirements

### Inspection Depth Expansion

- **INSPECT-02**: richer measurement types such as angles, polygon areas, or volume-like analysis remain deferred until snap-first accuracy and reproducibility prove stable in real alpha use.
- **EDIT-02**: editor-style clipping gizmos, transform handles, or general scene authoring remain deferred because `Videra` is a viewing/presentation stack, not a modeling tool.

### Platform Expansion

- **WAYLAND-04**: native compositor-hosted Wayland embedding remains deferred until inspection-fidelity alpha users prove it blocks adoption more than hit truth or support reproducibility does.

### Packaging and Extensibility Expansion

- **PKG-04**: promoting `Videra.SurfaceCharts.*` to public consumer packages remains deferred unless the strategy deliberately shifts from viewer depth to product-width expansion.
- **EXT-04**: plugin/package discovery for viewer extensibility remains deferred while the public model stays C#-first and in-process through `VideraEngine`.

### Deeper Performance Hardening

- **PERF-15**: another inspection-performance milestone beyond the same-API fast paths remains deferred until the new pick/clip/snapshot benchmarks identify the next dominant bottleneck.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Editor-style scene authoring, transform gizmos, or arbitrary object editing | `Videra` remains a viewer/presentation stack, not a modeling tool |
| Primitive-level public selection/editing APIs | this milestone improves internal hit truth without reopening the public selection contract |
| Native compositor-hosted Wayland embedding | current ROI is lower than making inspection workflows precise, reproducible, and supportable |
| Publishing `Videra.SurfaceCharts.*` as public packages | this milestone stays on the viewer-depth line rather than widening product surface |
| Plugin/package discovery or a new extensibility system | current public extensibility root remains `VideraEngine`; new loading/discovery seams are not needed for this scope |
| Another large viewer/runtime architecture rewrite | recent milestones already stabilized the main seams; this milestone should pressure them through more truthful inspection behavior |
