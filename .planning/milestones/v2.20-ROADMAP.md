# Milestone v2.20: API/DX Productization and Diagnostics Contract Closure

**Status:** shipped 2026-04-26
**Phases:** 212-216
**Total Plans:** 5
**Audit:** passed

## Overview

`v2.20` made the alpha consumer path easier to install, wire, diagnose, and support by tightening importer/load APIs, demo diagnostics, backend capability truth, and package/docs contracts without widening the architecture.

The milestone deliberately avoided a broad Core / Import / Backend / Avalonia / SurfaceCharts rewrite, generic 3D engine parity work, renderer feature breadth, new UI adapters, new backend/platform families, SurfaceCharts/viewer merger, and editor-style persistence.

**Status:** complete  
**Depends on:** Phase 212

## Milestone Summary

### Key Decisions

- Keep v2.20 focused on alpha productization rather than architecture expansion.
- Preserve explicit importer package installation; `Videra.Avalonia` does not regain transitive importer dependencies.
- Keep importer/load diagnostics structured and cancellation-aware without adding compatibility adapters.
- Report backend advanced API support through capability truth, not fallback behavior.
- Keep SurfaceCharts diagnostics inside the chart demo; do not make `SurfaceChartView` a `VideraView` mode.

### Issues Resolved

- Import package README drift against the package matrix and Avalonia package-reference truth.
- Consumer boilerplate around importer-backed loading and multi-format selection.
- Missing per-file import diagnostics and support-ready import reports.
- Demo support artifacts lacked one copyable OS/runtime/backend/import/render-capabilities bundle.
- Advanced backend seams were callable without a visible capability boundary.
- SurfaceCharts demo lacked a dedicated active-path rendering diagnostics panel.

### Issues Deferred

- Repo-only `Videra Doctor` remains future work.
- Additional allocation gates remain future work unless importer/demo diagnostics paths show measurable churn.
- Broader backend API redesign remains out of scope.
- Exhaustive GPU host/fallback matrix UI remains out of scope for SurfaceCharts demo.

### Technical Debt

- Historical phase 13/14 verification files still report old gaps; they remain acknowledged process debt outside `v2.20`.
- Existing quick-task artifact `260421-gre-videra-surfacecharts-demo` remains an acknowledged process metadata gap.
- Phase 216 solution build passed with two pre-existing SurfaceCharts integration-test warnings.
