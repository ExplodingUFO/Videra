# Roadmap

## Active Milestone

`v2.14 WPF Hosted Adapter Validation`

Goal: validate one second desktop host shell on the existing viewer/runtime line by tightening the current `WPF on Windows` smoke proof, without widening into a second public UI package or expanding runtime/chart/backend/import scope.

## Scope Guard

- No second public UI adapter/package line, no WinUI 3 / MAUI expansion, and no broader desktop-shell proliferation.
- No renderer/material/runtime feature breadth, no new chart family, no chart-kernel widening, and no backend/platform expansion beyond the existing Windows D3D11 path.
- No broader importer-format coverage, no animation/lighting/shadows/post-processing work, and no `WebGL` / `OpenGL` pursuit.
- No compatibility shims, migration adapters, downgrade paths, or transitional dual-shell public contracts.
- `.planning` remains local-only and does not define a repository release boundary by itself.
