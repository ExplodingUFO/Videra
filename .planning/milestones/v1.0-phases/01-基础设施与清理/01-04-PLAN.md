Per LOG-02: Remove ALL Console.WriteLine calls. Per LOG-03: Remove macOS debug counters.

**Step 1: Add Microsoft.Extensions.Logging to all platform and Avalonia projects.**

For each platform project (.csproj), add:
```xml
<PackageReference Include="Microsoft.Extensions.Logging.Abstractions" Version="9.0.11" />
```
Use Logging.Abstractions (not full Logging) to keep dependencies minimal. The platform projects use ILogger interface; the actual Serilog configuration lives in Videra.Demo.

**Step 2: MetalCommandExecutor.cs -- Remove debug counters AND Console.WriteLine.**

This is the most critical file per CONCERNS.md. Specific actions:
1. DELETE these static field declarations entirely:
   - `private static int _frameDebugCount = 0;`
   - `private static int _setBufferCallCount = 0;`
   - `private static int _drawCallCount = 0;`
   - `private static int _viewportCallCount = 0;`
2. DELETE all code blocks that use these counters:
   - `if (_frameDebugCount == 0)` ... `_frameDebugCount++;`
   - `if (_setBufferCallCount % 300 == 0)` ... `_setBufferCallCount++;`
   - `if (_drawCallCount % 60 == 0)` ... `_drawCallCount++;`
   - `if (_viewportCallCount % 60 == 0)` ... `_viewportCallCount++;`
   - ALL related Console.WriteLine calls inside these conditionals
3. Add `using Microsoft.Extensions.Logging;` at top
4. Add `private readonly ILogger _logger;` field
5. Inject ILogger through constructor or use a static logger
6. Replace any remaining Console.WriteLine with _logger.LogDebug or _logger.LogInformation
7. For hot-path methods (SetVertexBuffer, DrawIndexed, SetViewport) -- use LogDebug level ONLY, never LogInformation, to avoid perf impact in production

**Step 3: MetalResourceFactory.cs, MetalBackend.cs -- Replace Console.WriteLine.**
- Add ILogger field to each class
- Replace Console.WriteLine with appropriate logger calls
- Use LogInformation for resource creation events
- Use LogError for failures

**Step 4: Windows platform (D3D11ResourceFactory.cs, D3D11CommandExecutor.cs).**
- Replace Console.WriteLine with ILogger calls
- The D3D11ResourceFactory has HRESULT error messages -- preserve the HRESULT in structured logging: `logger.LogError("Failed to create vertex buffer. HRESULT: {HResult}", result);`

**Step 5: Linux platform (VulkanBackend.cs, VulkanResourceFactory.cs, VulkanCommandExecutor.cs).**
- Replace Console.WriteLine with ILogger calls
- Preserve Vulkan error information in structured log properties

**Step 6: Avalonia (VideraView.cs, VideraView.Input.cs, VideraNativeHost.cs, VideraLinuxNativeHost.cs, VideraMacOSNativeHost.cs).**
- Replace all Console.WriteLine with ILogger calls
- VideraView.cs has 15 Console.WriteLine calls -- these are UI-level logs, use LogInformation/LogDebug
- Input logging (VideraView.Input.cs) has 10 calls -- use LogDebug level
