# Requirements: v2.51 Professional Chart Output and Dataset Evidence

**Defined:** 2026-04-29
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## Milestone Intent

Make Plot-authored SurfaceCharts easier to export, attach, reproduce, and diagnose as professional 3D chart outputs.

This milestone starts from the v2.50 chart model where `VideraChartView` is the only shipped chart control and `VideraChartView.Plot.Add.Surface(...)`, `.Waterfall(...)`, and `.Scatter(...)` are the public runtime data-loading path. It should add chart-local output and reproducibility evidence without restoring removed alpha APIs, widening backend/runtime scope, or hiding unsupported behavior behind fallback/downshift paths.

## v2.51 Requirements

### Inventory

- [x] **INV-01**: Inventory current chart output, report, dataset metadata, demo, consumer smoke, Doctor, docs, tests, and guardrail surfaces before changing APIs.
- [x] **INV-02**: Classify output/export/report/dataset evidence gaps as implement, document, defer, or reject.
- [x] **INV-03**: Define concise target examples for professional chart output reports and reproducible dataset evidence.
- [x] **INV-04**: Record non-goals: no old chart view compatibility, no direct `Source` API, no generic plotting engine, no backend expansion, no hidden fallback/downshift, and no god-code workbench.

### Chart Output

- [x] **OUT-01**: Users can request deterministic chart output metadata or report evidence through chart-local Plot-owned APIs.
- [x] **OUT-02**: Output evidence identifies chart kind, series identity, active series, style, color map, precision profile, dimensions, and rendering status.
- [x] **OUT-03**: Unsupported output capabilities are reported through explicit diagnostics rather than fallback behavior.
- [x] **OUT-04**: Output contracts remain chart-local and do not expose backend internals or create a renderer/export subsystem rewrite.

### Dataset Reproducibility

- [x] **DATA-01**: Surface, waterfall, and scatter Plot series can report deterministic dataset identity and shape metadata.
- [x] **DATA-02**: Dataset evidence includes relevant dimensions, ranges, units, sampling/spacing where available, point or cell counts, and precision policy.
- [x] **DATA-03**: Dataset metadata supports support-bundle and minimal-reproduction workflows without leaking mutable internal runtime state.
- [x] **DATA-04**: Dataset evidence updates deterministically when Plot series are added, removed, cleared, or replaced.

### Demo And Support Evidence

- [x] **DEMO-01**: SurfaceCharts demo output/support artifacts use only `VideraChartView.Plot.Add.*` for chart data loading.
- [x] **DEMO-02**: Demo and consumer-smoke artifacts include chart output report status and dataset reproducibility evidence.
- [x] **DEMO-03**: Doctor or repository support checks can parse the new evidence without requiring a running UI.
- [x] **DEMO-04**: Demo growth remains bounded; no broad chart editor, workbench, or hidden fallback UI is introduced.

### Guardrails And Documentation

- [x] **GUARD-01**: Repository guardrails prevent old chart controls and direct public `Source` APIs from returning.
- [x] **GUARD-02**: Repository guardrails prevent chart output work from becoming backend expansion, generic plotting engine scope, compatibility shims, or hidden fallback/downshift behavior.
- [x] **GUARD-03**: Package/sample docs show concise Plot-owned output and dataset evidence usage.
- [x] **GUARD-04**: Public roadmap and Beads state identify v2.51 phases, dependencies, and status.

### Verification

- [x] **VER-01**: Each phase records Beads ownership, dependencies, status, and handoff notes.
- [x] **VER-02**: Implementation phases use isolated branch/worktree execution when they change code.
- [x] **VER-03**: Each phase ends with clean Beads status, branch/worktree cleanup, and pushed repository state.

## Future Requirements

- Actual image/PDF/vector export pipeline beyond deterministic output/report evidence.
- Publication-layout editor, visual-regression screenshot gates, or broad chart workbench.
- Additional chart families beyond surface, waterfall, and scatter.
- Generic plotting engine semantics beyond the current chart-local Plot model.
- Backend expansion, renderer rewrite, shader/material graph, or GPU-driven chart runtime.
- Compatibility shims for removed alpha chart APIs.

## Out of Scope

| Item | Reason |
|------|--------|
| Old `SurfaceChartView` / `WaterfallChartView` / `ScatterChartView` public views | `VideraChartView` is the single shipped chart control. |
| Direct public `Source` API | `Plot.Add.*` is the public runtime data-loading path. |
| Generic plotting engine | This milestone adds bounded output/evidence contracts for the current 3D chart model only. |
| Backend expansion or renderer rewrite | Professional output evidence should not widen native backend scope. |
| Hidden fallback/downshift | Unsupported output behavior must remain explicit diagnostics. |
| God-code workbench | Demo/support work must stay bounded and sample-first. |
