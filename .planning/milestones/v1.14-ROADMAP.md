# Milestone v1.14: Compatibility and Quality Hardening

**Status:** ✅ SHIPPED 2026-04-20
**Phases:** 80-83
**Total Plans:** 12

## Overview

`v1.14` focused on hardening alpha truth instead of widening product surface. The milestone tightened Linux host/display-server messaging, promoted packaged consumer and advanced sample validation into routine PR evidence, normalized the shipped backend minimum contract, and made the definition of alpha-ready `green` more actionable across CI, docs, diagnostics, and support.

The milestone intentionally did not add an `OpenGL` backend, compositor-native Wayland embedding, or any broader public API surface. The goal was to make the current support story more explicit and more testable before considering further expansion.

## Phases

### Phase 80: Backend Minimum Contract

**Goal:** Normalize the built-in backend abstraction around one documented common minimum contract.  
**Depends on:** v1.13 complete  
**Plans:** 3 plans

Plans:

- [x] 80-01: document the shipped minimum backend contract and normalize unsupported seams
- [x] 80-02: align built-in backend pipeline behavior and smoke coverage
- [x] 80-03: lock public/backend docs to the same minimum-contract truth

**Details:**
- The built-in backends now expose one documented minimum contract centered on the viewer’s real shipped behavior.
- `Vulkan` shader creation and `Metal` resource-set binding now fail explicitly instead of implying a portable contract.
- `Metal` pipeline creation now routes through the same built-in pipeline path as the viewer contract.
- Docs and repository tests now keep the support story on `D3D11` / `Vulkan` / `Metal` without implying `OpenGL`.

### Phase 81: Linux Host Compatibility Truth

**Goal:** Make Linux native-host/display-server truth explicit and consistent across runtime diagnostics, support wording, and validation evidence.  
**Depends on:** Phase 80  
**Plans:** 3 plans

Plans:

- [x] 81-01: add a stable display-server compatibility summary to diagnostics
- [x] 81-02: push Linux host truth into consumer-smoke artifacts and support docs
- [x] 81-03: lock Linux host wording and evidence semantics in repository tests

**Details:**
- Diagnostics snapshots now emit `DisplayServerCompatibility` as a stable support-ready line.
- Consumer smoke status, JSON output, and script output now all surface the same Linux host-path truth.
- Linux docs explicitly distinguish direct `X11`, Wayland-session `XWayland`, and unsupported compositor-native Wayland hosting.

### Phase 82: Consumer Path and Sample Validation Gates

**Goal:** Turn the packaged consumer path and advanced public references into executable merge-time evidence instead of best-effort documentation truth.  
**Depends on:** Phases 80-81  
**Plans:** 3 plans

Plans:

- [x] 82-01: make packaged consumer smoke routine PR and push evidence
- [x] 82-02: add explicit sample-contract evidence for the advanced public samples
- [x] 82-03: align README and release docs with the stronger evidence story

**Details:**
- `consumer-smoke.yml` now runs on `pull_request`, `push`, and manual dispatch across the supported host matrix.
- `sample-contract-evidence` now validates `Videra.ExtensibilitySample` and `Videra.InteractionSample` configuration/runtime contracts explicitly.
- Public docs now describe packaged consumer smoke and sample-contract evidence as part of the normal alpha-ready line.

### Phase 83: Quality Gate Tightening and Evidence Stewardship

**Goal:** Convert today’s advisory quality signals into an actionable alpha-ready definition of green.  
**Depends on:** Phase 82  
**Plans:** 3 plans

Plans:

- [x] 83-01: remove high-value analyzer findings on touched scene/runtime/inspection paths
- [x] 83-02: add packaged-consumer and curated-Core warnings-as-errors evidence
- [x] 83-03: align CI/docs around one shared definition of green

**Details:**
- Stateless scene/runtime helpers were simplified into static utilities, and inspection bundle export now awaits async file writes.
- `Invoke-ConsumerSmoke.ps1` gained a build-only warnings-as-errors mode so CI validates the real packaged consumer path rather than a misleading raw smoke-project build.
- `quality-gate-evidence` now means packaged consumer path plus curated Core warning-clean evidence, while benchmark review remains label-gated and intentionally non-blocking.

## Milestone Summary

**Key Decisions:**

- Prioritized compatibility truth and merge-time evidence over adding a fourth graphics backend.
- Treated Linux compatibility as a host/display-server truth problem first, not an `OpenGL` gap.
- Promoted packaged consumer and advanced sample validation into routine PR evidence.
- Tightened the definition of alpha-ready `green` without pretending benchmark review is already a hard numeric blocker.

**Issues Resolved:**

- Backend minimum-contract drift across `D3D11`, `Vulkan`, and `Metal`.
- Linux display-server ambiguity across diagnostics, docs, and consumer evidence.
- Lack of routine merge-time validation for packaged consumer and advanced sample paths.
- Mismatch between “quality evidence” wording and the actual package-consumer path CI needed to validate.

**Issues Deferred:**

- Strengthening the repository guard so it asserts “no `OpenGL` product promise” semantics instead of token presence only.
- Expanding warnings-as-errors beyond curated consumer/Core surfaces.
- Converting benchmark review from label-gated stewardship into a harder numeric blocker.
- Compositor-native Wayland embedding and any future `OpenGL` evaluation.

**Technical Debt Incurred:**

- The repo guard for `BACK-03` is still semantically weak.
- The strict warning gate is intentionally scoped rather than whole-repo.
- Benchmark review is still human-reviewed and label-gated by design.
