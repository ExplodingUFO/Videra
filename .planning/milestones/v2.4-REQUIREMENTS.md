# Requirements

Active milestone: `v2.4 Runtime Truth Unification`

Runtime-truth requirement numbering starts at `RUNT-01`.

- [x] `RUNT-01` Before implementation starts, lock `v2.4` to runtime-truth unification and freeze explicit non-goals around animation, lighting/shadows/post-processing, broader glTF feature breadth, new UI adapters, and `WebGL` / `OpenGL` or backend/API expansion.
- [x] `RUNT-02` `SceneDocument` and related runtime state preserve authoritative imported-asset identity and runtime asset truth instead of relying on `Object3D` as the long-term public scene boundary.
- [x] `RUNT-03` Upload, render, and backend-rebind paths consume the same runtime truth without duplicated or lossy scene semantics.
- [x] `RUNT-04` Docs, diagnostics, sample wording, and repository guardrails all describe the unified runtime truth and the current static-scene boundary honestly.

## Out of Scope

| Item | Excluded from `v2.4` |
|------|------------------------|
| Animation / skeleton / morph / mixer | Not part of this milestone. |
| Lighting / shadows / environment maps / post-processing | Not part of this milestone. |
| Broader static glTF/PBR breadth beyond the current shipped baseline | Deferred until runtime truth is unified. |
| New UI adapters | Not part of this milestone. |
| `Wayland` / `OpenGL` / `WebGL` / backend API expansion | Not part of this milestone. |
| New chart families or `SurfaceCharts` product-line widening | Not part of this milestone. |
| Broad runtime/platform rewrites unrelated to scene truth unification | Not part of this milestone. |
