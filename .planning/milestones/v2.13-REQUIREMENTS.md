# Requirements

Active milestone: `v2.13 Package Install Boundary Tightening`

Package/install-boundary requirement numbering starts at `PIS-01`.

## Milestone Goal

On the existing Avalonia viewer line, tighten the default install surface and make importer-backed model loading an explicit package/install choice so package truth, sample truth, and repository guardrails line up without widening runtime/chart/platform scope.

## Requirements

- [x] `PIS-01` Package/install boundary scope lock
  - Freeze `v2.13` to package/install-surface tightening on the shipped Avalonia viewer line.
  - Keep renderer/material/runtime feature breadth, chart-family widening, backend/UI/platform expansion, broader importer-format coverage, compatibility shims, and migration adapters out of scope.
  - Keep public package publishing and repository tag creation out of scope.

- [x] `PIS-02` Avalonia dependency surface slimming
  - Reduce the default dependency/install surface of `Videra.Avalonia` so viewer-only hosts stop carrying importer packages by default.
  - Keep the work inside package/install-surface reshaping instead of widening runtime or renderer behavior.
  - Prove the slimmer dependency truth with focused package, consumer, or repository evidence.

- [x] `PIS-03` Explicit importer-backed load path
  - Make the importer-backed `LoadModelAsync(...)` path explicit on the Avalonia line instead of depending on transitive importer installation.
  - Keep viewer-only usage coherent without silent importer pull-in.
  - Prove the explicit importer-enabled path with focused sample or consumer evidence.

- [ ] `PIS-04` Install truth, size evidence, and guardrails
  - Align package-size/install-surface evidence, docs, support wording, and repository guardrails with the tightened install boundary.
  - Keep one honest viewer-only story and one honest importer-enabled story on the shipped viewer line.
  - Leave broader runtime/chart/platform/importer breadth to future milestones.

## Out of Scope

| Item | Excluded from `v2.13` |
|---|---|
| Runtime/chart/platform breadth | No renderer/material/runtime feature expansion, no new chart family, no chart-kernel widening, no backend/UI/platform expansion. |
| Import breadth | No new importer formats, no animation/skeleton/morph, and no broader static-material feature work. |
| Release shape | No public package publishing, no repository release tagging, and no release-candidate workflow changes. |
| Compatibility | No compatibility shim, migration adapter, downgrade path, or transitional dual-default install contract. |
