# Milestone v1.6: Scene Pipeline Truth and Backend Surface Closure

**Status:** COMPLETE 2026-04-17
**Target phases:** 42-46
**Committed workstreams:** 5
**Research:** skipped — milestone strategy was supplied directly in the prompt

## Overview

`v1.6` 不再继续扩“大而全”的 demo，而是顺着 `v1.5` 已经建立的 runtime / scene / graphics seams，把最后几个高价值过渡接缝收干净。当前最关键的问题不是 viewer 壳层没立住，而是 `SceneDocument` 还没有真正成为 single source of truth，导入和上传仍紧耦合，backend rebuild 仍混着临时 software fallback，platform backends 也还在通过 `LegacyGraphicsBackendAdapter` 维持过渡态。

这个 milestone 先把 scene pipeline 做实，再把 backend abstraction v2 的平台迁移做完，最后只补一个聚焦型 lab 去证明 batch import、atomic replace、backend rebind 和 fallback explanation 这套新 contract。这样 demo 只是验证新内核，不会反过来绑架内核设计。

## Phase 42: SceneDocument Authoritative Scene Owner

**Status:** COMPLETE  
**Goal:** 让 `SceneDocument` 成为 `VideraViewRuntime` 的权威 scene owner，并把 viewer mutations 改成 document-first。  
**Depends on:** Previous milestones shipped runtime shell thinning and the first backend-neutral scene seam  
**Requirements:** `SCENE-03`, `SCENE-04`  
**Plans:** 3 complete

**Success criteria:**
1. `AddObject`、`ReplaceScene`、`ClearScene` 和 `Items` 不再先改 `Engine.SceneObjects` 再回填 document，而是先改 `SceneDocument`。
2. runtime 通过明确的 publish/apply 过程把 document truth 投到 engine，而不是继续依赖 engine state 反推 scene truth。
3. scene owner 在 source replacement、runtime recreation、backend rebind 这几类路径上保持一致，不再出现双真相。

## Phase 43: Import and Upload Pipeline Separation

**Status:** COMPLETE  
**Goal:** 把 import 与 upload 彻底拆开，并把批量导入改成有界并行 + 原子 replace 语义。  
**Depends on:** Phase 42  
**Requirements:** `IMPORT-01`, `IMPORT-02`  
**Plans:** 3 complete

**Success criteria:**
1. `LoadModelAsync()` 和批量加载路径先产出 CPU-side `ImportedSceneAsset` 或 document fragment，而不是立刻上传资源。
2. `LoadModelsAsync()` 不再串行 `foreach await`，而是有界并行 import，再由 runtime 做一次原子 apply/replace。
3. 取消、部分失败和 superseded load 不会破坏当前 active scene truth。

## Phase 44: Asset Rehydration and Rebind Truth

**Status:** COMPLETE  
**Goal:** 让 runtime 能从 `SceneDocument` / imported assets 在 backend 或 surface recreation 后重建资源，并收掉临时 software fallback 的 steady-state 混淆。  
**Depends on:** Phases 42-43  
**Requirements:** `ASSET-01`, `ASSET-02`  
**Plans:** 3 complete

**Success criteria:**
1. backend loss、surface recreate、session rebind 后，active scene 可以从 CPU-side truth 重新上传，不要求用户重新导入。
2. runtime 选择 resource factory / upload path 的逻辑对 diagnostics 是可解释、可验证的，不把 `SoftwareResourceFactory` 当成默认真相。
3. 重新绑定过程不会制造重复 upload、重复 scene ownership，或不透明的 fallback 行为。

## Phase 45: Backend Abstraction v2 Platform Migration

**Status:** COMPLETE  
**Goal:** 把 platform backend 实现进一步迁到 device/surface split 的新 seam，尽量缩小 `LegacyGraphicsBackendAdapter` 的 steady-state 作用。  
**Depends on:** Phase 44  
**Requirements:** `GFX-03`, `GFX-04`  
**Plans:** 3 complete

**Success criteria:**
1. platform backends 对 device、surface、frame lifecycle 的真实 ownership 进一步内收，不再只靠 adapter 包旧接口。
2. orchestrator / runtime / diagnostics 在 backend v2 migration 过程中继续讲同一套 truth，包括 fallback reason、backend rebind 和 capability projection。
3. built-in backends 直接满足 `IGraphicsDevice` / `IRenderSurface`，adapter 只保留给非迁移或自定义 legacy backend。

## Milestone Summary

**Key decisions:**
- `SceneDocument` now owns viewer scene truth directly; runtime mutations update the document first and only then project into engine state.
- Import and upload are now separate steps: imported assets stay backend-neutral until the active resource factory uploads or rehydrates them.
- Built-in software, D3D11, Vulkan, and Metal backends now satisfy `IGraphicsDevice` / `IRenderSurface` directly; `LegacyGraphicsBackendAdapter` remains only as a compatibility seam for non-migrated or custom backends.
- `Videra.Demo` got a narrow Scene Pipeline Lab instead of a larger catch-all viewer demo, and docs/tests were aligned around that narrower role.

**Issues deferred:**
- Public promotion of `SceneDocument` / imported-scene asset types
- Shared-device multi-surface or offscreen productization beyond validation-only paths
- Complete removal of `LegacyGraphicsBackendAdapter` for every external or custom legacy backend
