# Roadmap

## Active Milestone

`v2.18 Mixed Transparent Primitive Ordering Baseline`

Goal: allow mixed `Blend` and non-`Blend` imported primitives to survive the shipped static-scene viewer/runtime path with deterministic per-primitive transparent ordering on the current renderer path, without widening into order-independent transparency, refraction/transmission, shadows, environment maps, post-processing, animation, new public package lines, or broader transparency-system abstraction work.

## Scope Guard

- No order-independent transparency, weighted blended OIT, dual-depth peeling, refraction/transmission, or broader transparency-system framework work.
- No shadows, shadow maps, environment maps, post-processing, fullscreen pass framework, or broader lighting-system expansion.
- No animation, skeletons, morph targets, mixers, chart-family widening, UI-shell expansion, backend/platform expansion, or importer breadth work.
- No compatibility shims, migration adapters, downgrade paths, or transitional dual-path transparency contracts.
- Desktop proof validation in this milestone is limited to the existing repository-owned viewer proof hosts, and each desktop program used in validation must survive `10` seconds without crashing.
- `.planning` remains local-only and does not define a repository release boundary by itself.

## Current Baseline

- `v2.15-v2.18` established one bounded lighting/material-consumption and transparent-ordering line on the native static-scene viewer/runtime path without widening into shadows, environment maps, post-processing, animation, or a new public package line.
- The canonical runtime path now expands one imported entry into multiple internal runtime objects so mixed `Blend` and non-`Blend` primitives survive the shipped static-scene seam with deterministic per-object transparent ordering.
- The current public viewer story remains Avalonia-first: `Videra.Avalonia` plus one matching `Videra.Platform.*` package, with `WpfSmoke` staying repository-only validation/support evidence.
- The current public chart story remains the shipped `Videra.SurfaceCharts.*` package line on the existing Avalonia shell.
