# Requirements

Archived milestone: `v2.5 Static glTF/PBR Breadth`

Static-runtime-breadth requirement numbering starts at `GLTFB-01`.

- [x] `GLTFB-01` Before implementation starts, lock `v2.5` to one bounded static-scene glTF/PBR breadth slice on the unified runtime-truth path and freeze explicit non-goals around animation, skeletons, morph targets, lighting/shadows/post-processing, extra UI adapters, and `Wayland` / `OpenGL` / `WebGL` or backend/API expansion.
- [x] `GLTFB-02` The unified runtime path preserves per-primitive material participation for non-`Blend` static-scene mixes instead of collapsing imported assets to one object-wide alpha truth, while continuing to reject mixed `Blend` / non-`Blend` assets until transparent primitives become independently sortable.
- [x] `GLTFB-03` The imported-asset/runtime material contract then broadens with one richer static-scene glTF/PBR subset, chosen for `v2.5` as occlusion plus texture-transform-aware texture bindings, on top of the preserved runtime path.
- [x] `GLTFB-04` Docs, support wording, repository proof surfaces, and guardrails all describe the broadened static-scene baseline and its exclusions honestly.

## Out of Scope

| Item | Excluded from `v2.5` |
|------|------------------------|
| Animation / skeleton / morph / mixer | Not part of this milestone. |
| Lights / shadows / environment maps / post-processing | Not part of this milestone. |
| Broader engine/runtime parity or generic material frameworks | Not part of this milestone. |
| New UI adapters | Not part of this milestone. |
| `Wayland` / `OpenGL` / `WebGL` / backend API expansion | Not part of this milestone. |
| New chart families or `SurfaceCharts` widening | Not part of this milestone. |
| Runtime, renderer, shader, or backend consumption beyond imported-asset/runtime truth | Not part of this milestone. |
