# Roadmap

## Active Milestone

`v2.9 Release Candidate Final Validation`

Goal: make the existing public package/release-candidate boundary auditable, credential-free, abortable, and ready for later human-controlled release cutover without creating tags or publishing packages.

## Scope Guard

- No runtime/rendering/material/chart/backend/UI-adapter breadth.
- No compatibility shims, fallback release paths, migration adapters, or transitional contracts.
- No public package publishing, GitHub release creation, or repository release tag creation.
- `.planning` remains local-only and does not define a repository release boundary by itself.
