# Requirements

Active milestone: `v2.17 Bounded Emissive/Normal-Map Renderer Consumption Baseline`

Emissive/normal-map requirement numbering starts at `ENM-01`.

## Milestone Goal

Consume the already-retained emissive and normal-map-ready material truth on the existing static-scene viewer/runtime path without widening into shadows, environment maps, post-processing, animation, new UI/package lines, or broader runtime/package expansion.

## Requirements

- [x] `ENM-01` Emissive/normal-map scope lock
  - Freeze `v2.17` to emissive and normal-map-ready renderer consumption only on the existing viewer/runtime line.
  - Keep shadows, environment maps, post-processing, animation, new UI/package lines, backend/platform expansion, broader importer/chart widening, compatibility shims, and migration adapters out of scope.
  - Keep public package publishing and repository tag creation out of scope.

- [x] `ENM-02` Emissive/normal-map renderer contract
  - Add only the minimum renderer/shader seam needed to consume emissive and normal-map-ready inputs on the shipped static-scene path.
  - Avoid generic lighting/material frameworks, render-target abstraction widening, or speculative advanced-runtime abstractions.
  - Prove the new contract with focused code evidence.

- [x] `ENM-03` Repo-owned emissive/normal-map proof and validation
  - Strengthen one repository-owned viewer proof path so it demonstrates the emissive/normal-map renderer-consumption slice on a supported host path.
  - Ensure every desktop program used in milestone validation survives `10` seconds without crashing.
  - Keep diagnostics/support artifacts coherent with the new slice and keep the proof repository-owned and non-public.

- [x] `ENM-04` Emissive/normal-map truth and guardrails
  - Align docs, support wording, and repository guardrails with the shipped emissive/normal-map renderer-consumption baseline.
  - Keep shadows, environment maps, post-processing, animation, and broader advanced-runtime breadth explicitly deferred.
  - Leave broader lighting/material-system decisions to later milestones.

## Out of Scope

| Item | Excluded from `v2.17` |
|---|---|
| Advanced renderer breadth | No shadows, shadow maps, environment maps, post-processing, fullscreen pass framework, or broader lighting-system work. |
| Runtime abstraction breadth | No generic material-system rewrite, no new render-target/resource-set abstraction, and no wider shader-system public model. |
| Product breadth | No animation, skeletons, morph targets, mixers, new chart families, chart-kernel widening, UI-shell expansion, backend/platform expansion, or importer-format widening. |
| Compatibility | No compatibility shim, migration adapter, downgrade path, or transitional dual-path renderer contract. |
