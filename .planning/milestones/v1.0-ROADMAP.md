# Videra 开源准备 - 路线图

**项目目标：** 跨平台 3D 渲染引擎的可靠性 — 在三个主流平台上稳定运行，具备完整的测试覆盖和文档。

**粒度：** Coarse（激进组合，专注关键路径）

**阶段数：** 8

---

- [x] **Phase 1: 基础设施与清理** - 建立测试框架、日志系统，清理调试代码，并以 GitHub-hosted native validation 关闭 TEST-03
- [x] **Phase 2: 质量与可靠性** - 实现结构化错误处理、安全加固、资源管理
- [x] **Phase 3: 跨平台完善** - 完成深度缓冲一致性、ObjC runtime hardening、Linux X11 native + Wayland-session XWayland compatibility，并以 hosted native-validation 收口；compositor-native Wayland 与更高层 macOS safer binding deferred

- [x] **Phase 5: 架构解耦与原生边界硬化** - 解耦渲染组合层，收紧 native 生命周期边界，并明确 Rust 仅作为窄边界候选
- [x] **Phase 6: 分发与平台打包正确性** - 收敛跨平台包模型、发布校验和平台分发边界
- [x] **Phase 7: 渲染契约一致性与生命周期安全** - 修复引擎并发销毁风险，统一软件后端与风格/线框契约
- [x] **Phase 8: Demo 完整度与用户反馈真实性** - 收口 Demo 的状态真相、交互 capability gating 和对外文档叙事

### Phase 5: 架构解耦与原生边界硬化

**目标：** 将后端组合、渲染会话、native host 与平台互操作边界解耦，先通过 C# 硬化消除高风险 native 路径，再基于证据决定是否需要极小范围 Rust 引入

**依赖：** Phase 4（当前文档和验证入口已基本齐备，适合开始结构收口）

**需求：** PERF-01, PERF-02, SEC-02, MACOS-01, LINUX-02, PLAT-03

**成功标准**（必须为真）：
1. 后端解析与组合不再由 `Videra.Core` 直接承担，平台发现路径清晰可测试
2. `VideraView` 不再独占 backend/session/timer/native-handle 全部职责，渲染会话职责被显式抽离
3. macOS 与 Linux 高风险 native 边界完成第一轮返回值/句柄/回滚硬化
4. 形成书面的 Rust 决策结论：默认不引入；若引入，仅限粗粒度、证据驱动的小边界

**计划：** 5 plans complete

**Depends on:** Phase 4
**Plans:** 5 plans

## Deferred Beyond This Milestone

- **Linux compositor-native Wayland embedding**
  - Deferred because the current Avalonia `11.3.9` Linux native-host stack still embeds through X11 handles. The current milestone truthfully closes on `X11` native plus Wayland-session `XWayland` compatibility.
- **macOS higher-level safer binding replacement**
  - Deferred because the current milestone already centralized and hardened `ObjCRuntime`, but replacing manual Objective-C runtime calls with a different binding layer should be evaluated as a separate, higher-risk platform phase.
