# Roadmap

## Active Milestone

`v2.16 Bounded Broader Lighting Baseline`

Goal: broaden the shipped static-scene lighting baseline through the existing style/uniform seam, without widening into shadows, environment maps, post-processing, emissive/normal-map renderer consumption, animation, or a new public package line.

## Scope Guard

- No shadows, shadow maps, environment maps, post-processing, fullscreen pass framework, or new render-target/resource-set abstraction.
- No emissive or normal-map renderer consumption in this milestone; keep those retained runtime truths deferred.
- No animation, skeletons, morph targets, mixers, chart-family widening, UI-shell expansion, backend/platform expansion, or importer breadth work.
- No compatibility shims, migration adapters, downgrade paths, or transitional dual-lighting public contracts.
- Desktop proof validation in this milestone is limited to the repository-owned hosts already on the viewer proof path, and each desktop program used in validation must survive `10` seconds without crashing.
- `.planning` remains local-only and does not define a repository release boundary by itself.

**Status:** completed

**Requirement:** `BLI-01`

Success criteria:

- The milestone does not widen into shadows, environment maps, post-processing, emissive/normal-map renderer consumption, animation, or a new package line.
- The implementation phases stay narrow: one shader/uniform contract phase, one repo-owned proof/validation phase, and one docs/guardrails phase.

### Phase 197: Broader Style-Lighting Contract

**Status:** completed
**Depends on:** Phase 196
**Requirement:** `BLI-02`

**Status:** completed

Success criteria:

**Status:** completed

**Requirement:** `BLI-04`

Success criteria:

- The same sources keep shadows, environment maps, post-processing, emissive/normal-map renderer consumption, animation, and broader advanced-runtime breadth explicitly deferred.
- The milestone is ready for audit and closeout after implementation merges.
