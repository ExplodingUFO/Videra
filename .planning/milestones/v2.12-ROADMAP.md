# Roadmap

## Active Milestone

`v2.12 SurfaceCharts Residency and Probe Efficiency`

Goal: reduce tile residency churn, probe latency, and unnecessary invalidation/allocation on the existing `SurfaceChartView` line so large-matrix interaction gets steadier without widening chart/runtime/platform scope.

## Scope Guard

- No new chart family, generic `Chart3D`, chart-kernel widening, viewer/runtime breadth, backend/UI/platform expansion, or `WebGL` / `OpenGL` pursuit.
- No compatibility shims, migration adapters, downgrade paths, or transitional contracts.
- No public package publishing or repository release tag creation.
- `.planning` remains local-only and does not define a repository release boundary by itself.
