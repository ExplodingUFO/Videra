## Phase Details

### Phase 292: SurfaceCharts Support Gap Inventory

**Status:** complete  
**Bead:** Videra-0w9.1  
**Goal:** Map the current demo, support, Doctor, Performance Lab, and lifecycle gaps before changing code.

**Requirements:** INV-01, INV-02

**Success criteria:**
1. Current SurfaceCharts demo support/reporting behavior is documented with file paths and missing fields.
2. Current Performance Lab, Doctor, support-bundle, and visual-evidence vocabulary is mapped to SurfaceCharts needs.
3. Recent test-host or lifecycle hang lessons are translated into scoped reliability risks.
4. Broad architecture and feature requests are explicitly deferred before implementation starts.

### Phase 293: SurfaceCharts Support Report UX

**Status:** complete  
**Bead:** Videra-0w9.2  
**Goal:** Add a focused support report surface to the SurfaceCharts demo using existing diagnostics truth.

**Requirements:** SUX-01, SUX-02, SUX-03

**Success criteria:**
1. Demo users can copy or export a concise SurfaceCharts support report.
2. The report includes chart type, rendering status, fallback state, interaction quality, visible/resident tile or point data, streaming/FIFO counters, and runtime/backend context where those values already exist.
3. The implementation reuses existing diagnostics/runtime data instead of introducing a parallel support model.
4. English and Chinese docs explain how maintainers should use the report and what it does not prove.

### Phase 294: SurfaceCharts Lifecycle Reliability Guardrails

**Status:** complete  
**Bead:** Videra-0w9.3  
**Goal:** Add deterministic demo/test-host lifecycle evidence and timeout handling for automated verification.

**Requirements:** LIFE-01, LIFE-02, LIFE-03

**Success criteria:**
1. SurfaceCharts demo or integration-host smoke verification has a deterministic startup/shutdown or timeout contract.
2. Failure output includes enough timeout/shutdown evidence to diagnose lifecycle hangs.
3. The change does not add blanket test skips, broad CI suppressions, or hidden compatibility fallbacks.
4. Chart rendering semantics remain unchanged outside the lifecycle guardrail path.

### Phase 295: Performance Lab and Doctor Evidence Alignment

**Status:** complete  
**Bead:** Videra-0w9.4  
**Goal:** Align Performance Lab, SurfaceCharts support reports, and Doctor/support discovery vocabulary and guardrails.

**Requirements:** EVID-01, EVID-02, EVID-03

**Success criteria:**
1. Performance Lab and SurfaceCharts support evidence use consistent names for scenario, settings, rendering status, interaction quality, streaming/FIFO, and visual-evidence references.
2. Doctor or support-evidence discovery can surface the SurfaceCharts support report when present, while remaining passive and repo-local.
3. Tests cover present, missing, and unavailable support-evidence states.
4. Repository guardrails prevent documentation from overclaiming support evidence as benchmark, visual-regression, or GPU-performance proof.

### Phase 296: Milestone Closeout and Coordination Cleanup

**Status:** complete  
**Bead:** Videra-0w9.5  
**Goal:** Close Beads, branch, worktree, Dolt, and planning traceability cleanly after implementation.

**Requirements:** CLOSE-01

**Success criteria:**
1. All v2.38 phase Beads are closed or explicitly deferred with handoff notes.
2. `.beads/issues.jsonl` is exported and Docker Dolt state is pushed.
3. Feature branches and worktrees created for the milestone are merged or removed.
4. Git status, Beads ready queue, and Dolt status are clean at close.
