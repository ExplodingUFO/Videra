# Roadmap

## Active Milestone

- `v1.24 Performance and Size Gates`
- Status: `Phase 119 complete; v1.24 shipped locally and on master`
- Focus: performance evidence, benchmark thresholds, package-size budgets, and packaged publish-smoke are now explicit shipping gates; next step is defining the SurfaceCharts productization milestone

### Phase 117: Performance Benchmark Contract

**Goal:** define one stable benchmark contract and evidence set for the shipped viewer and SurfaceCharts workflows so later thresholds judge the same slices every time.
**Depends on:** `v1.20-v1.23` product/runtime baseline plus the existing benchmark and repository-guard infrastructure
**Plans:** 3 completed

Plans:

- [x] 117-01: normalize the benchmark scenarios and reported metrics for viewer first-frame, steady interaction, and SurfaceCharts interaction/probe slices
- [x] 117-02: emit one stable machine-readable evidence artifact that CI, review, and support workflows all reference
- [x] 117-03: align docs and repository truth around the same benchmark contract instead of label-gated interpretation alone

### Phase 118: Performance Threshold Enforcement

**Goal:** promote agreed benchmark metrics from evidence-only review into explicit CI failure gates with clear regression reporting.
**Depends on:** Phase 117
**Plans:** 3 completed

Plans:

- [x] 118-01: define numeric regression thresholds and failure semantics for the normalized benchmark slices
- [x] 118-02: wire CI and repository guards so those thresholds block regressions automatically
- [x] 118-03: align benchmark workflow, release guidance, and support wording with the new hard performance-gate contract

### Phase 119: Package Size and Publishability Gates

**Goal:** add explicit package-size budgets and publish-smoke validation so the shipped viewer stack stays bounded, reproducible, and releaseable.
**Depends on:** Phase 117
**Plans:** 3 completed

Plans:

- [x] 119-01: define package-size budgets and artifact deltas for the public package line
- [x] 119-02: add supported publish-path smoke validation for the shipped viewer stack
- [x] 119-03: align package/release docs and repository guards with the enforced shipping-gate story

## Operational Baseline

- `v1.20-v1.23` stabilized the viewer/runtime product boundary, scene/material runtime, static glTF/PBR baseline, and inspection product flow
- `v1.24` converted benchmark evidence, benchmark thresholds, package-size budgets, and packaged consumer smoke into explicit merge-time shipping gates on the existing public package path
- the highest-value next gap is no longer release-gate ambiguity on the viewer line; it is productizing the independently mature `SurfaceCharts` family without widening into generic `Chart3D` or second-UI scope first
