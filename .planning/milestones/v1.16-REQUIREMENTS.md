# Requirements: Videra v1.16 SurfaceCharts Adoption Surface

**Defined:** 2026-04-20
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v1.16 Requirements

### Product Boundary

- [ ] **CHART-01**: `SurfaceCharts` has one canonical `first chart` story, expressed consistently across root docs, SurfaceCharts module docs, and the chosen public sample.
- [ ] **CHART-02**: `SurfaceChartView` public contract is documented around `ViewState`, interaction, rendering status, overlays, and chart-local ownership without reintroducing `VideraView` or editor-style assumptions.
- [ ] **CHART-03**: `SurfaceCharts` docs and samples keep the chart stack explicitly independent from `VideraView` while still explaining how it fits into the broader repository.

### Adoption Path

- [ ] **ADOPT-01**: new consumers can follow one explicit source-first evaluation path for `SurfaceCharts` without reverse-engineering the repo from the full demo.
- [ ] **ADOPT-02**: the source-first evaluation path includes a narrow `first chart` sample or minimal entrypoint distinct from the broader demo application.
- [ ] **ADOPT-03**: CI validates the chosen `SurfaceCharts` evaluation path as a real outward-facing adoption surface instead of only a repo-internal build target.

### Support and Release Truth

- [ ] **TRUTH-01**: README, support matrix, package/release docs, and planning artifacts all state that `SurfaceCharts` remains source-first in this milestone and does not imply a public package promise.
- [ ] **TRUTH-02**: `SurfaceCharts` issue reporting and troubleshooting routes users through the same diagnostics/runtime truth emitted by the chosen evaluation path.
- [ ] **TRUTH-03**: maintainer release guidance distinguishes viewer package truth from `SurfaceCharts` source-first truth so releases do not imply unsupported consumer paths.

## Deferred Future Requirements

- **PKG-06**: publish `Videra.SurfaceCharts.*` as public consumer packages only after the source-first adoption path proves valuable and supportable.
- **PKG-07**: add preview-feed SurfaceCharts packaging only if source-first adoption evidence shows that package distribution is the real blocker.
- **PERF-17**: introduce harder SurfaceCharts performance thresholds only after product-surface truth and consumer evidence are stable.
- **WAYLAND-06**: compositor-native chart hosting remains deferred with the same native-host constraints as the rest of the repo.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Stable public `Videra.SurfaceCharts.*` package release | `v1.16` proves adoption surface first, not GA package promise |
| Editor-style chart authoring tools | Product direction remains viewer/inspection oriented |
| Whole-repo warnings-as-errors expansion | This milestone focuses on consumer-facing chart adoption, not repo-wide analyzer closure |
