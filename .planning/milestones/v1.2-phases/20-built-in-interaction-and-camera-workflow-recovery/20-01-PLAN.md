- Keep the full gesture path inside `Videra.SurfaceCharts.Avalonia`; do not add shared viewer routers or chart-specific members to `VideraView`.
- Preserve the already-shipped `Shift + LeftClick` pin-toggle behavior exactly; built-in navigation must coexist with it.
- Implement visible dolly through `ViewState.DataWindow` scaling around an anchor sample or window center; do not depend on camera-distance-only zoom the current projection cannot show.
</guardrails>

<threat_model>
- Medium: left-button navigation can conflict with the shipped pin-toggle gesture. Mitigation: reserve `Shift + LeftClick` for the existing pin path and gate orbit on non-shift left-drag with a travel threshold.
- Medium: panning or dolly math can move the active window outside metadata bounds. Mitigation: clamp all new data-window writes against metadata and cover edge-window regression cases.
- Medium: pointer capture bugs can leave the controller stuck in an active gesture. Mitigation: explicitly manage capture on drag start and always reset on release/capture loss.
- Block on high: any solution that requires host code or `VideraView` to orchestrate orbit / pan / zoom behavior.
</threat_model>
