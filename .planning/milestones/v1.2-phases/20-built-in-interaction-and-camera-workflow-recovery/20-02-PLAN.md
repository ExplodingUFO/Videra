<guardrails>
- Implement built-in focus as chart-local screen-to-data-window math; do not add renderer hit-testing or any viewer-owned interaction layer.
- Preserve the existing public Phase-19 commands as the deterministic reset/preset seam; built-in focus should complement them, not bypass them.
- Keep pinned probes, hover state, and overlay truth working before, during, and after focus interaction.
</guardrails>

<threat_model>
- Medium: focus selection can drift if it tries to infer 3D geometry instead of the current sample-space window. Mitigation: map drag rectangles directly against control bounds and the active `ViewState.DataWindow`.
- Medium: tiny selection rectangles can create degenerate or unstable focus windows. Mitigation: enforce a minimum span and treat below-threshold gestures as no-op/click behavior.
- Medium: reset/preset commands can become inconsistent after interaction updates camera and data window in different ways. Mitigation: keep all focus writes inside `SurfaceViewState` and explicitly test post-interaction `FitToData()` / `ResetCamera()` behavior.
- Block on high: leaving the demo dependent on host-authored overview/detail viewport presets after this built-in focus work lands.
</threat_model>
