# Phase 176 Plan: Static glTF/PBR Renderer Consumption Scope Lock

## Goal

Lock the `v2.11` milestone boundary before implementation work starts.

## Tasks

1. Freeze the milestone vocabulary.
   - Define `v2.11` as renderer consumption of already-imported static glTF/PBR metadata.
   - Keep the distinction between imported-only truth and renderer-consumed truth explicit.

2. Freeze milestone scope.
   - Keep animation, skeleton, morph, light/shadow/post-processing breadth, advanced transparency-system work, broader importer coverage, new chart families, extra UI adapters, and backend/API expansion out of scope.
   - Keep compatibility shims, downgrade paths, and migration adapters out of scope.

3. Freeze the implementation sequence.
   - `Phase 177`: texture transform + UV-set renderer consumption.
   - `Phase 178`: occlusion renderer consumption + golden-scene evidence.
   - `Phase 179`: docs/support/repository guardrails.

4. Update milestone state.
   - Mark `MDC-01` complete locally.
   - Set the next actionable step to Phase 177 on an isolated worktree/branch.

## Acceptance

- `v2.11` stays locked to renderer consumption of existing static glTF/PBR metadata.
- Broader renderer/runtime/import/chart/platform breadth remains out of scope.
- The next implementation phases have narrow acceptance criteria.
