# Requirements: v2.36 Beads Remote Sync PR Closure

**Defined:** 2026-04-28  
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v2.36 Requirements

Requirements for this milestone. Each maps to one roadmap phase.

### Beads Remote Sync

- [x] **BRS-01**: Maintainers can verify that Docker-backed Beads uses the `Videra` Dolt database at `127.0.0.1:3306` with the expected GitHub remote.
- [x] **BRS-02**: Maintainers can push Beads Dolt state from the Docker server to the configured GitHub remote without interactive credential prompts.
- [x] **BRS-03**: Beads working state, Dolt status, and Git working tree status are clean before PR execution resumes.

### PR Preparation

- [x] **PRP-01**: Maintainers can see the exact current branch state, upstream status, Beads issue state, and candidate PR diff before any push.
- [x] **PRP-02**: The PR branch is pushed with only intended source/docs/tooling changes and no local-only `.planning` leakage.
- [x] **PRP-03**: A GitHub PR is opened with a concise summary, verification evidence, risk notes, and Beads coordination references.

### CI Observation

- [x] **CIO-01**: Required GitHub checks for the PR are observed and recorded with pass/fail/pending status.
- [x] **CIO-02**: CI failures, if any, are classified into targeted Beads follow-up issues with owner, scope, and validation command.
- [x] **CIO-03**: No broad product refactor or unrelated cleanup is introduced while observing CI.

### Targeted Remediation

- [x] **FIX-01**: Each CI failure fix is scoped to the failing check and tracked by its own Beads issue.
- [x] **FIX-02**: Independent fixes can be assigned to separate worktrees/branches when they do not touch overlapping files.
- [x] **FIX-03**: Fixes are verified locally with the narrowest relevant command before updating the PR branch.

### Merge and Cleanup

- [x] **MRG-01**: The PR is merged only after required checks are green or explicitly accepted by the maintainer.
- [x] **MRG-02**: Remote/local feature branches and temporary worktrees are cleaned after merge without deleting unrelated user work.
- [x] **MRG-03**: Beads phase issues are closed with clear handoff notes, `.beads/issues.jsonl` is exported, and Docker Dolt state is pushed.

## Future Requirements

Deferred to future milestones.

### Release Operations

- **RELF-01**: Public package publication, release tag creation, and feed mutation remain deferred until a human-approved release milestone.
- **RELF-02**: Broader release automation should only be revisited after PR/CI merge flow is stable.

### Product Work

- **PRODF-01**: New renderer, chart, Demo, or API features are deferred until the current branch is merged cleanly.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Public release, package publication, release tag creation, or feed mutation | This milestone closes a PR/CI merge loop only. |
| Broad product refactor | CI remediation must stay targeted to observed failures. |
| New renderer, chart, Demo, or API feature work | The immediate goal is repository and Beads remote state convergence, not product breadth. |
| Replacing GitHub checks with Beads state | Beads coordinates work; GitHub CI remains the source of truth for PR checks. |
| Deleting unrelated branches, worktrees, or user changes | Cleanup must be scoped to this milestone's branches/worktrees only. |
| Treating remotesapi doctor warnings as product blockers | Docker Dolt remote push is the active sync path; remotesapi warning remains diagnostic-only unless it blocks handoff. |
