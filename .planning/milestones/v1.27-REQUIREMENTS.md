## v1 Requirements

Requirements committed for the `v1.27` milestone. Each requirement maps to one roadmap phase.

### Second Chart Family Scope

- [x] **CHRT-05**: Before implementation starts, the milestone locks one concrete second chart family and freezes explicit non-goals without widening into generic `Chart3D` or third-family work.

### Thin Second Family

- [x] **CHRT-06**: The chosen second chart family stays thin and direct on the existing Avalonia chart path; it must not introduce generic chart-platform scaffolding, a god kernel, or a second UI/runtime expansion.

### Dual-Family Product Truth

- [x] **CHRT-07**: Docs, support guidance, package/release truth, and repository guards all describe the same dual-family chart story without implying generic `Chart3D` breadth.

## Out of Scope

Explicitly excluded from `v1.27` to keep the milestone focused on one real second chart family.

| Feature | Reason |
|---------|--------|
| Generic `Chart3D` or broad chart-platform abstractions | This milestone validates one concrete second family, not a generic chart framework |
| A third chart family or broad series matrix | The goal is one reusable proof, not parallel chart-product expansion |
| New UI adapters or runtime/backend rewrites | UI/runtime scope was already stabilized before this milestone |
| Lighting/post-processing/animation expansion | Advanced runtime breadth remains a later `2.0` milestone |
