# Phase 187 Plan: Install Truth, Size Evidence, and Guardrails

## Goal

Align docs, package-size/install-surface evidence, and repository guardrails with the tightened Avalonia install boundary.

## Tasks

1. Update canonical docs/support truth.
   - Teach one viewer-only install story and one importer-enabled install story.
   - Remove wording that implies `Videra.Avalonia` still carries importer packages by default.

2. Update package/install evidence.
   - Refresh package-size or package-boundary evidence to match the slimmer default path.
   - Keep evidence focused on shipped install truth instead of runtime breadth.

3. Update repository guardrails.
   - Fix repository tests that still assume `Videra.Avalonia` directly references importer projects.
   - Add or adjust guards so docs and install truth stay aligned.

4. Verify the docs/evidence/guardrail slice.
   - Run focused repository guard tests.
   - Run any narrow package/install evidence checks touched by the phase.
   - Run `git diff --check`.

## Acceptance

- Docs and repository guards describe the viewer-only vs importer-enabled split honestly.
- Package/install evidence matches the slimmer default Avalonia boundary.
- The milestone is ready for local audit and closeout after the phase merges.
