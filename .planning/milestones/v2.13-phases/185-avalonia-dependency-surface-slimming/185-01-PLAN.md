# Phase 185 Plan: Avalonia Dependency Surface Slimming

## Goal

Slim the default `Videra.Avalonia` dependency surface without widening product scope.

## Tasks

1. Remove default importer coupling from `Videra.Avalonia`.
   - Delete direct importer project references from `src/Videra.Avalonia/Videra.Avalonia.csproj`.
   - Stop `SceneImportService` from hard-wiring importer packages.

2. Add the narrowest explicit importer seam.
   - Keep `LoadModelAsync(...)` / `LoadModelsAsync(...)` on `VideraView`.
   - Introduce one explicit `VideraViewOptions.ModelImporter` callback.
   - Clear importer-side reuse state when the configured importer changes.

3. Keep repository source consumers coherent.
   - Update the source-tree sample paths that actually call `LoadModelAsync(...)`.
   - Keep the demo importer button working with explicit importer wiring.
   - Update focused runtime tests to cover both the missing-importer failure and the configured-importer path.

4. Verify only the code seams touched by this phase.
   - Focus on `SceneImportService`, `VideraView` integration, and source-tree sample builds.
   - Defer package-consumer smoke, docs, support wording, and repository guardrails to later phases.

## Acceptance

- `Videra.Avalonia` no longer carries importer packages by default.
- Importer-backed loading requires explicit configuration on the Avalonia line.
- Focused tests and source-tree sample builds prove the slimmer dependency truth without widening runtime/chart/platform scope.
