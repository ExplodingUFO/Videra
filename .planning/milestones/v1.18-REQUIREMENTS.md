# Requirements: Videra v1.18 SurfaceCharts Analytics Core

**Defined:** 2026-04-20
**Archived:** 2026-04-20
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v1.18 Requirements

### Geometry and Axis Contracts

- [x] **MODEL-01**: Maintainers can represent surface geometry as either a regular grid or explicit `x/y` coordinate grids without flattening non-uniform spacing into index-linear assumptions.
- [x] **MODEL-02**: Surface axis contracts can express `Linear`, `Log`, `DateTime`, and explicit-coordinate scale semantics while preserving chart labels, units, camera fit inputs, and probe math inputs.
- [x] **MODEL-03**: `SurfaceMatrix` remains a supported convenience entry point that adapts cleanly onto the generalized regular-grid contract instead of becoming the only first-class data model.

### Scalar Fields and Missing Data

- [x] **FIELD-01**: Surface payloads can provide `HeightField` independently from an optional `ColorField` so color mapping is no longer forced to bind to `z`.
- [x] **FIELD-02**: Missing samples, holes, masks, and `NaN` regions are modeled as first-class data semantics rather than renderer-only special cases.

### Render Fast Paths

- [x] **RENDER-01**: Recoloring resident tiles flows through a shader/LUT upload path without rebuilding or rewriting full per-vertex color buffers.
- [x] **RENDER-02**: Surface shading uses derived normals from local surface gradients, including seam-safe handling across tile boundaries.
- [x] **RENDER-03**: Render residency no longer requires unconditional `ToArray()` duplication of source tile values when promoting tiles into resident state.

### Benchmark and Milestone Truth

- [x] **EVID-01**: Benchmarks cover resident-tile recolor, interactive orbit frame cost, probe latency, and tile residency churn instead of only compile/build correctness.
- [x] **EVID-02**: Benchmark evidence also captures cache-miss burst and native-host resize/rebind cost so the next analysis features can be prioritized against measured hot paths.
- [x] **STORY-01**: Planning and milestone truth describe `v1.18` as deepening `SurfaceCharts` into a professional analytics library while explicitly deferring generic `Chart3D`, public package expansion, and new backend work.

## Requirement Outcomes

### Validated

- `MODEL-01` / `MODEL-02` / `MODEL-03` are validated through `SurfaceGeometryGrid`, `SurfaceAxisScaleKind`, guarded unsupported `Log` construction, regular-grid compatibility preservation, and camera-fit / probe-path test coverage.
- `FIELD-01` / `FIELD-02` are validated through `SurfaceScalarField`, `SurfaceMask`, masked reduction logic, render/probe propagation, and repository/docs truth updates for source-first analytics adoption.
- `RENDER-01` / `RENDER-02` / `RENDER-03` are validated through shader/LUT recolor, derived normals, low-copy resident scalar handling, backend binding updates, and targeted rendering/native contract coverage.
- `EVID-01` / `EVID-02` / `STORY-01` are validated through the focused benchmark suite, benchmark smoke tests, benchmark build verification, benchmark docs alignment, and merged PR `#23` CI.

### Accepted Carry-Forward Debt

- `MODEL-02`: `Log` remains explicitly reserved until raw axis values and display-space coordinates are separated.
- `FIELD-02`: cache manifest v1 still cannot serialize explicit-grid or non-linear-axis metadata.
- `RENDER-03`: low-copy residency assumes `SurfaceTile` instances are immutable snapshots.
- `EVID-02`: render-host contract-path benchmarks intentionally do not claim real driver/swapchain/compositor cost.

### Deferred Future Requirements

- **ANALYTICS-01**: add interpolated probe output with explicit `Exact` / `Interpolated` / `Approximate` / `LOD-derived` confidence semantics after the core grid/scalar contracts stabilize.
- **ANALYTICS-02**: add dual/multi-probe delta readouts for `Δx` / `Δy` / `Δz` / distance / slope once probe truth is no longer grid-index bound.
- **ANALYTICS-03**: add contour and wireframe overlays after the scalar/normal pipeline can support them without backtracking.
- **CAMERA-01**: add orthographic view, camera presets, and perspective/orthographic switching after the generalized axis contract lands.
- **SLICE-01**: add axis-aligned slicing and arbitrary cross-section profiles once explicit coordinates and probe semantics are stable.
- **SERIES-01**: add `WaterfallSeries3D` only after the surface analytics core is deep enough to reuse.
- **SERIES-02**: add `PointLine/ScatterSeries3D` and surface-overlay markers after a second 3D series is justified.
- **SERIES-03**: add `SurfaceMeshSeries3D` after the explicit-grid/scalar work proves the right contract direction.
- **PKG-09**: publish `Videra.SurfaceCharts.*` as stable public consumer packages only after the analytics contracts stop changing rapidly.
- **PLATFORM-08**: revisit `OpenGL` or compositor-native Wayland only if the analytics roadmap exposes a real backend-coverage blocker instead of a contract/product-depth blocker.
