# Roadmap

## Active Milestone

### `v2.6 Render Pipeline Diagnostics Closure`

- Status: complete; ready for milestone audit and closeout
- Scope: make the existing viewer/runtime render-pipeline diagnostics contract coherent and assertable without widening into lighting/shadows, animation, post-processing, extra UI adapters, `Wayland` / `OpenGL` / `WebGL`, backend API breadth, or new glTF/material feature work

## Operational Baseline

- `v2.4` unified runtime truth around `SceneDocument`, imported assets, upload, render, and backend rebind.
- `v2.5` broadened static imported-material truth with per-primitive non-`Blend` participation plus occlusion and texture-transform metadata while keeping renderer/shader/backend consumption out of scope.
- The next highest-value gap is not more glTF breadth; it is making the existing render-pipeline diagnostics and proof surfaces tight enough to support future runtime work safely.

## Phases

### Phase 156: Render Pipeline Diagnostics Scope Lock

- Goal: lock `v2.6` to render-pipeline diagnostics closure before any code changes start.
- Depends on: `v2.5` closeout
- Status: `complete`

Plans:

- [x] lock the milestone to existing viewer/runtime pass and diagnostics truth
- [x] freeze non-goals around lighting/shadows/post-processing, animation, extra UI adapters, `Wayland` / `OpenGL` / `WebGL`, backend API expansion, and new material/glTF feature breadth
- [x] choose the exact diagnostics/proof slices that can be verified without a broad render framework rewrite

### Phase 157: Pipeline Snapshot Metrics Contract

- Goal: make the existing pipeline snapshot and diagnostics surfaces expose one coherent pass/feature story for shipped viewer/runtime passes.
- Depends on: Phase 156
- Status: `complete`

Plans:

- [x] expose/assert pass-slot and feature participation for `Opaque`, `Transparent`, `Overlay`, `Picking`, and `Screenshot`
- [x] add object count, transparent count, and fallback/status diagnostics only where the current runtime already owns that data
- [x] keep the contract backend-neutral and avoid public backend-specific APIs

### Phase 158: Render Evidence and Regression Guards

- Goal: validate the render-pipeline diagnostics contract with focused tests or smoke evidence across the existing viewer/runtime path.
- Depends on: Phase 157
- Status: `complete`

Plans:

- [x] prove transparent, picking, screenshot, and overlay diagnostics through narrow existing test/smoke seams
- [x] ensure software/native evidence remains truthful without adding fallback or compatibility paths
- [x] keep tests focused on regression detection instead of becoming broad visual-rendering coverage

### Phase 159: Render Diagnostics Truth and Guardrails

- Goal: align docs, support wording, demo/sample text, and repository guardrails with the render-pipeline diagnostics contract and explicit non-goals.
- Depends on: Phase 157 and Phase 158
- Status: `complete`

Plans:

- [x] update docs/support/sample wording so diagnostics users understand the shipped pass/feature evidence contract
- [x] keep advanced renderer breadth explicitly deferred
- [x] add or update repository guardrails so the milestone boundary remains enforceable
