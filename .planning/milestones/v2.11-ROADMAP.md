# Roadmap

## Active Milestone

`v2.11 Static glTF/PBR Renderer Consumption`

Goal: make already-imported static glTF/PBR metadata visible on the shipped renderer path so runtime truth and on-screen truth close the gap without widening product scope.

## Scope Guard

- No animation, skeleton, morph, lighting/shadows/post-processing breadth, advanced transparency-system expansion, extra UI adapters, new chart families, broader importer coverage, or backend/API breadth.
- No compatibility shims, migration adapters, downgrade paths, or transitional contracts.
- No public package publishing or repository release tag creation.
- `.planning` remains local-only and does not define a repository release boundary by itself.
