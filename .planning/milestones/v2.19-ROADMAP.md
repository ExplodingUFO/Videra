# Roadmap

## Target Milestone

`v2.19 Runtime Truth Hardening and Product Observability`

Goal: harden the primitive-first runtime path after `v2.18` transparent ordering closure by expanding benchmark hard gates, tightening SurfaceCharts efficiency, and completing runtime-truth guardrails before any broader product expansion.

## Scope Guard

- No new renderer features, transparency modes, OIT, refraction/transmission, shadows, environment maps, post-processing, or fullscreen pass framework work.
- No animation, skeletons, morph targets, mixers, new chart families, chart-kernel widening, UI-shell expansion, backend/platform expansion, or importer-format widening.
- No compatibility shims, migration adapters, downgrade paths, or transitional dual-path contracts.
- Benchmark gate expansion is limited to observability/regression prevention, not performance optimization for its own sake.
- SurfaceCharts work stays within the existing surface family and `SurfaceChartView` path.
- Desktop proof validation remains limited to repository-owned viewer proof hosts with the explicit `10`-second survival rule.
- `.planning` remains local-only and does not define a repository release boundary by itself.

## Phase Plan

### Phase 208: Benchmark Hard-Gate Expansion

**Status:** draft
**Depends on:** `v2.18 Mixed Transparent Primitive Ordering Baseline`
**Requirement:** `RTH-01`

Expand the existing benchmark regression-guard pipeline from 4 thresholds to cover allocation, scene upload drain, rehydrate, inspection snapshot, and chart residency churn.

Success criteria:
- Each new gate has a measured baseline, a hard threshold, and a CI integration.
- Gate names are stable; only thresholds may tighten in future milestones.
- No existing gate is widened or removed.

### Phase 209: SurfaceCharts Residency and Probe Efficiency

**Status:** draft
**Depends on:** Phase 208
**Requirement:** `RTH-02`

Tighten tile residency churn and probe latency on the existing `SurfaceChartView` path without widening into a generic analytics kernel or new chart family.

Success criteria:
- Residency churn under camera movement improves versus the `v2.12` baseline.
- Probe latency and overlay invalidation/allocation are reduced with benchmark evidence.
- The work stays within the existing surface family and Avalonia chart line.

### Phase 210: Runtime-Truth Guardrail Closure

**Status:** draft
**Depends on:** Phase 209
**Requirement:** `RTH-03`

Close the gap between documented primitive-first runtime truth and repository guardrails so the canonical hot path cannot silently regress to `Object3D`-centric assumptions.

Success criteria:
- `SceneDocumentEntry`, `SceneDeltaPlanner`, and `SceneUploadQueue` carry explicit primitive-first contracts in code and docs.
- No new code can re-introduce object-level upload/delta assumptions without an explicit override.
- The guardrails are enforceable via existing CI quality gates.

### Phase 211: Docs and Support Truth Synchronization

**Status:** draft
**Depends on:** Phase 210
**Requirement:** `RTH-04`

Align all public docs, support matrices, and localized mirrors with the post-`v2.18` runtime boundary and its explicit deferred items.

Success criteria:
- README, capability matrix, support matrix, and Chinese mirrors describe the primitive-first runtime truth honestly.
- Animation, skeletal/morph, second UI adapter, Wayland native embedding, OpenGL/WebGL backend, new chart families, lighting/shadows/post-processing remain explicitly deferred.
- The milestone is ready for audit and closeout after implementation merges.

## Predecessor

- `v2.18 Mixed Transparent Primitive Ordering Baseline` — active; phases `204-207`; requirements `MTP-01` through `MTP-04`

## Current Baseline

- `v2.10-v2.18` established and progressively hardened one primitive-centric runtime seam on the native static-scene viewer/runtime path.
- The next roadmap-aligned gap is observability and guardrail closure (benchmark expansion, SurfaceCharts efficiency, runtime-truth enforcement), not new renderer features or product expansion.
