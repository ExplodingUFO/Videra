# Phase 23-02: Render Inputs and Runtime Migration

## Objective

Make the render-host path consume `SurfaceViewState` and `SurfaceCameraFrame` as its primary truth without breaking the current GPU compatibility seam.

## Tasks

1. Add failing tests that assert render inputs and render-state projection dirtiness follow camera-frame changes.
2. Update `SurfaceChartRenderInputs` so `ViewState` and `CameraFrame` are primary inputs, with compatibility accessors kept for older callers.
3. Update runtime/rendering code so `SurfaceChartView.Rendering.cs` produces one shared camera frame per render sync.
4. Update render-state dirty checks to follow the new contracts instead of the old split fields.

## Acceptance Criteria

- Render-host and incremental-rendering tests fail first, then pass.
- `SurfaceChartView` no longer constructs render inputs from `_runtime.CurrentViewport` plus `_runtime.ProjectionSettings`.
- Projection-only renders still avoid resident rebuilds when geometry/color inputs are unchanged.
