# Phase 23-01: Core Camera Frame and Projection Math

## Objective

Add the shared core contracts that make `SurfaceViewState` renderable without passing through the old viewport/projection shell.

## Tasks

1. Add failing core tests for:
   - project/unproject round-trip
   - default camera frame containing the active plot bounds
   - camera-frame projection settings remaining compatible with current yaw/pitch semantics
2. Add `SurfaceCameraFrame` and shared plot-bounds support under `Videra.SurfaceCharts.Core/Rendering`.
3. Add `SurfaceProjectionMath` for:
   - plot-bounds creation from metadata + data window
   - camera-frame creation from metadata + view-state + output size
   - project / unproject helpers
4. Extend `SurfaceCameraPose` with an explicit camera-frame creation path while keeping `ToProjectionSettings()` as a compatibility adapter.

## Acceptance Criteria

- Core tests fail first, then pass.
- `SurfaceCameraFrame` and `SurfaceProjectionMath` exist as reusable core contracts.
- The default frame for `SurfaceViewState.CreateDefault(...)` keeps all plot-box corners within the viewport.
