# Milestone v1.3: Camera-True Surface Charts

**Status:** COMPLETE 2026-04-16  
**Target phases:** 23-28  
**Committed PR spine:** 6  
**Optional follow-through:** Processing/native hotspot work remains deferred until post-PR5 benchmark evidence

## Overview

`v1.3` 的目标不是继续在现有 `Viewport + ProjectionSettings` shell 上堆功能，而是把 surface-chart 的内部主线统一成真正 camera-true 的专业图表管线。当前仓库已经有 `SurfaceViewState` / `SurfaceCameraPose`、built-in orbit/pan/dolly/focus、hover/pinned probe、`Interactive` / `Refine`、independent demo、view-aware scheduler 和 `GPU-first` renderer；下一轮要做的是把这些能力接到同一套相机、投影、拾取和 request-planning 数学上，然后再收紧 GPU resident path 与 overlay 专业功能。

这个里程碑继续保持 surface-chart 与 `VideraView` 的 sibling boundary。所有修改都限制在 `Videra.SurfaceCharts.*` family，避免把 chart-specific semantics 反推回 viewer 主线。`Processing/native` follow-through 只作为 benchmark-gated future scope 记录，不在 committed roadmap 里伪装成已承诺 phase。

## Phase 23: Unified Camera, Projection, and Render Inputs

**Status:** COMPLETE 2026-04-16  
**Goal:** 把内部主线从 `Viewport + ProjectionSettings` 驱动升级为 `SurfaceViewState + SurfaceCameraFrame` 驱动，让 renderer、overlay、picking 和后续 LOD 使用同一套投影数学。  
**Depends on:** Phases 19-22 shipped baseline  
**Requirements:** `CAM-01`, `CAM-02`  
**Plans:** 3 completed (`23-01` / `23-02` / `23-03`)

**Expected plan split:**
- `23-01` Core camera frame and shared projection math contracts
- `23-02` Render-input/runtime migration to `SurfaceViewState + SurfaceCameraFrame`
- `23-03` Overlay projection migration and round-trip regression coverage

**Success criteria:**
1. `SurfaceChartRenderInputs` 主吃 `SurfaceViewState` / `SurfaceCameraFrame`，renderer 与 overlay 不再依赖分裂的 projection truth。
2. 默认视角、orbit / pan / wheel 视觉行为不明显回归。
3. Core/integration tests 锁住 project/unproject round-trip、default fit framing 和多角度轴投影稳定性。

## Milestone Summary

**Key decisions:**
- 按 `真相机 -> 真投影 -> 真拾取 -> 真 LOD` 的顺序先贯通正确性 spine，再做 resident slimming、shader color mapping 和 overlay 专业化。
- 所有改动继续限制在 `Videra.SurfaceCharts.*` family，不把 chart semantics 回灌到 `VideraView`。
- `Processing/native` follow-through 继续保持 measurement-gated deferred scope，不进入 `v1.3` committed spine。

**Issues being resolved:**
- Render inputs 仍围绕 `Viewport + ProjectionSettings`，与公开 `SurfaceViewState` / `SurfaceCameraPose` 形成双系统。
- Probe 仍主要依赖屏幕到 viewport 的线性映射，而不是 camera-aware 3D picking。
- LOD/request planning 仍偏 viewport density 驱动，相机变化对 detail selection 的影响不够真实。
- GPU path 仍保留 software-scene 影子，并在 resident / upload 路径上承担不必要的 CPU 成本。
- Color-map 切换与 overlay layout 还没有达到专业图表的性能和可读性基线。

**Issues deferred:**
- Processing/native hotspot follow-through before post-PR5 benchmark evidence
- Contours / slices / multi-view analysis layers
- Streaming sources and richer data-scale features beyond offline cache-backed datasets
- Viewer-side rendering feature expansion and platform-scope milestones such as compositor-native Wayland embedding
