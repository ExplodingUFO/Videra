## v2 Requirements

Requirements committed for the `v2.0` milestone. Each requirement maps to one roadmap phase.

### Transparency Slice Scope

- [x] **TRAN-01**: Before implementation starts, the milestone locks one concrete transparency slice and freezes explicit non-goals without widening into broader lighting or material-system work.

### Transparent Contract and Diagnostics

- [x] **TRAN-02**: The transparent feature contract and diagnostics truth are explicit on the existing runtime path.

### Alpha Mask Baseline

- [x] **TRAN-03**: Alpha mask rendering works as a stable baseline without widening into generic material abstractions or OIT.

### Alpha Blend Ordering

- [x] **TRAN-04**: Alpha blend ordering is deterministic and matches the documented runtime truth.

### Product Truth and Guardrails

- [x] **TRAN-05**: Docs, support guidance, release/package truth, and repository guards all describe the same transparency story.

## Out of Scope

Explicitly excluded from `v2.0` to keep the milestone focused on transparency depth.

| Feature | Reason |
|---------|--------|
| Lighting-system expansion | This milestone is about transparency truth, not lighting breadth |
| Shadows | This milestone does not widen into shadow integration |
| Environment maps | This milestone stays on the existing runtime path and does not widen the shading model |
| Post-processing | This milestone does not open a general effect pipeline |
| Animation | This milestone is not moving into animation or timeline features |
| Generic material abstractions | This milestone keeps material truth concrete and slice-specific |
| New UI adapters | This milestone does not reopen UI-shell scope |
| OIT / order-independent transparency | This milestone keeps ordering truth explicit instead of adding a broad OIT system |
