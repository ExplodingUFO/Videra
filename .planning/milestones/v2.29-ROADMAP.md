# Roadmap

## Target Milestone

`v2.29 CI/PR Reality for Streaming Performance`

**Goal:** Take the completed v2.28 streaming-performance branch through remote PR, GitHub CI, review, merge, and cleanup reality while preserving the narrow performance-hardening scope.

**Scope lock:** This milestone is process and integration closure for the existing v2.28 code. It does not add new chart families, renderer breadth, public package publication, release tags, or new performance features unless a CI failure proves the existing implementation or CI contract is wrong.

### Phase 252: PR Branch and Scope Truth

**Status:** completed  
**Depends on:** `v2.29 milestone start`  
**Requirement:** `CPR-01`, `CPR-02`

**Goal:** Establish the remote branch and PR boundary for the existing v2.28 streaming-performance commits.

**Success criteria:**
1. The local feature branch is clean before push.
2. The remote branch contains only the intended v2.28 streaming-performance commits relative to `master`.
3. A GitHub PR exists with a scope summary, verification evidence, and explicit non-goals.
4. Any unexpected diff or branch divergence is classified before CI closure starts.

### Phase 253: GitHub CI Reality and Failure Closure

**Status:** completed  
**Depends on:** Phase 252  
**Requirement:** `CPR-03`, `CPR-04`

**Goal:** Observe required GitHub CI checks for the PR and close failures with the smallest justified code or CI-contract changes.

**Success criteria:**
1. Required PR checks are listed with conclusion, URL, and failure reason where applicable.
2. CI failures are classified as product-code issue, test/contract issue, environment issue, or flaky/noisy threshold.
3. Fixes stay scoped to the failure and preserve the v2.28 performance-hardening intent.
4. The PR reaches a mergeable state or a concrete blocker is documented.

### Phase 254: PR Merge and Branch Cleanup

**Status:** completed  
**Depends on:** Phase 253  
**Requirement:** `CPR-05`

**Goal:** Merge the PR after required checks pass and clean up local/remote branch state.

**Success criteria:**
1. The PR is merged to `master` only after required checks are green or an explicitly documented repository policy allows the merge.
2. Local `master` is synchronized with `origin/master`.
3. The merged feature branch is removed locally and remotely when safe.
4. The local worktree is clean after cleanup.

### Phase 255: Evidence and Next-Step Closure

**Status:** completed  
**Depends on:** Phase 254  
**Requirement:** `CPR-06`

**Goal:** Record the merge evidence, residuals, and next milestone recommendation without adding product breadth.

**Success criteria:**
1. `PROJECT.md`, `STATE.md`, and `MILESTONES.md` reflect the v2.29 PR/CI/merge result.
2. Any CI waivers, threshold changes, or residual risks are explicitly recorded.
3. The next recommended milestone is stated from the post-merge repo state.
4. The workspace is ready for `$gsd-complete-milestone v2.29` or a blocker report.
