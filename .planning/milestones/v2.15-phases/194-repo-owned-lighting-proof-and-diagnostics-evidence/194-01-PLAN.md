# Plan 194-01: Repo-Owned Lighting Proof and Diagnostics Evidence

## Success Criteria

1. One repository-owned viewer proof path demonstrates the direct-lighting baseline on a supported host path.
2. The proof apps used in validation can stay alive for `10` seconds without crashing when proof hold mode is enabled.
3. Existing smoke defaults stay fast when proof hold mode is not requested.
4. Focused repository tests pin the proof-only contract and diagnostics knobs.

## Assumptions

- `smoke/Videra.ConsumerSmoke` and `smoke/Videra.WpfSmoke` are the narrowest viewer proof apps for this phase.
- An opt-in environment-variable hold is smaller and cleaner than changing app defaults or widening to more sample apps.
- Docs truth/guardrails stay in `Phase 195`; this phase only adds proof-path code and narrow tests.

## Steps

1. Add one opt-in proof-hold parameter to the existing smoke scripts.
2. Wire the same hold knob into the two smoke apps so successful runs can stay alive for `10` seconds before shutdown.
3. Add narrow source-text tests that pin the proof-hold contract.
4. Run real proof commands with `10`-second hold enabled and capture artifact evidence.
