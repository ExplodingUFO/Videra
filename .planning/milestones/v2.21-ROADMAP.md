# Roadmap

## Target Milestone

`v2.21 Repo Doctor and Quality Gate Closure`

Goal: improve release and support confidence by adding a repo-only `Videra Doctor` validation surface, tightening quality-gate contract truth, and adding focused allocation gates for the importer/demo diagnostics paths delivered in `v2.20`, without widening product architecture.

## Scope Guard

- No public `Videra.Doctor` package, global tool, or supported public API.
- No auto-remediation or configuration mutation; Doctor reports, routes, and writes artifacts only.
- No broad Core / Import / Backend / Avalonia / SurfaceCharts architecture rewrite.
- No backend API redesign or compatibility migration for advanced seams.
- No renderer feature breadth, animation, skeletal/morph support, lighting/shadows/post-processing, WebGL/OpenGL, new UI adapter, or new backend/platform family.
- No public release publication, release-tag push, package feed mutation, or publishing workflow execution.
- Allocation gates are limited to stable, focused evidence for existing importer/demo diagnostics paths; noisy measurements remain evidence-only.
- `.planning` remains local-only and does not define a repository release boundary by itself.
