# Roadmap

## Target Milestone

`v2.24 Analyzer 10 and Dependency Hygiene`

Goal: upgrade maintenance quality deliberately by defining and applying the analyzer 10 rule policy, tightening dependency-update hygiene, and proving the resulting quality gates without broad architecture rewrites, compatibility layers, release publishing, or product feature breadth.

## Scope Guard

- No broad Core / Import / Backend / Avalonia / SurfaceCharts architecture rewrite.
- No public API churn solely to satisfy low-value analyzer style recommendations.
- No compatibility shim, downgrade path, or fallback layer unless a phase proves a direct fix is impossible and documents why.
- No central package management migration in this milestone.
- No release tag creation, public package publication, or package feed mutation.
- No renderer feature breadth, animation, skeletal/morph support, lighting/shadows/post-processing, WebGL/OpenGL, new UI adapter, or new backend/platform family.
- No new chart family or merging SurfaceCharts semantics into `VideraView`.
- `.planning` remains local-only and does not define a repository release boundary by itself.
