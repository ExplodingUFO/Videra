# Requirements: Videra v1.2 Professional Surface Charts

**Defined:** 2026-04-14  
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v1.2 Requirements

### Runtime & API

- [x] **VIEW-01**: Host application can control a `SurfaceChartView` through a persisted `SurfaceViewState` that separates data window from 3D camera pose.
- [x] **VIEW-02**: Host application can call `FitToData()`, `ResetCamera()`, and `ZoomTo(...)` without owning tile-cache or camera-controller internals.
- [x] **VIEW-03**: Surface-chart orchestration moves into an independent runtime layer without requiring new chart-specific APIs on `VideraView`.

### Interaction

- [x] **INT-01**: End user can orbit the surface around the active focus point through built-in pointer interaction.
- [x] **INT-02**: End user can pan and dolly zoom through built-in interaction, including a predictable reset/preset workflow.
- [x] **INT-03**: End user can focus a selected data region with built-in zoom behavior rather than host-only viewport presets.
- [x] **INT-04**: The chart automatically switches between interactive quality and refine quality so motion stays responsive while detail recovers after input stops.

### Axes & Readout

- [x] **AXIS-01**: End user sees X/Y/Z axes with titles, units, adaptive major/minor ticks, and camera-aware edge placement.
- [x] **AXIS-02**: End user sees a color legend that stays consistent with the active color map, value range, clipping, and value transform.
- [x] **PROBE-01**: End user can hover the chart and see X/Y/Z readout plus approximate-versus-exact state without host-specific overlay code.
- [x] **PROBE-02**: End user can pin or highlight selected probe points for persistent comparison.

### Rendering

- [x] **REND-01**: Surface charts render through a GPU-first renderer that keeps a software fallback path for unsupported or test environments.
- [x] **REND-02**: The main renderer keeps incremental scene and residency state instead of rebuilding the full scene snapshot on every viewport/input change.
- [x] **REND-03**: Tile selection and refinement use view-aware prioritization so interactive navigation remains responsive on large datasets.

### Data & Processing

- [x] **DATA-01**: Tile scheduling supports cancellation, prioritization, and bounded parallel requests instead of sequential detail-tile loading.
- [x] **DATA-02**: Cache-backed datasets support persistent readers or batch tile reads so repeated interaction does not reopen and recopy payload data for each tile.
- [x] **DATA-03**: Pyramid/cache processing supports richer reducers and tile statistics than simple averaging so peaks, legends, and probe values remain truthful.
- [x] **DATA-04**: Lower-level hotspots expose coarse integration seams suitable for optional Rust acceleration without moving UI or runtime orchestration out of C#.

### Demo & Validation

- [x] **DEMO-01**: The independent surface-chart demo and repository docs/tests show the professional chart behavior while preserving the explicit sibling boundary from `VideraView`.

## v1.3+ / Future Requirements

### Advanced Analysis

- **ANLY-01**: End user can overlay contours, slices, or derived analysis layers on top of the surface.
- **ANLY-02**: End user can compare multiple linked views or synchronized markers across related datasets.

### Data Scale Extensions

- **SCALE-01**: Surface charts support streaming or progressive sources beyond static offline cache files.
- **SCALE-02**: Surface charts support richer compression/quantization options and GPU-side picking refinements after the base runtime and renderer stabilize.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Folding surface-chart semantics back into `VideraView` | Violates the sibling-boundary contract already locked by docs, demo, and repository tests |
| Viewer-side PBR / shadow / shader-graph expansion | This milestone is chart-focused, not a generic rendering-feature expansion |
| Full editor-style scene tooling | Not part of the engine's current product direction |
| Broad Rust rewrite across UI or runtime layers | Rust is only being considered for narrow data/processing hotspots after C# seams are stable |
| compositor-native Wayland embedding | Separate platform milestone, not part of the chart roadmap |
