# Plan 197-01: Broader Style-Lighting Contract

## Goal

Ship one bounded broader-lighting baseline on the current native static-scene path by extending the existing style/uniform shader seam with a single fill/wrap scalar only.

## Success Criteria

1. The broader-lighting work stays entirely inside the existing style/uniform seam and native shader implementations.
2. Windows, Linux, and macOS native backends remain aligned on the same bounded lighting contract.
3. The phase does not introduce shadow maps, environment sampling, fullscreen post-processing, emissive renderer consumption, or normal-map renderer consumption.
4. Focused tests cover the new contract and the repository truth remains honest about what is still deferred.

## Assumptions

- The current direct-lighting baseline is the right substrate for this follow-on slice.
- A single fill/wrap scalar is smaller and safer than a second directional light or hemisphere model.
- Software fallback does not need to expand in this phase if docs and proof surfaces keep that boundary honest.

## Proposed Contract

- `LightingParameters.FillIntensity : float`
- `StyleUniformData.FillIntensity : float` at `[FieldOffset(28)]`
- shader logic uses the existing `lightDirection` and adds one bounded fill/wrap contribution to the current diffuse term
- default value remains `0f`, so the existing direct-lighting baseline stays unchanged until styles opt in
