# Plan 198-01: Repo-Owned Broader-Lighting Proof and Validation Wiring

## Goal

Tighten the repository-owned broader-lighting proof surface by wiring explicit `10`-second survival holds through the validation entrypoints that actually launch viewer desktop hosts.

## Success Criteria

1. Every desktop program used in the phase validation is invoked with an explicit `LightingProofHoldSeconds=10` path.
2. The proof scope stays on the existing viewer hosts and does not widen to new apps or SurfaceCharts paths.
3. Repository tests lock the new workflow/wrapper truth.
4. Manual proof runs on the selected desktop hosts can still complete without crashes.

## Assumptions

- `ConsumerSmoke` viewer mode plus `WpfSmoke` are the only viewer proof hosts this phase needs.
- `SurfaceCharts.ConsumerSmoke` can stay out of scope because it does not yet carry the hold seam and is not needed to prove the broader-lighting baseline.
- Wiring the existing hold parameter is enough; no additional app lifecycle code should be needed in this phase.
