# Phase 201 Plan

## Goal

Consume emissive and normal-map-ready material truth on the existing static-scene renderer path by extending the current CPU-side bake seam, not by introducing a new material or resource-binding system.

## Tasks

1. Extend static-scene material baking
   - Reuse the current CPU texture-sampling seam.
   - Add emissive contribution to retained vertex color inputs.
   - Add normal-texture contribution to retained vertex normal inputs.

2. Keep both renderer entry paths aligned
   - Update `ImportedSceneAssetPayloadBuilder` so `asset.Payload` consumes the same material truth.
   - Update `ImportedSceneRuntimeObjectBuilder` so deferred runtime-object creation keeps parity.

3. Add focused tests
   - Cover payload/runtime-object consumption with targeted scene/import/render tests.
   - Keep repository docs/guardrail wording out of this phase.

## Verification

- Focused `Videra.Core.Tests` scene/import/graphics filters for emissive + normal-map consumption
- Focused integration tests on static-scene rendering paths if touched
- `git diff --check`

## Non-Goals

- Generic material-system or lighting-system expansion
- Docs/support/repository guardrail updates (reserved for Phase 203)
