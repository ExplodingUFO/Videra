# Phase 218 Plan

## Goal

Add a minimal repo-only Doctor command that writes human-readable and structured support snapshots without becoming a product package or a god-script.

## Steps

1. Add repository tests that require Doctor to remain repo-only, non-mutating, non-public, and artifact-producing.
2. Add `scripts/Invoke-VideraDoctor.ps1`.
3. Report SDK/runtime, OS, git state, contract file presence, validation script presence, platform project presence, and known support artifact paths.
4. Write `doctor-summary.txt` and `doctor-report.json` under `artifacts/doctor` by default.
5. Add minimal docs and index linkage that explain the support boundary.
6. Verify with targeted tests, a manual default Doctor run, diff check, and solution build.

## Success Criteria

- Doctor runs from the repository without package installation.
- Doctor writes human-readable plus JSON reports under `artifacts/doctor`.
- Code, docs, and tests explicitly keep Doctor repo-only and non-mutating.
