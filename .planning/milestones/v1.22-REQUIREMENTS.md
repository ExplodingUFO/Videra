# Requirements: Videra v1.22 Static glTF/PBR

**Defined:** 2026-04-21
**Core Value:** 跨平台 3D 渲染引擎的可靠性

## v1 Requirements

Requirements committed for the `v1.22` milestone. Each requirement maps to one roadmap phase.

### Static glTF Asset and Texture Truth

- [x] **GLTF-01**: Maintainer can import static glTF UV coordinates and texture references into explicit `Texture2D` / `Sampler` runtime assets without leaking importer-shaped or backend-specific public types.
- [x] **GLTF-02**: Static glTF texture usage carries one explicit color-space contract so color textures and data textures bind with correct sRGB/linear intent on the shipped viewer path.

### Static PBR Material Semantics

- [x] **PBR-01**: Viewer-path materials can represent metallic-roughness and alpha semantics for static glTF scenes through explicit runtime contracts.
- [x] **PBR-02**: Viewer-path materials can represent emissive and normal-map-ready inputs for static glTF scenes without widening into a broader advanced-runtime feature surface.

### Runtime Reuse and Product Truth

- [x] **RUNTIME-01**: Repeated static glTF scenes reuse material/texture upload state predictably instead of rebuilding importer-shaped GPU payloads ad hoc.
- [x] **RUNTIME-02**: Tangent-aware shading inputs flow through the static glTF importer/runtime path as explicit runtime data rather than importer-only side channels.
- [x] **TRUTH-02**: Docs, samples, and repository tests describe and validate one clear static glTF/PBR baseline and its deliberate exclusions consistently.

## Future Requirements

Deferred after `v1.22`. These remain visible but are not part of the current roadmap.

### Inspection and Productization

- **INSP-01**: Inspection workflows compose directly on top of the scene/material runtime and static glTF/PBR baseline without special-case scene bridges.
- **PERF-01**: CI fails on agreed performance regressions and package-size budgets once the static asset/material path stabilizes.
- **UIAD-01**: A second UI adapter proves the host seams outside Avalonia after the viewer path is stable.

### Advanced Runtime Features

- **ADV-01**: Lighting expansion, shadows, environment maps, transparent-sorting overhauls, post-processing, and animation remain an opt-in `2.0` line rather than a `1.x` runtime-core requirement.

## Out of Scope

Explicitly excluded from `v1.22` to keep the milestone focused on a credible static glTF/PBR baseline.

| Feature | Reason |
|---------|--------|
| Animation, skeletons, morph targets, or mixers | `v1.22` is static-scene-only on purpose |
| New backend work such as `OpenGL` / `WebGL` | Backend expansion still ranks below deepening the native viewer/runtime product |
| A second UI adapter | Avalonia remains the only validated UI shell in this milestone |
| Lighting-system expansion beyond the shipped static PBR baseline | Advanced runtime features remain a later milestone instead of inflating the first static PBR line |
| Generic `Chart3D` extraction or new chart product lines | `v1.22` is about static glTF/PBR on the viewer/runtime path, not widening the chart surface |
