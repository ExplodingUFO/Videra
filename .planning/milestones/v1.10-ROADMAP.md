# Milestone v1.10: Alpha Consumer Integration and Feedback Loop

**Status:** COMPLETE 2026-04-17
**Target phases:** 64-67
**Committed workstreams:** 4
**Research:** skipped — the milestone strategy comes directly from the current repo state and the supplied alpha-consumer prioritization

## Overview

`v1.10` 不再继续沿着 scene/runtime 内部性能线往下钻。它把下一轮重点切到公开 alpha 组件的消费面：先把 `VideraView` 的默认 happy path 收短，把 consumer package install 到 first scene 的公共路径做成 smoke，随后把 benchmark evidence 变成 workflow gate，最后把 alpha feedback / support / supported-path truth 对齐。目标不是新增更多底层能力，而是尽快降低接入门槛、拿到真实外部反馈，并把当前 alpha 边界说清楚。

这轮里程碑有一个明确边界：不做 native Wayland compositor-hosted 嵌入、不继续大规模内核改写、不把 surface charts 变成公开 package，也不把 demo 扩成大而全 showcase。先把公开 alpha 的接入、验证、反馈闭环打通。

## Phase 64: Happy-Path Viewer API and Minimal Sample Simplification

**Status:** COMPLETE  
**Goal:** Make the default public `VideraView` story short and unambiguous so new alpha consumers reach first scene without first learning extensibility internals.  
**Depends on:** v1.9 complete  
**Requirements:** `API-04`, `API-05`, `API-06`  
**Plans:** 3 complete

**Success criteria:**
1. Root README, Avalonia README, and the default minimal sample all use the same public `Options -> LoadModel(s) -> FrameAll/ResetCamera -> BackendDiagnostics` flow.
2. `Engine`, frame hooks, and pass contributors are removed from default examples and moved behind explicit extensibility documentation.
3. The shortest supported first-scene path is visible in one place and does not require engine-internal concepts.

## Phase 65: Consumer Package Smoke and Host Validation

**Status:** COMPLETE  
**Goal:** Validate the public package-consumer flow from install to first scene instead of relying only on repo-local builds and integration tests.  
**Depends on:** Phase 64  
**Requirements:** `CONS-01`, `CONS-02`, `CONS-03`  
**Plans:** 3 complete

**Success criteria:**
1. A consumer-grade smoke app/test uses `nuget.org` packages for `Videra.Avalonia` plus a platform package and validates initialize → load → frame → diagnostics.
2. Workflow entrypoints exist so package-consumer regressions are rerunnable and visible.
3. Linux/X11/XWayland support boundaries remain explicit in both smoke assets and public docs.

## Phase 66: Benchmark Workflows and Regression Gates

**Status:** COMPLETE  
**Goal:** Turn viewer/surface-charts benchmark evidence into a repeatable workflow gate instead of leaving it as a one-off local capability.  
**Depends on:** Phases 64-65  
**Requirements:** `PERF-05`, `PERF-06`, `PERF-07`  
**Plans:** 3 complete

**Success criteria:**
1. Viewer and surface-charts benchmarks have explicit workflow entrypoints.
2. Manual or labeled benchmark runs publish artifacts/results that can be compared across runs.
3. Benchmark docs clearly explain which signals are informative and which are intended to block regressions.

## Phase 67: Alpha Feedback and Support Surfaces

**Status:** COMPLETE  
**Goal:** Make the alpha support boundary, reproduction expectations, and feedback loop obvious so external adopters can report useful problems quickly.  
**Depends on:** Phases 64-66  
**Requirements:** `FB-01`, `DOC-04`, `DOC-05`

## Milestone Summary

**Key decisions:**
- Default consumer path now takes priority over another internal optimization cycle.
- Public examples should optimize time-to-first-scene and defer extensibility to separate docs.
- Consumer smoke must validate the current public package surface through packed consumer packages, not repo-local project references or stale public alpha packages.
- Benchmark evidence should become a workflow-visible gate before the next round of deeper performance work.
- Native Wayland embedding stays deferred unless alpha consumer feedback proves it is the highest-value blocker.
- Alpha feedback surfaces should require diagnostics-rich reproduction data and the same canonical minimal-sample / consumer-smoke story used elsewhere in the milestone.

**Issues deferred:**
- Native compositor-hosted Wayland embedding
- Publishing `Videra.SurfaceCharts.*` as public consumer packages
- Another deep internal scene/runtime rewrite before adoption evidence justifies it
