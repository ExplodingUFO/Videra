# Roadmap

## Target Milestone

`v2.22 Alpha Release Readiness Dry Run`

Goal: prove the alpha candidate can be validated from clean package artifacts through Doctor, release dry-run, consumer smoke, benchmark/native evidence, and support docs without publishing, tagging, or widening product architecture.

## Scope Guard

- No public package publishing, feed mutation, or GitHub Packages/nuget.org upload.
- No release tag creation or remote push.
- No public `Videra.Doctor` package, global tool, or supported public Doctor API.
- No auto-remediation or machine/user configuration mutation.
- No broad Core / Import / Backend / Avalonia / SurfaceCharts architecture rewrite.
- No renderer feature breadth, animation, skeletal/morph support, lighting/shadows/post-processing, WebGL/OpenGL, new UI adapter, or new backend/platform family.
- No compatibility shim, downgrade path, or fallback layer unless a phase proves it is a release blocker and documents why a direct fix is impossible.
- Fixes in the final phase are blocker-only and must be directly tied to dry-run evidence.
- `.planning` remains local-only and does not define a repository release boundary by itself.
