# Requirements

Active milestone: `v2.16 Bounded Broader Lighting Baseline`

Broader-lighting requirement numbering starts at `BLI-01`.

## Milestone Goal

Broaden the shipped static-scene lighting baseline through the existing style/uniform seam, without widening into shadows, environment maps, post-processing, emissive/normal-map renderer consumption, animation, new UI/package lines, or broader runtime/package expansion.

## Requirements

- [x] `BLI-01` Bounded broader-lighting scope lock
  - Freeze `v2.16` to one bounded broader-lighting slice on the existing viewer/runtime line.
  - Keep shadows, environment maps, post-processing, emissive/normal-map renderer consumption, animation, new UI/package lines, backend/platform expansion, broader importer/chart widening, compatibility shims, and migration adapters out of scope.
  - Keep public package publishing and repository tag creation out of scope.

- [x] `BLI-02` Broader style-lighting contract
  - Add only the minimum style/uniform and native-shader seam needed for one honest broader-lighting baseline on the shipped static-scene path.
  - Avoid generic scene-lighting frameworks, shadow-map infrastructure, environment sampling abstractions, fullscreen post-processing seams, or speculative advanced-runtime abstractions.
  - Prove the new contract with focused code evidence.

- [x] `BLI-03` Repo-owned broader-lighting proof and validation wiring
  - Strengthen one repository-owned viewer proof path so it demonstrates the broader-lighting baseline on a supported host path.
  - Ensure every desktop program used in the milestone validation survives `10` seconds without crashing.
  - Keep diagnostics/support artifacts coherent with the new lighting baseline and keep the proof repository-owned and non-public.

- [x] `BLI-04` Broader-lighting truth and guardrails
  - Align docs, support wording, and repository guardrails with the shipped broader-lighting baseline.
  - Keep shadows, environment maps, post-processing, emissive/normal-map renderer consumption, animation, and broader advanced-runtime breadth explicitly deferred.
  - Leave package-split and additional advanced-runtime breadth decisions to later milestones.

## Out of Scope

| Item | Excluded from `v2.16` |
|---|---|
| Advanced renderer breadth | No shadows, shadow maps, environment maps, post-processing, fullscreen pass framework, emissive renderer consumption, or normal-map renderer consumption. |
| Product breadth | No animation, skeletons, morph targets, mixers, new chart families, chart-kernel widening, UI-shell expansion, backend/platform expansion, or importer-format widening. |
| Public surface changes | No new public runtime package line, no resource-set/render-target abstraction widening, and no broader lighting-system public model. |
| Compatibility | No compatibility shim, migration adapter, downgrade path, or transitional dual-lighting public contract. |
