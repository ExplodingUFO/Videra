# Milestone v1.12: Viewer-First Inspection Workflow

**Status:** COMPLETE 2026-04-18
**Target phases:** 72-75
**Committed workstreams:** 4
**Research:** skipped — the milestone strategy comes directly from the current repo state, the approved inspection-workflow direction, and the explicit priority to pressure the viewer through end-user features instead of more internal churn

## Overview

`v1.12` 把下一轮 viewer 工作收成一条很窄的 inspection 主线：section / clipping、measurement / probe、camera state persistence 和 snapshot export。目标不是把 `Videra` 推向 editor、建模工具或更大 product surface，而是用一组真正用户可见、宿主能感知价值、又会压到现有 render/runtime/overlay seams 的 inspection workflow 来证明当前 alpha viewer 不是“只能接入”，而是“已经能干活”。

这轮里程碑有清晰边界：不做 editor-style gizmos，不重开另一轮大架构重构，不深挖 Wayland/native，不把 SurfaceCharts 变成公开包，也不把 benchmark evidence 立刻升级成新一轮性能主线。inspection workflow 本身会给现有边界施压，后续是否继续扩性能或平台能力，再由这轮结果和真实 alpha 反馈决定。

## Phase 72: Section and Clipping Workflow

**Status:** COMPLETE  
**Goal:** Add viewer-first section/clipping state that can be enabled from the public `VideraView` story without dropping users into engine-level pass extensibility.  
**Depends on:** v1.11 complete  
**Requirements:** `CLIP-01`, `CLIP-02`, `CLIP-03`  
**Plans:** 3 complete

**Success criteria:**
1. Host code can define, enable, disable, and clear clipping planes through a typed viewer contract.
2. Rendered geometry, overlays, and exported images all respect the same active clipping state.
3. Backend diagnostics and inspection diagnostics clearly report active clipping truth and fallback/limit information.

## Phase 73: Measurement and Probe Workflow

**Status:** COMPLETE  
**Goal:** Build lightweight viewer-first measurements on top of the existing selection / annotation / overlay seams without drifting into editor semantics.  
**Depends on:** Phase 72  
**Requirements:** `MEAS-01`, `MEAS-02`, `MEAS-03`  
**Plans:** 3 complete

**Success criteria:**

3. Host code can observe and own measurement results through a stable high-level viewer contract.

## Phase 74: Camera State Persistence and Snapshot Export

**Depends on:** Phases 72-73

## Milestone Summary

**Key decisions:**
- Use viewer-first inspection features to pressure the current runtime/render/overlay seams instead of opening another architecture-first milestone.
- Keep the scope on presentation/inspection value; do not drift toward editor or modeling capabilities.
- Land clipping first because it stresses pass contribution, scene/runtime truth, overlays, diagnostics, and export in one bounded feature.
- Treat measurement as a lightweight extension of the existing selection/annotation seams, not as a new editing subsystem.
- Make camera/view-state persistence and snapshot export part of the same inspection flow so the viewer can produce shareable outputs, not just interactive state.

**Delivered outcomes:**
- Added typed clipping planes, measurement state, inspection-state capture/restore, and snapshot export to the public `VideraView` inspection surface.
- Extended diagnostics, consumer smoke, sample UI, and docs so clipping/measurement/export truth can be reproduced from one support workflow.
- Proved the inspection workflow with focused runtime tests, integration tests, consumer smoke, and full repository verification.

**Issues deferred:**
- Editor-style clipping gizmos and authoring workflows
- Native compositor-hosted Wayland work
- Publishing `Videra.SurfaceCharts.*` as public packages
- Another deep performance pass before inspection workflows produce new evidence
