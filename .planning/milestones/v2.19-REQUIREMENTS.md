# Requirements

Target milestone: `v2.19 Runtime Truth Hardening and Product Observability`

Requirement numbering starts at `RTH-01`.

## Milestone Goal

Strengthen the primitive-first runtime path established in `v2.10-v2.18` by expanding observable quality gates, tightening SurfaceCharts residency/probe efficiency, and ensuring the runtime hot-path truth is fully guardrailed before any broader product expansion.

This milestone does **not** introduce new renderer features, new chart families, new UI adapters, new backends, animation, or lighting/shadow/post-processing work.

## Requirements

- [ ] `RTH-01` Benchmark hard-gate expansion
  - Expand the current 4-threshold benchmark gate to cover allocation, scene upload drain, rehydrate, inspection snapshot, and chart residency churn.
  - Each new gate must have a baseline, a hard threshold, and a regression-guard integration in CI.
  - Keep gate names stable once introduced; only thresholds may tighten.

- [ ] `RTH-02` SurfaceCharts residency and probe latency optimization
  - Reduce cache-backed / overview-first surface path tile residency churn below the current `v2.12` baseline.
  - Reduce probe latency and overlay invalidation/allocation on the `SurfaceChartView` interaction path.
  - Prove improvement with before/after benchmark evidence on the existing hard-gate pipeline.

- [ ] `RTH-03` Runtime-truth guardrail closure
  - Align repository guardrails so that no new code can re-introduce `Object3D`-centric assumptions on the canonical hot path without explicit override/escape hatch.
  - Ensure `SceneDocumentEntry`, `SceneDeltaPlanner`, and `SceneUploadQueue` contracts are documented as primitive-first in both code contracts and repository truth.

- [ ] `RTH-04` Docs and support truth synchronization
  - Align README, capability matrix, support matrix, and localized docs with the post-`v2.18` runtime boundary.
  - Explicitly defer animation, skeletal/morph, second UI adapter, Wayland native embedding, OpenGL/WebGL backend, new chart families, lighting/shadows/post-processing.

## Out of Scope

| Item | Excluded from `v2.19` |
|---|---|
| Renderer feature breadth | No new transparency modes, OIT, refraction/transmission, shadows, environment maps, post-processing, or fullscreen pass framework. |
| Runtime abstraction breadth | No generic animation system, skeletal/morph targets, scene-graph widening, or new render-target/resource-set abstraction. |
| Product breadth | No new chart families, chart-kernel widening, UI-shell expansion, backend/platform expansion, or importer-format widening. |
| Compatibility | No compatibility shim, migration adapter, or downgrade path for pre-primitive runtime contracts. |
