Per CONTEXT.md decisions on static analysis:
1. Create Directory.Build.props in repository root (parent of src/ and tests/):
   ```xml
   <Project>
     <PropertyGroup>
       <TargetFramework>net8.0</TargetFramework>
       <ImplicitUsings>enable</ImplicitUsings>
       <Nullable>enable</Nullable>
       <!-- Enable NetAnalyzers and treat warnings as errors in Debug -->
       <EnableNETAnalyzers>true</EnableNETAnalyzers>
       <AnalysisLevel>latest-recommended</AnalysisLevel>
       <TreatWarningsAsErrors Condition="'$(Configuration)' == 'Debug'">true</TreatWarningsAsErrors>
       <RunAnalyzersDuringBuild>true</RunAnalyzersDuringBuild>
     </PropertyGroup>
     <ItemGroup>
       <PackageReference Include="Microsoft.CodeAnalysis.NetAnalyzers" Version="9.0.0" PrivateAssets="all" />
       <PackageReference Include="SonarAnalyzer.CSharp" Version="10.6.0.109712" PrivateAssets="all" />
     </ItemGroup>
   </Project>
   ```
   IMPORTANT: This file applies to ALL projects. Remove <TargetFramework>, <ImplicitUsings>, <Nullable> from individual .csproj files ONLY if they duplicate what's in Directory.Build.props (do NOT modify existing .csproj files now -- that's a later cleanup). The existing individual settings will override Directory.Build.props per MSBuild precedence, so no conflict.

   Actually -- DO NOT set TargetFramework, ImplicitUsings, Nullable in Directory.Build.props since Videra.Demo.csproj has different settings (OutputType=WinExe, AvaloniaUseCompiledBindingsByDefault). Only set analyzer-related properties:
   ```xml
   <Project>
     <PropertyGroup>
       <EnableNETAnalyzers>true</EnableNETAnalyzers>
       <AnalysisLevel>latest-recommended</AnalysisLevel>
       <TreatWarningsAsErrors Condition="'$(Configuration)' == 'Debug'">true</TreatWarningsAsErrors>
       <RunAnalyzersDuringBuild>true</RunAnalyzersDuringBuild>
     </PropertyGroup>
     <ItemGroup>
       <PackageReference Include="Microsoft.CodeAnalysis.NetAnalyzers" Version="9.0.0" PrivateAssets="all" />
       <PackageReference Include="SonarAnalyzer.CSharp" Version="10.6.0.109712" PrivateAssets="all" />
     </ItemGroup>
   </Project>
   ```

   NOTE: Because TreatWarningsAsErrors is enabled in Debug, the build may fail on existing warnings. If `dotnet build` fails with analyzer warnings, add a NoWarn override for specific warning codes in the individual project .csproj files that have pre-existing issues. List the specific codes. Do NOT disable TreatWarningsAsErrors globally -- per CONTEXT.md this is a locked decision.
