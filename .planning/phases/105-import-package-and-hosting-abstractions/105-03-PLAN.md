# Plan 105-03: Keep Import Packages Core-Based

## Objective

Guard that the dedicated import packages stay layered on `Videra.Core` rather than drifting toward Avalonia or backend coupling.

## Tasks

1. Add repository or reflection tests over `Videra.Import.Gltf` and `Videra.Import.Obj`.
2. Prove their public APIs stay anchored on `Videra.Core` runtime types and standard system/logging contracts without drifting toward Avalonia, platform, or chart assemblies.
3. Keep the checks simple enough that future importer work does not need ceremony to stay clean.

## Verification

- `dotnet test tests/Videra.Core.Tests/Videra.Core.Tests.csproj -c Release --no-build -m:1 --filter "FullyQualifiedName~ImportPackages"`
