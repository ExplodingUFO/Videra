# Plan 93-02: Fallback Artifact Persistence

## Objective

Ensure the consumer smoke wrapper persists actionable failure artifacts even when the smoke app exits before managed completion.

## Tasks

1. Add a failing repository test that requires the wrapper to capture session context and fallback failure artifacts.
2. Teach `Invoke-ConsumerSmoke.ps1` to write a session-environment snapshot plus fallback `consumer-smoke-result.json` / `diagnostics-snapshot.txt` on abnormal child exit or missing managed artifacts.
3. Functionally verify both failure modes with a temporary fixture: non-zero exit and zero exit without managed artifacts.
