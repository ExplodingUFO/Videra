# Waterfall Recipe

This recipe uses the shipped `VideraChartView` control and the native `Plot.Add.Waterfall` entrypoint. It is intended for data that is still a heightfield, but should read visually as a row-by-row waterfall instead of a continuous surface.

## Minimal Waterfall

```csharp
using Videra.SurfaceCharts.Avalonia.Controls;

var chart = new VideraChartView();

chart.Plot.Add.Waterfall(new double[,]
{
    { 0.10, 0.35, 0.62, 0.50 },
    { 0.18, 0.48, 0.88, 0.72 },
    { 0.25, 0.58, 1.00, 0.83 },
    { 0.20, 0.44, 0.78, 0.66 },
}, "Waterfall response");

chart.Plot.Axes.X.Label = "Sweep";
chart.Plot.Axes.X.Unit = "index";
chart.Plot.Axes.Y.Label = "Amplitude";
chart.Plot.Axes.Y.Unit = "g";
chart.Plot.Axes.Z.Label = "Band";
chart.Plot.Axes.Z.Unit = "Hz";

chart.FitToData();
```

`Plot.Add.Waterfall` accepts the same heightfield data shapes as the surface path: a `double[,]`, a `SurfaceMatrix`, or an `ISurfaceTileSource`. Use the tiled source overload when the dataset is already cache-backed or generated by the surface pyramid builder; use the matrix overload for small in-memory examples and tests.

## Production Setup

```csharp
using Videra.SurfaceCharts.Avalonia.Controls;
using Videra.SurfaceCharts.Core;

SurfaceMatrix matrix = CreateWaterfallMatrix();

var chart = new VideraChartView();
chart.Plot.Clear();

var series = chart.Plot.Add.Waterfall(matrix, "Machine response");

chart.Plot.Axes.X.Label = "Distance";
chart.Plot.Axes.X.Unit = "mm";
chart.Plot.Axes.Y.Label = "Velocity";
chart.Plot.Axes.Y.Unit = "mm/s";
chart.Plot.Axes.Z.Label = "Frequency";
chart.Plot.Axes.Z.Unit = "Hz";

chart.Plot.Axes.X.SetBounds(0, 120);
chart.Plot.Axes.Z.SetBounds(0, 8_000);
chart.Plot.Axes.Y.AutoScale();

chart.Plot.OverlayOptions = new SurfaceChartOverlayOptions
{
    ShowMinorTicks = true,
    MinorTickDivisions = 4,
    GridPlane = SurfaceChartGridPlane.XZ,
    AxisSideMode = SurfaceChartAxisSideMode.Auto,
    TickLabelFormat = SurfaceChartNumericLabelFormat.Engineering,
    TickLabelPrecision = 2,
    LegendLabelFormat = SurfaceChartNumericLabelFormat.Engineering,
    LegendLabelPrecision = 2,
    LegendTitleOverride = series.Name,
};

chart.FitToData();
```

`Plot.Axes.X`, `Plot.Axes.Y`, and `Plot.Axes.Z` are chart-local facades. Label and unit assignments update the same overlay options used by rendered axes, probes, support summaries, and copied evidence. `SetBounds(...)` constrains future limit and autoscale calls for that axis. `AutoScale()` uses the active series metadata while respecting any bounds that remain set.

## View State

`VideraChartView.ViewState` is the persisted camera and data-window contract for surface-family charts, including waterfall. A host can save it, restore it later, and still use `FitToData()` when the active waterfall dataset changes.

```csharp
var savedViewState = chart.ViewState;

// Rebuild or replace the waterfall data.
chart.Plot.Clear();
chart.Plot.Add.Waterfall(CreateWaterfallMatrix(), "Updated response");

// Restore the user's camera when it still applies.
chart.ViewState = savedViewState;

// Or intentionally recenter on the active dataset.
chart.FitToData();
```

Use `FitToData()` when changing to a materially different waterfall shape or range. Use `ViewState` when preserving an operator's camera, zoom window, and display-space choice is more important than immediately showing the full dataset.
